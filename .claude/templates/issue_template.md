# 이슈 등록 템플릿

> 중앙 안내: 상세한 에이전트 활용 원칙 및 공통 가이드라인은 `.claude/guides/common_guide.md`의 '2. 에이전트 활용 원칙' 섹션을 참조하십시오.

## d0004_todo.md 등록용 형식

### 진행 중인 이슈

```markdown
#### 2.2.X [이슈 제목] [우선순위]
- **파일**: `파일 경로`
- **위치**: `함수명:라인번호`
- **문제**: 문제 설명
- **영향**: 영향 범위 설명
- **조치**: 권장 해결 방안
```

### 해결된 이슈

```markdown
#### 8.X.X [이슈 제목] - 해결됨 (YYYY-MM-DD)
- **파일**: `파일 경로`
- **원인**: 원인 설명
- **해결**: 해결 방법
- **방지 대책**: 향후 방지 방안
```

## d0010_history.md 등록용 형식

### 버그 수정

```markdown
### [BUGFIX] 이슈 제목
- **파일**: 수정된 파일 경로
- **원인**: 문제 원인
- **해결**: 적용된 해결 방법
- **날짜**: YYYY-MM-DD
```

### 핫픽스

```markdown
### [HOTFIX] 긴급 이슈 제목
- **파일**: 수정된 파일 경로
- **원인**: 문제 원인
- **해결**: 적용된 해결 방법
- **영향**: 영향 범위
- **날짜**: YYYY-MM-DD
```

### 개선

```markdown
### [IMPROVE] 개선 제목
- **파일**: 수정된 파일 경로
- **변경**: 변경 내용
- **이유**: 개선 이유
- **날짜**: YYYY-MM-DD
```

---

## 우선순위 분류 가이드

| 분류 | 설명 | 대응 시간 | 이력 태그 |
|------|------|----------|----------|
| [CRITICAL] | 시스템 장애, 보안 취약점, 데이터 손실 | 즉시 | [HOTFIX] |
| [ERROR] | 기능 오작동, 예외 미처리 | 24시간 내 | [BUGFIX] |
| [WARNING] | 잠재적 문제, 성능 이슈 | 1주일 내 | [IMPROVE] |
| [INFO] | 코드 스타일, 개선 권장 | 백로그 | [ENHANCE] |

## 예시

### P1 이슈 예시

```markdown
#### 2.2.1 [SQL 인젝션 취약점] [CRITICAL]
- **파일**: `src/api/user.py`
- **위치**: `get_user:45`
- **문제**: 사용자 입력이 SQL 쿼리에 직접 삽입됨
- **영향**: 데이터베이스 전체 노출 위험
- **조치**: 파라미터 바인딩 사용
```

### P2 이슈 예시

```markdown
#### 2.2.2 [Bare Except 패턴] [ERROR]
- **파일**: `core/frame_composer.py`
- **위치**: `_get_system_font:38-41`
- **문제**: except: 구문이 모든 예외를 무시함
- **영향**: 디버깅 어려움, 예상치 못한 동작
- **조치**: except OSError: 로 구체화
```

### P3 이슈 예시

```markdown
#### 2.2.3 [deprecated 라이브러리 사용] [WARNING]
- **파일**: `scripts/rearrange.py`
- **위치**: `import six`
- **문제**: Python 3에서는 six 라이브러리 불필요
- **영향**: 불필요한 의존성
- **조치**: six.iteritems() → .items() 변경
```
