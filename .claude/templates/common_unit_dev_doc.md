# 공통 단위개발문서 템플릿 (common_unit_dev_doc)

## 사용법

이 템플릿은 페이지/모듈/API/알고리즘 등 개발 단위별 상세 문서 작성용 **공통 템플릿**입니다.
특수한 유형(알고리즘, API 전용 등)은 별도 템플릿을 생성하여 사용합니다.

**파일명 규칙**: `00_doc/d{문서번호}_{메뉴명}_단위개발.md`

**문서번호 규칙** (SP=서브프로젝트):
```
d[SP][1][ID뒷자리2자리]0
예: d21000 = SP(2) + 단위개발(1) + ID(00) + 접미사(0)
```

## 섹션 연계 구조

> **핵심 원칙**: 앞 섹션의 내용을 기반으로 뒤 섹션을 작성

```
1. 개요 ─────► 2. 기획 ─────► 3. 설계 ─────► 4. 구현 ─────► 5. 검증
   │              │              │              │              │
   │              │              │              │              │
   ▼              ▼              ▼              ▼              ▼
 목적/아이템    아이템→기능    기능→아키텍처   설계→코드     구현→테스트
 정의          시나리오 도출   데이터 구조    파일/함수     케이스 검증
```

| 섹션 | 입력 (이전 섹션) | 출력 (현재 섹션) |
|------|-----------------|-----------------|
| 1. 개요 | - | 목적, 핵심 가치, 아이템, 메모 |
| 2. 기획 | 아이템 목록 | 기능 요구사항, 시나리오, UI/UX |
| 3. 설계 | 기능/시나리오 | 아키텍처, 데이터 구조, API |
| 4. 구현 | 설계 명세 | 파일 구조, 함수, 의존성, DB |
| 5. 검증 | 시나리오/구현 | 테스트 케이스, 검증 결과 |
