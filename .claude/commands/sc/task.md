## 1. 개요

지능적인 워크플로우 관리 및 크로스 세션 지속성을 갖춘 복잡한 태스크를 실행합니다.

## 2. 사용법

```
task [action] [target] [--strategy systematic|agile|enterprise] [--persist] [--hierarchy]
```

## 3. 액션

| 액션 | 설명 |
|------|------|
| `create` | 프로젝트 수준 태스크 계층 구조 생성 |
| `execute` | 지능적 오케스트레이션으로 태스크 실행 |
| `status` | 세션 간 태스크 상태 확인 |
| `analytics` | 태스크 성능 및 분석 대시보드 |
| `optimize` | 태스크 실행 전략 최적화 |
| `delegate` | 여러 에이전트에 태스크 위임 |
| `validate` | 증거 기반 태스크 완료 검증 |

## 4. 인자 및 옵션

| 인자/옵션 | 설명 |
|-----------|------|
| `target` | 태스크 설명, 프로젝트 범위 또는 기존 태스크 ID |
| `--strategy` | 실행 전략 (systematic, agile, enterprise) |
| `--persist` | 크로스 세션 태스크 지속성 활성화 |
| `--hierarchy` | 계층적 태스크 분해 생성 |
| `--delegate` | 다중 에이전트 태스크 위임 활성화 |
| `--wave-mode` | 웨이브 기반 실행 활성화 |
| `--validate` | 품질 게이트 및 검증 적용 |

## 5. 실행 전략

### Systematic 전략 (기본)
1. **발견 단계**: 포괄적 프로젝트 분석 및 범위 정의
2. **계획 단계**: 의존성 매핑과 계층적 태스크 분해
3. **실행 단계**: 검증 게이트와 순차적 실행
4. **검증 단계**: 증거 수집 및 품질 보증
5. **최적화 단계**: 성능 분석 및 개선 권장사항

### Agile 전략
1. **스프린트 계획**: 우선순위 기반 태스크 구성
2. **반복 실행**: 지속적 피드백과 짧은 주기
3. **적응형 계획**: 결과에 따른 동적 태스크 조정
4. **지속적 통합**: 실시간 검증 및 테스트
5. **회고 분석**: 학습 및 프로세스 개선

### Enterprise 전략
1. **이해관계자 분석**: 다중 도메인 영향 평가
2. **리소스 할당**: 태스크 간 최적 리소스 분배
3. **리스크 관리**: 포괄적 리스크 평가 및 완화
4. **컴플라이언스 검증**: 규정 및 정책 준수 점검
5. **거버넌스 보고**: 상세 진행 상황 및 준수 보고

## 6. 고급 기능

### 태스크 계층 관리
| 레벨 | 범위 | 기간 |
|------|------|------|
| Epic | 대규모 프로젝트 목표 | 주~월 |
| Story | 기능별 구현 | 일~주 |
| Task | 구체적 실행 항목 | 시간~일 |
| Subtask | 세부 구현 단계 | 분~시간 |

### 지능형 태스크 오케스트레이션
- **의존성 해결**: 자동 의존성 감지 및 순서화
- **병렬 실행**: 독립 태스크 병렬화
- **리소스 최적화**: 지능적 리소스 할당 및 스케줄링
- **컨텍스트 공유**: 태스크 간 컨텍스트 및 지식 공유

### 크로스 세션 지속성
- **태스크 상태 관리**: 세션 간 지속적 태스크 상태
- **컨텍스트 연속성**: 보존된 컨텍스트 및 진행 추적
- **히스토리 분석**: 태스크 실행 이력 및 학습
- **복구 메커니즘**: 중단으로부터 자동 복구

## 8. 사용 예시

```bash
# 프로젝트 수준 태스크 계층 생성
task create "사용자 인증 시스템 구현" --hierarchy --persist --strategy systematic

# 다중 에이전트 위임으로 실행
task execute AUTH-001 --delegate --wave-mode --validate

# 분석 및 최적화
task analytics --project AUTH --optimization-recommendations

# 크로스 세션 태스크 관리
task status --all-sessions --detailed-breakdown
```
