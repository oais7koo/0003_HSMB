## 1. 개요

oobook은 도서 및 유튜브 영상에 대한 상세 서머리를 자동 생성하는 스킬입니다.

### 1.1 핵심 기능

- 도서 서머리 생성 (7개 섹션, 약 50KB)
- 유튜브 서머리 생성 (8개 단계)
- 서머리 → Word 문서 변환
- 배치 처리 및 진행 상태 관리

### 1.2 참조 문서

- `01_algorithm/ps9003_park_summary_v03.md` (마스터 문서)
- `01_algorithm/ps9003e_도서서머리생성.md` (도서 서머리 명령어)
- `01_algorithm/ps9003f_도서서머리워드문서생성.md` (워드 변환 명령어)
- `01_algorithm/ps9003c_스크립트.md` (스크립트 라이브러리)

## 2. 워크플로우

### 2.1 도서 서머리 생성 흐름

```
1. 진행 상태 확인 (ps9003_park_summary_v03.md 섹션 4)
    ↓
2. 첫 번째 미완성 도서 선택
    ↓
3. 7개 섹션 순차 생성 및 저장
    ↓
4. 진행 상태 업데이트 (완성본 +1, 미완성 -1)
    ↓
5. 다음 도서로 자동 진행
```

### 2.2 서브에이전트 활용

| 작업 | 서브에이전트 | 용도 |
|------|--------------|------|
| 책 정보 검색 | `Explore` | 도서 정보 탐색 |
| 섹션 생성 | `task-executor` | 각 섹션 콘텐츠 작성 |
| 완성도 검증 | `task-checker` | 섹션별 완성도 확인 |
| 문서 변환 | `task-executor` | MD → DOCX 변환 |

## 3. 상세 사용법

### 3.1 명령어

| 명령어 | 설명 | 역할 |
|--------|------|------|
| `oobook` | 도서 서머리 생성 (기본) | 미완성 도서 순차 처리 |
| `oobook version` | 스킬 버전 정보 (v02) | 버전 확인 |
| `oobook youtube` | 유튜브 서머리 생성 | 미완성 영상 순차 처리 |
| `oobook word` | 서머리 → Word 변환 | MD → DOCX 변환 |
| `oobook status` | 진행 상태 확인 | 완성/미완성 현황 |
| `oobook verify` | 완성도 검증 | 섹션별 검증 |

### 3.2 옵션

| 옵션 | 설명 |
|------|------|
| `--title <도서명>` | 특정 도서만 처리 |
| `--index <번호>` | 특정 인덱스 항목 처리 |
| `--batch` | 전체 배치 처리 |
| `--section <N>` | 특정 섹션만 재생성 |
| `--force` | 기존 파일 덮어쓰기 |

### 3.3 입출력 경로

#### 도서 서머리

**입력**:
- Excel 파일: `data/ps1001_박기성/BOOKQAI.xlsx`
  - Q1 시트: 도서명 목록 (A열)
  - Q2 시트: 섹션 주제 (A열 - 7개 섹션)
- 샘플 파일: `data/ps1001_박기성/노화도 설계하는 시대가 온다.md`

**출력**:
- 출력 폴더: `data/ps9003/01_도서/`
- 파일 형식: `[도서명].md`
- 파일 크기: 약 50KB

#### 유튜브 서머리

**입력**:
- Excel 파일: `data/ps1001_박기성/BOOKQAI.xlsx`
  - YUTLIST 시트: 유튜브 URL 목록
  - Q2 시트: 단계별 주제 (B열 - 8개 단계)

**출력**:
- 출력 폴더: `data/ps9003/02_유튜브/`
- 파일 형식: `[번호]_[영상제목].md`

## 4. 사용 예시

### 4.1 도서 서머리 생성

```bash
# 기본 실행 (미완성 도서 자동 처리)
oobook

# 특정 도서만 처리
oobook --title "깊은 생각의 비밀"

# 배치 처리
oobook --batch

# 특정 섹션만 재생성
oobook --title "도서명" --section 3
```

### 4.2 유튜브 서머리 생성

```bash
# 미완성 유튜브 영상 처리
oobook youtube

# 특정 영상만 처리
oobook youtube --index 1
```

### 4.3 Word 변환

```bash
# 서머리 → Word 문서 변환
oobook word

# 입력: data/ps9003/01_도서/*.md
# 출력: data/ps9003/02_워드/*.docx
```

### 4.4 진행 상태 확인

```bash
# 현황 조회
oobook status

# 완성도 검증
oobook verify
```

### 4.5 backup/restore

```bash
# 백업 경로 설정
uv run python .claude/skills/oobook/scripts/oobook_backup.py config --path "C:/path/to/backup"
uv run python .claude/skills/oobook/scripts/oobook_backup.py config --list

# 대상 확인 (dry-run)
oobook backup --dry-run
oobook restore --dry-run

# 전체 백업/복원
oobook backup
oobook restore

# 특정 분류만 백업
oobook backup --code BK0200
oobook backup --subdir 01_book_ko

# 현황 확인
uv run python .claude/skills/oobook/scripts/oobook_backup.py status
```

## 5. 7개 섹션 구조 (도서)

| 섹션 | 제목 | 설명 |
|------|------|------|
| 1 | 각 장별로 핵심내용 정리 | 장별 30줄 내외, 핵심키워드별 3줄 내외 |
| 2 | 핵심 통찰과 울림을 주는 구절 30개 | 맥락/의미/원문 + 5분 낭독 스크립트 |
| 3 | 핵심키워드 정리 (30분 서사적 강의용) | 문제제기/원인분석/해결전략/삶의변화 |
| 4 | 강의안 작성 | 시간 배분, 서사적 전개, 청중 참여 |
| 5 | 강의전문 (Full Script) | 4,500~5,500자 분량 연설문 |
| 6 | 구글 슬라이드 작성 가이드 | 15-20장 분량 프레젠테이션 설계 |
| 7 | 플래시카드 및 퀴즈 | 플래시카드 20개, 퀴즈 18문항 |

## 7. AI 실행 규칙

### 7.1 필수 실행 원칙

- ✅ 명령어 인식 즉시 첫 번째 도서 섹션 1부터 시작
- ✅ 섹션 7까지 완료 → 자동으로 다음 도서
- ✅ 각 섹션 완료 후 파일 저장 → 다음 섹션 자동 진행
- ✅ **반드시 섹션별로 순차 생성** (토큰 제한 회피)

### 7.2 절대 금지 사항

- ❌ "몇 권만 하시겠습니까?" 질문
- ❌ "시범적으로 3권만 해볼까요?" 제안
- ❌ 작업 범위 축소 제안
- ❌ 간략 버전 제시
- ❌ 옵션 선택지 제시

### 7.3 토큰 관리

- **문제**: 전체 서머리 한 번에 생성 시 토큰 제한(32,000) 초과
- **해결**: 반드시 섹션별로 순차 생성
- **방법**: 각 섹션 완료 후 파일에 저장

## 8. Word 문서 변환

### 8.1 스타일 설정

| 항목 | 설정값 |
|------|--------|
| 용지 크기 | A4 |
| 여백 | 상하좌우 2.5cm |
| 한글 폰트 | 맑은 고딕 |
| 영문 폰트 | Calibri |
| 본문 크기 | 11pt |
| 줄간격 | 1.5줄 |

### 8.2 변환 규칙

| Markdown | Word 스타일 |
|----------|------------|
| `# 제목` | 제목 1 (18pt, 굵게) |
| `## 제목` | 제목 2 (16pt, 굵게) |
| `### 제목` | 제목 3 (14pt, 굵게) |
| `- 항목` | 글머리 기호 |
| `1. 항목` | 번호 매기기 |
| `**굵게**` | 굵게 |
