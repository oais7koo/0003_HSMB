# 월간보고

## meta
- version: v01
- description: 사내 월간보고 작성 4단계 파이프라인 (양식 추출 → 양식·참고 분석 → 섹션별 인터뷰 → HWP 붙여넣기용 MD 생성)

## folder_structure
- 기본 세트: 01_양식, 02_참고, 03_작성
- 01_양식: HWP/HWPX 양식 파일 (사용자 제공)
- 02_참고: HWP/HWPX 과거 작성본 (사용자 제공, 스타일 학습용)
- 03_작성: 월간보고 산출물 저장 위치
- 네이밍: 산출물 = `{YY-MM}_월간보고({부서명}).md`

## 작성 원칙
- **양식 절대 준수**: 01_양식 구조·섹션 순서·범례를 그대로 유지
- **단순 명료**: 사실 중심, 형용사·수식어 최소화
- **들여쓰기 트리**: 02_참고의 `- 항목 → - 세부` 2단 깊이 패턴 적용
- **상태 표기**: □(신규) / ◨(진행중) / ■(완료) 일관 적용
- **HWP 붙여넣기 즉시성**: 산출물의 본문은 양식의 좌(실적)·우(계획) 셀에 그대로 붙여넣을 수 있어야 함
- **인용·출처 미포함**: 자기 활동 보고이므로 외부 인용 없음, 본문 그대로 양식에 투입

## steps

### step.0 양식·참고 추출
- tool: oowork extract (.claude/skills/oowork/scripts/hwp_to_md.py)
- input: 01_양식/*.hwpx, 02_참고/*.hwp
- output:
    form: 01_양식/{N}_*.md
    refs: 02_참고/{N}_*.md
- confirm: false (자동 실행)
- skip_if: 동일 이름 .md 이미 존재
- sub_steps:
  1. 01_양식 폴더 내 HWP/HWPX 파일 자동 감지
  2. hwp_to_md.py 실행 → 양식 MD 생성 (넘버링 +1)
  3. 02_참고 폴더 내 HWP/HWPX 파일 자동 감지
  4. hwp_to_md.py 실행 → 참고본 MD 생성 (넘버링 +1)
  5. 변환 중간물(HWPX/PDF)은 보존, 임시 파일만 정리

### step.1 양식·참고 분석
- tool: 직접 분석 (Read)
- input: ${step.0.form}, ${step.0.refs}
- output: 메모리 (섹션 구조 + 스타일 패턴)
- depends: step.0
- confirm: false (자동)
- sub_steps:
  1. **양식 골격 추출** — 헤더 메타필드(업무기간/보고자/작성자/보고일), 제목 패턴(`{소속} ’{YY}년 {MM}월 보고`), 섹션 트리(1.실적및계획 / 2.이슈사항 / 3.기타사항)
  2. **범례 확인** — □(신규), ◨(진행중), ■(완료)
  3. **섹션 구조 확인** — 1번 섹션의 좌(실적) / 우(계획안) 2단 구조
  4. **참고본 스타일 학습** — 부서별 분류 패턴(정보화사업/수주현황/사업준비 등), 들여쓰기 트리 깊이, 표·수치 활용 패턴, 항목명 명명 규칙
  5. **양식과 참고본 차이 정리** — 양식이 빈 골격이고 참고본은 채워진 사례임을 인지

### step.2 사용자 인터뷰 (섹션별 단계적)
- tool: 직접 질의 응답 (turn-by-turn)
- input: ${step.1}
- output: 인터뷰 응답 누적 (메모리)
- depends: step.1
- confirm: each_section (각 섹션 응답 후 다음 섹션으로 이동 확인)
- sub_steps:
  1. **메타 정보** — 한 번에 질문
     - 소속 (본부/지사/팀명)
     - 업무기간 (YYYY.MM.DD ~ YYYY.MM.DD)
     - 보고자 (성명 + 직급)
     - 작성자 (성명 + 직급, 보고자와 동일 시 자동 채움)
     - 보고일 (YYYY.MM.DD)
  2. **실적 (좌측 컬럼)** — 항목 단위 반복 질문
     - 항목 분류명 (예: "정보화사업", "수주현황", "사업 준비")
     - 항목별: 사업/업무명 + 상태(□◨■) + 세부 작업 리스트
     - "항목 추가하시겠습니까?" 반복 (없음 → 다음 섹션)
  3. **계획 (우측 컬럼)** — 항목 단위 반복 질문 (실적과 동일 패턴, 미래형)
     - 진행 예정·신규 사업 위주
     - 실적 항목과 1:1 매칭이 자연스러우면 매칭, 아니면 독립
  4. **이슈 사항** — 자유 입력 (없으면 "특이사항 없음")
  5. **기타 사항** — 자유 입력 (없으면 "특이사항 없음")
  6. 응답 요약 출력 → 사용자 최종 확인

### step.3 보고서 작성
- tool: 직접 작성 (Write)
- input: ${step.2} 인터뷰 응답, ${step.1} 양식 골격, templates/월간보고/10_월간보고_템플릿.md
- output: {workdir}/03_작성/{YY-MM}_월간보고({부서명}).md
- depends: step.2
- confirm: true (저장 전 사용자 확인)
- sub_steps:
  1. 템플릿 로드 → 메타 헤더 채우기 (업무기간/보고자/작성자/보고일)
  2. 제목 작성 — `{소속} ’{YY}년 {MM}월 보고`
  3. **1. 실적 및 계획** 작성
     - 좌측: 실적 항목 → 분류 → 사업명 + 상태 + 세부 작업 트리
     - 우측: 계획 항목 → 분류 → 사업명 + 상태 + 세부 작업 트리
     - 범례 헤더 유지 (`* 범례 (□ : 신규, ◨ : 진행중, ■ : 완료)`)
  4. **2. 이슈 사항** 작성 — `ㅇ` 머리표 사용
  5. **3. 기타 사항** 작성 — `ㅇ` 머리표 사용
  6. 산출물 저장 — 03_작성/ 하위 파일명 규칙 적용
  7. 사용자에게 산출물 경로 + 미리보기 출력 → "HWP 양식에 그대로 붙여넣을 수 있습니다" 안내
