- description: RFP 분석부터 제안서 작성까지 5단계 기본 파이프라인 (확장 가능)

## folder_structure
- 기본 세트: 01_RFP, 02_자료, 03_기획, 04_기술연구, 05_제안작성
- 확장: 06_검토, 07_최종, 08_부록 등 필요 시 추가
- 네이밍: {폴더번호}{순번}_{내용}.md (예: 11_양식분석.md)
- N 할당: 기존 파일 최대 번호 + 1

## steps

### step.0 RFP 분석
- tool: 직접 작성
- input: RFP 문서 (공고문, 과업지시서, 제안요청서, 양식)
- output: {workdir}/01_RFP/11_RFP분석.md, {workdir}/01_RFP/12_양식분석.md
- confirm: true
- sub_steps:
  1. RFP 문서 전문 읽기 (HWP → HWPX → PDF → MD 변환 후 읽기, PDF → Read/pdfplumber)
  2. 원본 문서 목록 테이블 작성 (11_RFP분석.md 상단 필수) — 파일명, 유형(공고/과업/양식), 페이지수, 분석여부(O/X)
  3. 핵심 요구사항 추출 (사업 개요, 목적, 범위, 일정, 예산)
  4. 평가 기준 분석 (배점표, 가점/감점 항목) — 해당 시
  5. 필수 제출 서류·양식 항목 목록 정리
  6. 기술 요구사항 항목별 분해
  7. 제약 조건 정리 (기간, 인력, 장비, 기술 스택)
  8. 차별화 포인트 도출
  9. RFP 분석 결과 문서 생성 (11_RFP분석.md)
  10. 제안서 양식 분석 문서 생성 (12_양식분석.md) — 양식별 필드 목록, 입력 형식(텍스트/표/체크), 글자수 제한, 필수/선택 구분

### step.1 자료 수집
- tool: 직접 작성 + ooresearch (필요 시)
- input: ${step.0.output}
- output: {workdir}/02_자료/2N_{자료명}.md
- depends: step.0
- confirm: true
- sub_steps:
  1. 기업 기본 정보 정리 (개요, 대표자, 사업장, 연혁)
  2. 경영진·조직·주주 현황 정리
  3. 매출·거래처·재무 현황 정리
  4. 기술 실적·인증·지재권 정리
  5. 유사 사업 수행 실적 정리
  6. 기타 참고자료 수집 (기존 제안서, 논문, 기술문서)

### step.2 기획
- tool: 직접 작성
- input: ${step.0.output}, ${step.1.output}
- output: {workdir}/03_기획/31_작성전략.md
- depends: step.0, step.1
- confirm: true
- sub_steps:
  1. 제안서 목차·구조 설계 (양식 섹션 매핑)
  2. 평가항목-목차 매핑표 작성 — 해당 시
  3. 배점 비중별 분량 배분 (장별 페이지 + 총 페이지 합계 표시) — 해당 시
  4. 장별 핵심 메시지 1줄 정의
  5. 그림/표/다이어그램 배치 계획
  6. 작성 마일스톤 (초안→검토→수정→최종)
  7. 참조 자료-섹션 매핑

### step.3 기술 연구
- tool: 직접 작성 + ooresearch/oopaper (필요 시)
- input: ${step.2.output}, ${step.1.output}
- output: {workdir}/04_기술연구/41_{주제}.md
- depends: step.2
- confirm: true
- rules: 차별화 강조, 정량 근거 포함, 경쟁 비교
- sub_steps:
  1. 기술 현황 분석 (동향, 문제점, 개선 방향)
  2. 핵심 기술 상세 (아키텍처, 알고리즘, 방법론)
  3. 경쟁사·경쟁 기술 비교 (품질/기술/가격)
  4. 차별화 포인트 정리 (최소 3개)
  5. 기술 검증·사업화 계획
  6. SOTA/선행연구 조사 — 필요 시

### step.4 제안 작성
- tool: 직접 작성 + ooword/oopdf (최종 변환)
- input: ${step.2.output}, ${step.3.output}, ${step.1.output}
- output: {workdir}/05_제안작성/51_{섹션명}.md
- depends: step.2, step.3
- confirm: true
- rules: **RFP/양식의 입력 형식에 맞는 아웃풋 생성** (기획·검토는 04에서 완료)
- output_principle: |
    05_제안작성의 산출물은 **양식에 직접 입력(붙여넣기)할 수 있는 형태**로 작성한다.
    - 마크다운 표/헤더 등 기획용 서식이 아닌, **양식 필드별 입력값** 형태
    - 각 칸에 들어갈 텍스트를 필드명: 값 형태로 정리
    - 장문 기재란(기술내용 등)은 **코드블록(```)으로 감싸서** HWP 붙여넣기 영역을 명확히 구분
    - 체크박스 항목은 선택 결과만 명시
- writing_style: 사용자 선택 (references/writing_styles.md에서 선택 또는 신규 등록)
- writing_style_default: S01 (개조식 3단)
- sub_steps:
  1. **본문 스타일 선택** — references/writing_styles.md의 등록 스타일 테이블을 표시하고 사용자에게 선택 요청. 미선택 시 기본값(S01) 적용. 신규 스타일 등록 시 writing_styles.md에 추가.
  2. RFP/양식 필드 목록 확인 (01_RFP/12_양식분석.md 기준)
  3. 필드별 입력값 작성 (양식 형식에 맞게)
  4. 장문 기재란 본문 작성 (HWP 붙여넣기용, 선택한 본문 스타일 적용)
  5. 양식 항목 전수 대응 검증 (누락 0건)
  6. 용어 일관성·수치 정합성 검수
  7. 오탈자/문법 검수
  8. hwp 붙여넣기용 txt 추출 — 해당 시
  9. (선택) Word/PDF 변환 — `ooword run` 또는 `oopdf run`

## validation
- step.0: 분석 파일(11_RFP분석.md) + 양식분석 파일(12_양식분석.md) 존재 + 요구사항·양식 항목 추출 완료
- step.1: 자료 파일 1개 이상 존재 + 핵심 정보 기재
- step.2: 기획 파일 존재 + 구조/목차 정의 + 마일스톤 설정
- step.3: 기술연구 파일 존재 + 차별화 포인트 3개 이상 + 정량 근거
- step.4: 본문 파일 존재 + 양식 항목 전수 대응 + 통합본 생성
