# 소개서 체크리스트

> 공통 규칙 상속: `../checklist_common.md` (G01~G18)
> 이 문서는 소개서 프로세스에 특화된 추가 규칙만 정의

## step.0 자료 수집

| # | 규칙 | 검증 방법 |
|---|------|---------|
| A01 | 소개 대상 명확 | 회사/프로젝트/기술/제품 중 대상이 명시됨 |
| A02 | 핵심 수치 존재 | 성능/규모/비용 등 정량적 수치 3개 이상 |
| A03 | 대상 청중 정의 | 누구에게 보여줄 것인지 명시됨 |
| A04 | 기존 문서 참조 | PRD/README/결과물 등 소스 문서 목록 존재 |

## step.1 콘텐츠 구성

| # | 규칙 | 검증 방법 |
|---|------|---------|
| B01 | 목차/슬라이드 구성안 | 전체 구조가 정의됨 |
| B02 | 항목별 핵심 메시지 | 각 슬라이드/섹션에 1줄 메시지 존재 |
| B03 | 결과 중심 검증 | 과정/방법론이 아닌 결과/산출물 중심인가 |
| B04 | 수치 근거 매핑 | 핵심 메시지에 뒷받침 수치가 연결됨 |

## step.2 원고 작성

| # | 규칙 | 검증 방법 |
|---|------|---------|
| C01 | PPT 원고 형식 | 슬라이드별 제목/본문/발표포인트 구조 |
| C02 | 수치 정확성 | 원본 데이터와 원고 수치 일치 |
| C03 | 톤&매너 통일 | 간결/수치중심/전문적 톤 일관 |
| C04 | 10슬라이드 이내 | 표지+Q&A 포함 10~12장 이내 |

## step.3 산출물 제작

| # | 규칙 | 검증 방법 |
|---|------|---------|
| D01 | PPTX 생성 | ooppt로 정상 생성됨 |
| D02 | 레이아웃 확인 | 텍스트 잘림/겹침 없음 |
| D03 | 파일 위치 | 04_산출물/ 폴더에 저장됨 |
