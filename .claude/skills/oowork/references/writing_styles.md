# 본문 스타일 레지스트리

> oowork 제안 step.4에서 사용할 본문 스타일 목록. 사용자가 선택하거나 새로 등록할 수 있다.

## 등록된 스타일

### S01. 개조식 3단 (기본)

- **용도**: 기술사업계획서, 정부과제 제안서, 공공사업 제안
- **구조**:
  - 1단(□): 대항목 — 핵심 주제/문제
  - 2단(○): 중항목 — 세부 설명/근거
  - 3단(-): 소항목 — 구체적 내용/수치
- **어미**: "~임", "~수행", "~불가" (서술식 금지)
- **원칙**: 불필요한 접속사·수식어 최소화, 한 항목에 하나의 핵심 메시지
- **예시**:
  ```
  □ 사내 코드베이스 특화 AI 코딩 어시스턴트 개발
    ○ 사내 코드를 학습하여 코딩 컨벤션과 아키텍처에 맞는 코드 자동 생성
      - GraphRAG 기반 의미론적 코드 검색으로 기존 벡터 검색 대비 정확도 향상
  ```

### S02. 서술식 (논문/보고서)

- **용도**: 연구보고서, 기술보고서, 백서, 학술 제안
- **구조**: 문단 중심, 제목-본문-결론 흐름
- **어미**: "~이다", "~한다", "~있다" (정중체)
- **원칙**: 논리적 흐름 (배경→문제→해결→기대효과), 정량 근거 인용, 참고문헌 표기
- **예시**:
  ```
  본 사업은 사내 코드베이스에 특화된 AI 코딩 어시스턴트를 개발하는 것을 목적으로 한다.
  기존 벡터 검색 방식의 한계를 극복하기 위해 GraphRAG 기반 의미론적 코드 검색 기술을
  적용하여 코드 검색 정확도를 향상시킨다.
  ```

### S03. 개조식 2단 (간결)

- **용도**: 요약서, 발표자료 원고, 견적 설명, 짧은 제안
- **구조**:
  - 1단(■): 항목명
  - 2단(-): 세부 내용
- **어미**: "~임", "~완료", "~예정" (명사형 종결)
- **원칙**: 항목당 2줄 이내, 핵심만 기술
- **예시**:
  ```
  ■ AI 코딩 어시스턴트 개발
    - GraphRAG 기반 코드 검색, 정확도 30% 향상 목표
    - 개발 기간: 6개월, 투입 인력: 3명
  ```

### S04. 혼합식 (서술+개조)

- **용도**: 민간 제안서, 사업계획서, 투자 제안, 기획서
- **구조**: 도입부 서술(2~3문장) + 핵심 내용 개조식
- **어미**: 서술부 "~합니다" (경어체), 개조부 "~임", "~예정"
- **원칙**: 도입부에서 맥락 설명, 개조식에서 핵심 전달
- **예시**:
  ```
  사내 코드베이스의 규모가 증가함에 따라 개발자의 코드 검색 및 재사용에
  소요되는 시간이 전체 개발 시간의 30%를 차지하고 있습니다.
  이를 해결하기 위해 다음과 같은 AI 코딩 어시스턴트를 개발합니다.

  □ 핵심 기능
    ○ GraphRAG 기반 의미론적 코드 검색
      - 기존 벡터 검색 대비 정확도 30% 향상
    ○ 코딩 컨벤션 자동 적용
      - 사내 아키텍처 패턴 학습 기반
  ```

## 스타일 등록 형식

새 스타일을 등록할 때 아래 형식을 따른다:

```markdown
### S{번호}. {스타일명}

- **용도**: {적합한 문서 유형}
- **구조**: {계층 구조 또는 문단 구조 설명}
- **어미**: {종결 어미 규칙}
- **원칙**: {핵심 작성 원칙}
- **예시**:
  ```
  {실제 예시 텍스트}
  ```
```

번호(S{N})는 기존 최대 번호 + 1로 할당한다.
