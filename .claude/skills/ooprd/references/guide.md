## 1. 개요

ooprd는 PRD(Product Requirements Document) 생성 및 정합성 검증을 담당하는 스킬입니다. 프로젝트의 요구사항을 명확히 정의하고, 코드베이스 및 관련 문서와의 일관성을 유지합니다.

### 1.1 핵심 역할

- **PRD 생성**: 프로젝트 유형에 맞는 템플릿 기반 PRD 작성
- **정합성 검증**: 코드, 계획, 테스트 문서와 PRD 간 일치 여부 확인
- **모호성 해소**: Clarify 기능으로 불명확한 요구사항 명확화
- **단위개발문서 현행화**: 개요 기준 전체 섹션 정합성 유지

### 1.2 출력 문서

| 문서 | 역할 | 비고 |
|------|------|------|
| `d{SP}0001_prd.md` | 프로젝트 요구사항 정의서 | 주 출력 |
| `d{SP}0004_todo.md` | 정합성 불일치 이슈 등록 | 부 출력 |
| `d{SP}1XXX_*_단위개발.md` | 단위개발문서 | unitdev 서브명령어 |

### 1.3 서브프로젝트 컨텍스트

- **기본(SP=00)**: `d0001_prd.md`
- **서브프로젝트(SP=02)**: `d20001_prd.md`
- **설정**: `--sp N` 옵션 또는 `oocontext N` 명령어

## 2. 워크플로우

### 2.1 전체 흐름도

```

              │   │                             │
              │   └─ 있음 → 정합성 검증         │
              │             ├─ PRD 로드         │
              │             ├─ 코드/문서 스캔   │
              │             ├─ 비교 분석        │
```

### 2.2 신규 PRD 생성 워크플로우

```
ooprd run (PRD 없음)
    │
    ├─► 1. 프로젝트 정보 수집
    │      ├─ 프로젝트명
    │      ├─ 프로젝트 유형 (웹앱/알고리즘/에이전트)
    │      ├─ 핵심 기능
    │      └─ 기술 스택
    │
    ├─► 2. 템플릿 선택
    │      ├─ Type A: algorithm_prd_template.md
    │      ├─ Type B: streamlit_prd_template.md
    │      ├─ Type C: agent_prd_template.md
    │      └─ 기타: common_prd_template.md
    │
    ├─► 3. PRD 초안 작성
    │      ├─ 1장: 문서관리 (버전, 이력)
    │      ├─ 2장: 개요 (목표, 범위)
    │      ├─ 3장: 사용자 요구사항
    │      ├─ 4장: 기능 요구사항
    │      ├─ 5장: 기술 요구사항
    │      ├─ 6장: 비기능 요구사항
    │      └─ 7~10장: 테스트/배포/관리
    │
    └─► 4. PRD 저장
           └─ 00_doc/d{SP}0001_prd.md
```

### 2.3 정합성 검증 워크플로우

```
ooprd run (PRD 있음)
    │
    ├─► 1. PRD 로드
    │      └─ 00_doc/d{SP}0001_prd.md 읽기
    │
    ├─► 2. 비교 대상 스캔 (병렬)
    │      ├─ 코드베이스 (Explore)
    │      │  ├─ 구현 파일/함수
    │      │  ├─ API 엔드포인트
    │      │  └─ DB 테이블
    │      │
    │      └─ 문서 (Read)
    │         ├─ d{SP}0002_plan.md (계획)
    │         ├─ d{SP}0003_test.md (테스트)
    │         ├─ d{SP}0005_lib.md (라이브러리)
    │         ├─ d{SP}0006_db.md (DB)
    │         └─ d{SP}0008_user.md (사용자)
    │
    ├─► 3. 정합성 비교 (task-checker)
    │      ├─ 기능 구현 여부
    │      ├─ 기술 스택 일치
    │      ├─ API 명세 일치
    │      └─ DB 스키마 일치
    │
    ├─► 4. 불일치 항목 추출
    │      ├─ 누락된 기능
    │      ├─ 구현되지 않은 API
    │      ├─ 문서 불일치
    │      └─ 우선순위 판단
    │
    └─► 5. 리포트 생성 및 이슈 등록
           ├─ 터미널: 불일치 N건 + 상세 항목
           └─ d{SP}0004_todo.md 등록
              └─ | ID | 발생일 | [PRD] | 내용 | 우선순위 | 상태 |
```

### 2.4 Clarify 워크플로우 (모호성 해소)

```
ooprd clarify
    │
    ├─► 1. PRD 로드
    │      └─ 00_doc/d{SP}0001_prd.md
    │
    ├─► 2. Coverage 스캔 (6개 영역)
    │      ├─ 기능 범위 (핵심 목표, 제외 선언, 사용자 역할)
    │      ├─ 도메인/데이터 (엔티티, 관계, 상태 전이)
    │      ├─ UX 흐름 (핵심 여정, 에러 상태, 접근성)
    │      ├─ 비기능 (성능, 확장성, 보안)
    │      ├─ 통합 (외부 서비스, 실패 모드)
    │      └─ 엣지케이스 (네거티브 시나리오)
    │
    │      각 영역 상태 판단:
    │      └─ Clear / Partial / Missing
    │
    ├─► 3. 질문 생성 (최대 5개)
    │      ├─ 영향도 × 불확실성 우선순위화
    │      ├─ 다중선택 또는 단답형
    │      └─ 권장 옵션 제시
    │
    ├─► 4. 순차 질문
    │      ├─ 질문 1 출력 → 응답 대기
    │      ├─ 질문 2 출력 → 응답 대기
    │      ├─ ...
    │      └─ "done" 입력 시 조기 종료
    │
    ├─► 5. PRD 업데이트
    │      ├─ ## Clarifications 섹션 추가
    │      └─ 해당 섹션에 명확화 내용 반영
    │
    └─► 6. 완료 리포트
           ├─ 질문/응답 수
           ├─ 수정된 섹션
           └─ 커버리지 요약
```

**질문 형식 예시**:

```markdown
## Question 1: 사용자 인증 범위

**Context**: PRD 4.1 로그인 기능 섹션에서 "사용자 인증"만 언급

**What we need to know**: 소셜 로그인 지원 여부는?

**Suggested Answers**:

| Option | Answer | Implications |
|--------|--------|--------------|
| A (권장) | 이메일 + 소셜(구글, 카카오) | 높은 전환율, OAuth 연동 필요 |
| B | 이메일만 | 빠른 개발, 낮은 전환율 |
| C | 소셜만 | 간단한 구현, 이메일 없는 사용자 불가 |
```

### 2.5 단위개발문서 현행화 워크플로우 (unitdev)

```
ooprd unitdev [문서명]
    │
    ├─► 1. 문서 로드
    │      ├─ 단일: d{SP}1XXX_*.md
    │      └─ 전체: 00_doc/d{SP}1*_*_단위개발.md (문서명 생략 시)
    │
    ├─► 2. 개요(1장) 분석 (기준점)
    │      ├─ 1.1 목적 추출
    │      ├─ 1.2 핵심가치 추출
    │      └─ 1.3 아이템/상세 추출
    │
    ├─► 3. 섹션별 정합성 검증 (1장 → 2~5장)
    │      │
    │      ├─ 2장 (기획) 검증
    │      │  ├─ 1.3 아이템 → 2.1 기능요구사항 매핑
    │      │  ├─ 1.1 목적 → 2.1 기능 도출
    │      │  └─ 불일치: 누락된 기능 자동 추가
    │      │
    │      ├─ 3장 (설계) 검증
    │      │  ├─ 1.3 아이템 상세 → 3.N 데이터구조 필드
    │      │  ├─ 2.3 UI/UX → 3.1 아키텍처 컴포넌트
    │      │  └─ 불일치: 데이터구조 동기화
    │      │
    │      ├─ 4장 (구현) 검증
    │      │  ├─ 2.1 기능요구사항 → 4.2 주요함수
    │      │  ├─ 3.N 데이터구조 → 4.5 DB 테이블
    │      │  └─ 불일치: 함수/DB 매핑 추가
    │      │
    │      └─ 5장 (검증) 검증
    │         ├─ 1.2 핵심가치 → 5.2 검증체크리스트
    │         ├─ 2.2 시나리오 → 5.1 테스트케이스
    │         └─ 불일치: TC 생성
    │
    ├─► 4. 불일치 항목 자동 업데이트
    │      ├─ 누락된 기능/함수 추가
    │      ├─ 데이터구조 필드 동기화
    │      └─ 테스트케이스 생성
    │
    └─► 5. 문서 저장 및 버전 업데이트
           ├─ 문서 이력 관리 섹션 업데이트
           └─ 버전 증가 (v05 → v06)
```

**섹션 연계 규칙 (핵심)**:

| 입력 섹션 | 출력 섹션 | 연계 규칙 |
|----------|----------|----------|
| 1.1 목적 | 2.1 기능요구사항 | 목적 달성을 위한 기능 도출 |
| 1.2 핵심가치 | 5.2 검증체크리스트 | 핵심가치별 검증항목 생성 |
| 1.3 아이템 | 2.1 기능요구사항 | 아이템별 기능 매핑 |
| 1.3.N 아이템상세 | 3.N 데이터구조 | 상세정보 → 필드 정의 |
| 2.1 기능요구사항 | 4.2 주요함수 | 기능별 함수 매핑 |
| 2.2 시나리오 | 5.1 테스트케이스 | 시나리오별 TC 생성 |
| 2.3 UI/UX | 3.1 아키텍처 | UI 구성요소 → 컴포넌트 |
| 3.N 데이터구조 | 4.5 DB | 필드 → 테이블 정의 |

**출력 예시**:

```
[unitdev] d21720 분석 중...
  - 1.1 목적: 사용자 맞춤형 복지혜택 표출
  - 1.3 아이템: 복지정보카드 (7개 필드)

[정합성 검증]
  - 2.1 기능요구사항: 2건 누락 → 추가
    * 복지정보카드 필터링 기능
    * 카드 상세보기 모달
  - 4.2 주요함수: 3건 누락 → 추가
    * filter_welfare_cards()
    * display_card_detail()
    * get_user_preferences()
  - 5.1 테스트케이스: 4건 누락 → 생성
    * TC01: 필터링 정확성
    * TC02: 카드 렌더링
    * TC03: 상세보기 동작
    * TC04: 사용자별 맞춤화

[완료] d21720 v05 → v06 업데이트
```

## 3. 상세 사용법

### 3.1 서브명령어

#### 3.1.1 ooprd status

서브명령어 리스트 및 PRD 상태 요약 출력

```bash
ooprd status
```

**출력 예시**:
```
=== ooprd 서브명령어 ===
- run: PRD 생성/정합성 검증
- optimize: PRD 최적화
- validate: 구조 검증
- section [N]: 특정 섹션 갱신
- unitdev: 단위개발문서 현행화
- clarify: 모호성 해소
- template: 템플릿 목록
- version: v09

=== PRD 상태 ===
- 파일: 00_doc/sp00/d0001_prd.md
- 버전: v03
- 최종 수정: 2026-02-03
- 정합성 이슈: 5건 (d0004_todo.md 참조)
```

#### 3.1.2 ooprd run

PRD 생성 또는 정합성 검증

**신규 생성**:
```bash
ooprd run                               # 기본 템플릿
ooprd run --template streamlit          # Streamlit 템플릿
ooprd run --template algorithm          # 알고리즘 템플릿
```

**정합성 검증**:
```bash
ooprd run                               # 전체 검증
ooprd run --sp 02                       # 서브프로젝트 02 검증
```

#### 3.1.3 ooprd optimize

PRD 구조/내용 검증 및 최적화

```bash
ooprd optimize
```

**체크 항목**:
- 템플릿 완성도 (필수 섹션 존재)
- 모호한 표현 0건
- MoSCoW 우선순위 할당
- 중복 항목 제거

#### 3.1.4 ooprd validate

PRD 구조 검증만 실행 (코드 비교 없음)

```bash
ooprd validate
```

#### 3.1.5 ooprd section [N]

특정 섹션만 갱신

```bash
ooprd section 4                         # 4장만 업데이트
ooprd section 5.2                       # 5.2 섹션만 업데이트
```

#### 3.1.6 ooprd unitdev

단위개발문서 현행화

```bash
# 현재 컨텍스트(SP)의 모든 단위개발문서 현행화
ooprd unitdev

# 특정 문서만 현행화
ooprd unitdev d21720
ooprd unitdev d21720_앱_시니어복지_단위개발.md
```

#### 3.1.7 ooprd clarify

PRD 모호성 해소 (최대 5개 질문)

```bash
ooprd clarify                           # 기본 5개 질문
ooprd clarify --max 3                   # 최대 3개로 제한
```

#### 3.1.8 ooprd template

템플릿 목록 조회

```bash
ooprd template                          # 전체 목록
ooprd template streamlit                # Streamlit 템플릿 상세
```

### 3.2 정합성 검증 항목

#### 3.2.1 코드 vs PRD

| 검증 항목 | PRD 섹션 | 비교 대상 |
|----------|---------|----------|
| 기능 구현 | 4. 기능 요구사항 | 구현 파일/함수 |
| 기술 스택 | 5.2 기술 스택 | pyproject.toml, requirements.txt |
| API 엔드포인트 | 5.3 API 명세 | 라우터/컨트롤러 파일 |
