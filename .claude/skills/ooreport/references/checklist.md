# ooreport check 체크리스트

> ooreport 스킬 자체 건강 상태 검증 항목
> 템플릿: .claude/templates/oo_checklist_template.md

| ID | 항목 | 검증 내용 | 심각도 |
|----|------|----------|--------|
| C01 | 필수 파일 존재 | SKILL.md 등 핵심 파일 존재 여부 | CRITICAL |
| C02 | 버전 일치 | metadata version <-> 서브명령어 테이블 (vXX) 일치 | ERROR |
| C03 | 리포트 템플릿 존재 | 리포트 생성 템플릿 파일 존재 | ERROR |
| C04 | 출력 형식 지원 | MD/PPTX/DOCX 변환 가능 | WARNING |
| C05 | 의존 스킬 확인 | ooppt/ooword 스킬 존재 | WARNING |
| C06 | 문체 확인 | 리포트 생성 전 논문식/개조식 문체를 사용자에게 확인하고 전체 문장 스타일을 통일 | ERROR |

## check 출력 형식

```
[ooreport check]

C01 필수 파일 존재             [OK]
C02 버전 일치                [OK]
C03 리포트 템플릿 존재           [OK]
C04 출력 형식 지원             [OK]
C05 의존 스킬 확인             [OK]

소계: OK:N | WARN:N | ERROR:N
```

## 심각도 기준

| 심각도 | 기호 | 의미 |
|--------|------|------|
| CRITICAL | [ERROR] | 스킬 동작 불가 수준 |
| ERROR | [ERROR] | 즉시 수정 필요 |
| WARNING | [WARN] | 권장 수정 |
| INFO | [INFO] | 참고용 |

## review 출력 형식

> `ooreport review` 실행 시 아래 형식으로 출력. 심각도 우선 그룹핑 → 위치 정보 → 수정 제안 순.

```
[ooreport review] 파일명.md

── ERROR (즉시 수정) ──────────────────────────────
P05 [ERROR] 실험 코드명 노출
  위치: §4.2 "ps4194를 통해 실험하였다"
  수정: 내부 스크립트명 제거 → "실험을 통해" 또는 익명 표기

P16 [ERROR] 제안 모델명 최초 정의 형식
  위치: §1 Abstract 2번째 문단
  수정: "DBCANet" 최초 등장 시 "DBCANet (Dual-Branch Cross-Attention Network)" 형식으로 풀네임 병기

── WARN (권장 수정) ───────────────────────────────
P09 [WARN] 약어 최초 풀어쓰기
  위치: §2.3 "ASPP 모듈 적용"
  수정: ASPP 최초 등장 시 "ASPP (Atrous Spatial Pyramid Pooling)" 병기

── OK ────────────────────────────────────────────
P01 용어/모듈명 통일            [OK]
P02 수치 파라미터 일치           [OK]
...

소계: ERROR:N | WARN:N | OK:N
```

### 출력 규칙
- **ERROR/WARN 항목**: 위치(섹션·문장 키워드) + 수정 제안 1줄 필수 출력
- **OK 항목**: ID + 항목명 + [OK] 한 줄로 압축
- **그룹 순서**: ERROR → WARN → OK

## 논문 교정 체크리스트 (reviewlist / ooreport review 시 적용)

> `ooreport review` 실행 시 적용

### 1. 일관성 (Consistency)

| ID | 항목 | 검증 내용 | 심각도 |
|----|------|----------|--------|
| P01 | 용어/모듈명 통일 | 기준 섹션(방법론) 대비 전체 섹션의 용어·모듈명 일치 여부 | ERROR |
| P02 | 수치 파라미터 일치 | 동일 파라미터(T_max, batch_size 등)가 여러 섹션에 기재된 경우 수치 일치 | ERROR |
| P03 | 테이블 번호 참조 | 본문 내 Table 참조 번호가 실제 테이블과 일치하는지 | ERROR |
| P04 | 데이터셋·모델명 통일 | 데이터셋명, 모델명이 전 섹션에서 동일하게 사용되는지 | ERROR |

| ID | 항목 | 검증 내용 | 심각도 |

| P06 | Phase 번호 노출 | Phase 6.1, Phase 19 등 내부 개발 단계 번호 노출 여부 | ERROR |
| P07 | 임시 표현 잔존 | 개발 중 임시 메모·주석성 표현이 본문에 남아있는지 | WARNING |

### 5. 문서 구조 (Document Structure)

| ID | 항목 | 검증 내용 | 심각도 |
|----|------|----------|--------|
| P13 | 섹션 과분화 금지 | (1) 내용이 너무 적은 섹션(2~3문장 이하)은 하위 섹션으로 나누지 않고 상위 섹션에 통합. (2) 부모 섹션 안에 서브섹션이 1개뿐인 경우 상위 섹션에 통합. (3) 서브섹션이 단락 1개로만 구성되고 독립 소제목 없이도 이해에 지장이 없는 경우 상위 섹션에 통합. **적용 범위**: ### 헤딩뿐 아니라 볼드 텍스트 번호(`**X.Y.Z ...**`) 형태의 서브섹션에도 동일 기준 적용. 특히 Related Work에서 논문 1편당 서브섹션 1개를 부여하는 패턴은 과분화에 해당하므로 부모 섹션으로 통합할 것. | WARNING |
| P14 | 단문 문단 병합 | 문장이 5개 이하인 짧은 문단은 가급적 인접 문단과 합쳐 문단 밀도를 높임 | WARNING |
| P19 | 과도한 계층 구조 금지 | #### (h4) 레벨 서브섹션(X.Y.Z) 존재 여부 확인. 학술 논문은 ### (h3, X.Y) 까지만 사용하는 것이 표준이며, #### 레벨 서브섹션은 상위 레벨(X.Y)로 승격하고 기존 부모 섹션 인트로는 상위 섹션 도입 단락으로 흡수한다. | WARNING |
| P20 | 관례적 섹션 내용 충분성 | "논문 구성(Paper Organization)", "연구 배경", "문제 정의" 등 학술 논문에서 관례적으로 독립 섹션으로 사용하는 항목이 단일 문단(5문장 이하)으로 내용이 빈약한 경우, P13(상위 통합)이 아닌 **내용 보완**으로 각 장의 핵심 내용을 구체적으로 기술하여 섹션 밀도를 확보할 것 | WARNING |
| P21 | 부모 섹션 도입부 빈약 | 하위 서브섹션을 가진 부모 섹션의 고유 도입 텍스트가 1~2문장에 불과하여 서브섹션 내용의 단순 요약·나열에 그치는 경우. (1) 도입부를 확장하여 설계 동기·서브섹션 간 관계를 설명하거나, (2) 도입 문장을 첫 번째 서브섹션에 흡수할 것 | WARNING |
