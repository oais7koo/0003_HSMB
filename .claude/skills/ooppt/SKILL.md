## 1. 개요

대상 마크다운 문서를 읽고, PPT 발표용 슬라이드 노트 형식으로 **내용을 재구조화**하여 새로운 MD 문서를 생성한다.

**입력**: 일반 마크다운 문서 (보고서, 기획서, 기술문서 등)
**출력**: 슬라이드별로 구조화된 PPT 노트 마크다운 (`{원본명}_ppt노트.md`)

## 2. 서브명령어

| 명령어 | 설명 |
|--------|------|
| `ooppt help` | 서브명령어 목록 표시 |
| `ooppt version` | 스킬 버전 정보 (v05) |
| `ooppt run [경로]` | **MD → PPT노트 MD 변환 실행** |
| `ooppt update [경로]` | 현행화 — 소스 변경분 감지 → PPT 재생성 | .pptx |
| `ooppt update --dry-run` | 변경 예정 내용 미리 출력 (실제 수정 안 함) | 터미널 |
| **`ooppt run this`** | **직전 작업 PPT 노트 변환** (→ common_guide.md §9) |
| `ooppt run --slides N` | 슬라이드 수 지정 (기본: 자동) |
| `ooppt run --style [비즈니스\|기술\|교육]` | 발표 스타일 지정 |

## 3. 워크플로우

```
입력 MD 읽기 → 핵심 내용 추출 → 슬라이드 구조 설계 → PPT노트 MD 생성
```

### 3.1 핵심 내용 추출

- 제목/섹션 계층 분석
- 핵심 메시지 식별 (□/■ 마커, 굵은 텍스트, 결론 문장)
- 데이터/수치 추출 (표, 통계, 비교)
- 불필요 내용 제거 (상세 코드, 긴 설명, 참조 링크)

### 3.2 슬라이드 구조 설계

| 슬라이드 유형 | 용도 | 콘텐츠 |
|-------------|------|--------|
| 표지 | 제목, 부제, 발표자 | 문서 제목 + 요약 1줄 |
| 개요/목차 | 발표 흐름 | 주요 섹션 3~5개 |
| 내용 | 핵심 메시지 전달 | 제목 + 불릿 3~5개 |
| 비교 | 대조/분석 | 2~3칼럼 비교표 |
| 데이터 | 수치/통계 | KPI, 차트용 데이터 |
| 요약 | 마무리 | 핵심 메시지 3개 |

### 3.3 슬라이드당 콘텐츠 규칙

| 항목 | 규칙 |
|------|------|
| 제목 | 1줄, 10단어 이내 |
| 불릿 | 3~5개, 각 2줄 이내 |
| 키워드 | 문장이 아닌 핵심 키워드 중심 |
| 데이터 | 수치는 반드시 포함 |
| 원문 유지 | 환각 금지, 원본 내용만 사용 |

## 4. 출력 형식

```markdown
# {제목} - PPT 노트

> 원본: {입력파일 경로} | 생성: {날짜} | 슬라이드: {N}장

---

## 슬라이드 1. 표지

챕터:
제목: {문서 제목}
부제: {핵심 메시지 1줄}

---

## 슬라이드 2. 개요

챕터: 01 Overview
제목: 발표 목차
부제:

□ 첫 번째 주제
□ 두 번째 주제
□ 세 번째 주제

---

## 슬라이드 3. {섹션 제목}

챕터: 02 {영문 키워드}
제목: {슬라이드 제목}
부제: {보조 설명}

□ {핵심 포인트 1}
  ○ {세부 내용}
  ○ {세부 내용}

□ {핵심 포인트 2}
  ○ {세부 내용}

---
```

## 5. 출력 규칙

| 항목 | 규칙 |
|------|------|
| **출력 위치** | 입력 파일과 동일한 디렉토리 |
| **파일명** | `{원본명}_ppt노트.md` |
| **슬라이드 수** | 10~15장 (자동), `--slides N`으로 조정 |
| **환각 방지** | 원본에 없는 내용 추가 금지 |

**예시**:
- 입력: `04_기술연구/42_제품핵심기술.md`
- 출력: `04_기술연구/42_제품핵심기술_ppt노트.md`

## 6. 발표 스타일

| 스타일 | 특징 | 슬라이드 구성 |
|--------|------|-------------|
| 비즈니스 | 숫자/성과 중심 | 표지→문제→솔루션→시장→로드맵→요약 |
| 기술 | 아키텍처/구현 중심 | 표지→개요→아키텍처→핵심기술→검증→요약 |
| 교육 | 설명/이해 중심 | 표지→배경→개념→예시→실습→정리 |

## 7. 서브에이전트 활용

| 단계 | 에이전트 | 모델 | 역할 | 병렬 |
|------|---------|------|------|:----:|
| 변환 | task-executor | sonnet | MD 읽기 → 핵심 추출 → PPT노트 MD 생성 | - |

> 콘텐츠 재구조화는 sonnet으로 충분. opus 위임 불필요.

## 8. 관련

- 생성된 PPT노트 MD는 외부 도구(파워포인트, Gamma, Beautiful.ai 등)에서 슬라이드로 변환 가능
- `ooreport`와 연계: 보고서 → PPT노트 → 발표 자료 파이프라인

## run과 update 분리 원칙

> 이 스킬은 `.claude/guides/run_update_separation.md` 원칙을 따른다.

| 서브커맨드 | 역할 |
|-----------|------|
| `run` | 이 스킬의 **배치 실행** 또는 구체적인 명령 실행 (일회성) |
| `update` | 최상의 상태로 유지되어야 하는 **모든 상태·설정 현행화** (멱등) |

> `run`에서 자동으로 `update`를 호출하지 않는다. 현행화는 별도 명령으로 실행.
