## 1. 개요

ooplan은 PRD를 기반으로 구현 계획을 수립하고 상세 설계를 수행하는 Architect 역할의 스킬입니다. PRD의 요구사항을 Epic/Feature/Task로 분해하여 실행 가능한 개발 계획을 생성합니다.

### 1.1 핵심 역할

- **PRD → WBS 변환**: 요구사항을 계층적 작업 분해 구조로 전환
- **상세 설계**: Task 단위 구현 계획 수립
- **리스크 관리**: 기술적 리스크 식별 및 대응 방안
- **스프린트 계획**: 일정 및 마일스톤 설정

### 1.2 출력 문서

| 문서 | 역할 | 비고 |
|------|------|------|
| `d{SP}0002_plan.md` | 구현 계획 | 주 출력 |
| `d{SP}0004_todo.md` | 이슈 등록 | 부 출력 (리스크/장애물) |

### 1.3 계층 구조

```
PRD (What)
    ↓
Epic (도메인/비즈니스 영역)
    ↓
Feature (사용자 기능)
    ↓
Task (구현 단위)
    ↓
TC (테스트 케이스) - oodev에서 생성
```

### 2.2 run 워크플로우 (PRD → Plan)

**정보 수집 프로토콜** (PRD → Plan 생성 전):

```
1. PRD 로드 및 분석
   d{SP}0001_prd.md 읽기 → Plan 생성에 필요한 항목 추출

2. 필수 정보 누락 확인
   | 항목 | PRD 출처 | 누락 시 질문 |
   |------|---------|------------|
   | Epic 분해 근거 | §기능요구사항 | "어떤 도메인/기능 그룹으로 나눌까요?" |
   | 기술 스택 | §기술스택 | "사용할 언어/프레임워크는?" |
   | 구현 우선순위 | MoSCoW | "우선 구현할 Feature는?" |
   | 일정/스프린트 | §마일스톤 | "스프린트 기간과 목표 완료 시점은?" |
   | 제약/리스크 | §비기능 | "기술적 제약이나 리스크가 있나요?" |

3. 누락 항목 순차 질문 (1개씩)
   - PRD에서 파악 가능 → 자동 채우고 다음
   - PRD에 없음 → 사용자에게 직접 질문
   - "skip" 입력 시 → AI 판단으로 채우고 계속

4. Epic 도출 → Feature 분리 → Task 세분화 → d{SP}0002_plan.md 저장
```

**질문 형식** (1개씩 순차):

```
[Plan 정보 수집 N/M] 항목명

PRD 파악 내용: [추출된 내용 또는 "PRD에 정보 없음"]
질문: [구체적 질문]

(skip 입력 시 AI 판단으로 대체)
```

```
ooplan run [epic|feature|task]
    │
    ├─► 1. PRD 로드 및 검증
    │      ├─ 00_doc/d{SP}0001_prd.md 읽기
    │      ├─ 웹 프로젝트 조건 확인
    │      │  ├─ epic: 5.1 페이지 개요 필수
    │      │  ├─ feature: 5.2 페이지 상세 필수
    │      │  └─ task: 5.2 + 상세기획 필수
    │      └─ 진행=아니오 페이지 필터링
    │
    ├─► 2. Epic 도출 (run epic)
    │      ├─ PRD 5.1 페이지 개요 "대분류" 추출
    │      ├─ Epic 생성
    │      │  ├─ E001: 인증/권한
    │      │  ├─ E002: 정책정보
    │      │  └─ E003: 업무관리
    │      └─ 기존 Feature/Task 삭제 (하위 레벨)
    │
    ├─► 3. Feature 도출 (run feature)
    │      ├─ PRD 5.2 페이지 상세 "주요 기능" 추출
    │      ├─ Feature 생성 (Epic 하위)
    │      │  ├─ F001-1: 로그인
    │      │  ├─ F001-2: 권한 관리
    │      │  ├─ F002-1: 중앙정부 정책
    │      │  └─ F002-2: 지방정부 정책
    │      └─ 기존 Task 삭제 (하위 레벨)
    │
    ├─► 4. Task 도출 (run task, 기본)
    │      ├─ 상세기획 문서 (d{SP}1XXX.md) 읽기
    │      ├─ Task 생성 (Feature 하위)
    │      │  ├─ F002-1.1: 보건복지부 탭
    │      │  ├─ F002-1.2: 고용노동부 탭
    │      │  └─ F002-1.3: 필터링 기능
    │      └─ 상태: "대기" (기본값)
    │
    ├─► 5. 기술 설계 (섹션 5)
    │      ├─ 아키텍처 다이어그램
    │      ├─ 기술 스택 상세
    │      ├─ API 설계
    │      └─ DB 설계
    │
    ├─► 6. 스프린트 계획 (섹션 4)
    │      ├─ Sprint 1: 인증/권한 (E001)
    │      ├─ Sprint 2: 정책정보 (E002)
    │      └─ Sprint 3: 업무관리 (E003)
    │
    ├─► 7. 리스크 분석 (섹션 6)
    │      ├─ 기술적 리스크
    │      ├─ 일정 리스크
    │      └─ 의존성 리스크
    │
    └─► 8. Plan 저장
           └─ 00_doc/d{SP}0002_plan.md
```

**Epic/Feature/Task 네이밍 규칙**:

| 레벨 | 형식 | 예시 | 기준 |
|------|------|------|------|
| Epic | `E{순번}` | E002 | 도메인/비즈니스 영역 |
| Feature | `F{Epic}-{순번}` | F002-1 | 사용자 기능 |
| Task | `F{Epic}-{Feature}.{순번}` | F002-1.1 | 구현 단위 |

```

    │      └─ 예외 상황 정의
    │
    ├─► 3. 실행 계획 생성
```

```
    │      └─ 병렬 작업 기회 식별
    │
    ├─► 5. 리스크 검토
    │      ├─ 기술적 리스크 재평가
    │      ├─ 대응 방안 구체화
    │      └─ 우선순위 재조정
    │
    └─► 6. Plan 업데이트
           └─ 최적화 결과 반영
```

**최적화 체크리스트**:

| 항목 | 검증 기준 |
|------|----------|
| PRD 커버리지 | 100% (모든 Must 기능 포함) |
| 순환 의존성 | 없음 |
| 크리티컬 패스 | 최소화 (병렬 작업 최대화) |
| 스프린트 균형 | 각 스프린트 작업량 ±20% 이내 |
| 리스크 대응 | 모든 High 리스크에 대응 방안 |

## 3. 상세 사용법

### 3.1 서브명령어

#### 3.1.1 ooplan status

서브명령어 리스트 및 Plan 상태 요약

```bash
ooplan status
```

**출력 예시**:
```
=== ooplan 서브명령어 ===
- run: PRD → Plan 생성
- run epic: Epic까지만 생성
- run feature: Feature까지만 생성
- run task: Task까지 생성 (기본)
- detail: Task 구체화
- optimize: Plan 최적화
- sync: PRD 동기화
- version: v06

=== Plan 상태 ===
- 파일: 00_doc/sp00/d0002_plan.md
- 버전: v05
- Epic: 3개
- Feature: 8개
- Task: 24개
- 완료율: 15/24 (62.5%)
- 스프린트: 3/3 진행 중
```

#### 3.1.2 ooplan run [epic|feature|task]

PRD 기반 Plan 생성 (단계별)

```bash
# Epic까지만 생성 (대분류 기반)
ooplan run epic

# Feature까지 생성 (페이지별 기능 기반)
ooplan run feature

# Task까지 생성 (기본값, 상세 기획 기반)
ooplan run task
ooplan run                              # task 생략 가능
```

**PRD 필요 조건 (웹 프로젝트)**:

| 옵션 | PRD 필요 섹션 | 근거 |
|------|-------------|------|
| `run epic` | 5.1 페이지 개요 | 대분류 → Epic |
| `run feature` | **5.2 페이지 상세** | 주요 기능 → Feature |
| `run task` | 5.2 + 상세기획(d{SP}1XXX.md) | 기능 분해 → Task |

**진행 필터링**:
- PRD 5.1 페이지 개요의 `진행=아니오`인 페이지는 자동 제외
- `진행=예`인 페이지만 Plan에 포함

**하위 레벨 처리**:

| 옵션 | 생성 범위 | 기존 하위 레벨 처리 |
|------|----------|------------------|
| `run epic` | Epic만 | Feature/Task 전체 삭제 |
| `run feature` | Epic + Feature | Task 전체 삭제 |
| `run task` | Epic + Feature + Task | 전체 업데이트 (추가/수정) |

Task 구체화 및 실행 계획 수립

**출력 예시**:

#### 3.1.4 ooplan optimize

Plan 구조/일정 최적화

### 3.2 PRD → Plan 매핑

#### 3.2.1 계층 매핑

| PRD 요소 | Plan 요소 |
|----------|----------|
| 비전/목표 | 섹션 2. 구현 개요 - 목표 요약 |
| 기능 요구사항 (4장) | Epic/Feature/Task (섹션 3 WBS) |
| 기술 요구사항 (5장) | 섹션 5. 기술 설계 |
| 아키텍처 (5.1) | 섹션 7. 의사결정 - 아키텍처 결정 |
| 마일스톤 (9장) | 섹션 4. 스프린트 계획 |
| 리스크 (9.4) | 섹션 6. 리스크 관리 (상세화) |

#### 3.2.2 분해 체계

```
PRD 4. 기능 요구사항
    │
    ├─► Epic (도메인/비즈니스)
    │      예: 인증/권한, 정책정보, 업무관리
    │
    ├─► Feature (사용자 기능)
    │      예: 로그인, 중앙정부 정책, 업무 조회
    │
    └─► Task (구현 단위)
           예: 보건복지부 탭, 필터링 기능
```

**분해 기준**:

| 레벨 | 기준 | 예시 | 크기 |
|------|------|------|------|
| Epic | 도메인/비즈니스 영역 | 정책정보 시스템 | 2주~1개월 |
| Feature | 사용자 가치 기능 | 중앙정부 정책 조회 | 3~5일 |
| Task | 단일 구현 단위 | 보건복지부 탭 | 0.5~1일 |

### 3.3 Streamlit 개발 기획 (3계층)

```
PRD (What)
  ↓
Plan (Where)
  ↓
개발기획 d{SP}1XXX.md (How)
```

#### 3.3.1 3계층 역할

| 문서 | 역할 | 수준 |
|------|------|------|
| `d{SP}0001_prd.md` | **무엇을** 만들 것인가 | 요구사항 |
| `d{SP}0002_plan.md` | **어디에** 배치할 것인가 | 구조/일정 |
| `d{SP}1XXX.md` | **어떻게** 구현할 것인가 | 상세 설계 |

#### 3.3.2 개발기획 문서 네이밍

```
d{서브프로젝트번호}{4자리순번}_페이지명_단위개발.md
```

**예시**:
- `d21720_앱_시니어복지_단위개발.md` (02번 서브프로젝트, 1720번 문서)

- `d21722_앱_정책정보_단위개발.md`

```markdown

```

**메트릭 원칙**: 체크포인트/로그/CSV 원본 기준. 불일치 시 원본 우선.

## 4.8 Plan 콘텐츠 범위 규칙 (중복 방지)

Plan은 "어떻게(How)/언제(When)" 문서. **PRD 내용 복사 금지**:

| Plan이 소유하는 항목 | PRD에서 금지하는 항목 |
|--------------------|-------------------|
| 기술 스택 상세 (버전/라이브러리) | 목적/범위 반복 서술 → PRD 참조 1줄로 대체 |
| 폴더 구조 트리 | 기능 요구사항 목록 복사 |
| DB DDL/스키마 | UI/UX 요구사항 |
| WBS, 스프린트, 마일스톤 | 비기능 요구사항 |
| 기술/일정 리스크 | 사용자 스토리/페르소나 |
| 의사결정 로그 | |
