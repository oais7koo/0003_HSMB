## 1. 개요

### 1.1 목적

세션 시작 시 프로젝트 전체의 준비 상태를 점검하고, 필요한 문서를 로드하며, 코드 품질을 검증하는 자동화 워크플로우입니다.

### 1.2 주요 기능

- 공통 가이드라인 자동 로드 (common_guide.md)
- 핵심 문서 상태 점검 (PRD, TODO, HISTORY)
- 프로젝트 구조 스캔 및 검증
- 환경/설정 파일 유효성 확인
- 코드 에러 체크 (oocheck 연동)

### 1.3 실행 시점

- 새 세션 시작 시
- 장기간 작업 중단 후 재개 시
- 프로젝트 구조 변경 후 검증 필요 시
- 팀원 간 작업 인수인계 시

## 2. 워크플로우

### 2.1 전체 흐름도

```
세션 시작
    ↓
common_guide.md 로드 (자동)
    ↓
문서 상태 점검 (PRD, TODO, HISTORY)
    ├─ 누락 문서 경고
    └─ 버전 일관성 확인
    ↓
프로젝트 구조 스캔 (병렬)
    ├─ Explore: 디렉토리/파일 구조
    └─ task-checker: 환경/설정 검증
    ↓
동기화 체크리스트 출력
    ├─ 최근 변경 사항
    ├─ 미완료 TODO
    └─ 주요 에러
    ↓
oocheck 연동 (선택)
    └─ 정적 분석 실행
    ↓
준비 완료 메시지
```

### 2.2 단계별 상세

#### 1. common_guide.md 자동 로드

**목적**: 프로젝트 공통 가이드라인 컨텍스트 로드

**동작**:
- `.claude/guides/common_guide.md` 내용 읽기
- 주요 섹션 추출 (디렉토리 구조, 코딩 표준, 에이전트 규칙)
- 세션 컨텍스트에 로드

**출력**:
```
✓ common_guide.md 로드 완료
  - 디렉토리 구조
  - 코딩 표준
  - 에이전트 위임 규칙
```

#### 2. 문서 상태 점검

**목적**: 핵심 문서 존재 및 최신성 확인

**검증 문서**:
| 문서 | 필수 여부 | 검증 항목 |
|------|----------|----------|
| `00_doc/d{SP}0001_prd.md` | 필수 | 존재, 버전 정보 |
| `00_doc/d{SP}0004_todo.md` | 필수 | 미완료 TODO 개수 |
| `00_doc/d{SP}0010_history.md` | 필수 | 최근 변경 날짜 |
| `00_doc/d{SP}0003_test.md` | 권장 | 테스트 실행 이력 |

**출력**:
```
✓ 핵심 문서 상태
  - d0001_prd.md (v05, 2026-01-20)
  - d0004_todo.md (5개 미완료)
  - d0010_history.md (최근: 2026-02-01)
  ⚠ d0003_test.md 누락 (ootest로 생성 권장)
```

#### 3. 프로젝트 구조 스캔 (병렬)

**목적**: 파일 시스템 구조 및 환경 검증

**병렬 에이전트**:
```
[Explore] 프로젝트 구조 탐색
    ├─ 디렉토리 구조 스캔
    ├─ oo 모듈 목록 (oo/*.py)
    ├─ 페이지 목록 (pages/*.py)
    └─ 문서 목록 (00_doc/*.md)

[task-checker] 환경/설정 검증
    ├─ .env 존재 확인
    ├─ pyproject.toml 유효성
    └─ 데이터베이스 파일 (db/*.db)
```

**출력**:
```
✓ 프로젝트 구조
  - oo 모듈: 43개
  - 페이지: 52개
  - 문서: 12개

✓ 환경 설정
  - .env 존재 (5개 변수)
  - pyproject.toml 유효
  - db/vibe.db (크기: 2.3MB)
```

#### 4. 동기화 체크리스트

**목적**: 세션 시작 전 확인 사항 정리

**출력 형식**:
```markdown
## 동기화 체크리스트

### 최근 변경 사항 (d0010_history.md)
- 2026-02-01: FEATURE - 보건복지부 탭 구현 완료
- 2026-01-28: BUGFIX - 로그인 세션 타임아웃 수정

### 미완료 TODO (d0004_todo.md)
- T042: [BUGFIX] oo\seal.py:148 - cv2 import 오류 (높음)
- T043: [UPDATE] requests 보안 업데이트 필요 (중간)
- T044: [IMPROVE] DB 쿼리 최적화 (낮음)

### 주의사항
- [ ] T042 우선 처리 필요 (높음)
- [ ] 테스트 실행 필요 (d0003 갱신 1주 경과)
```

#### 5. oocheck 연동 (선택)

**목적**: 정적 분석으로 코드 품질 사전 검증

**실행 조건**:
- 1주일 이상 코드 체크 미실행
- 다수 파일 변경 감지
- 사용자 요청

**동작**:
```bash
# oocheck status 실행
oocheck status

# 에러 발견 시 d0004_todo.md 자동 등록
```

**출력**:
```
⚠ 마지막 코드 체크: 2026-01-20 (16일 경과)
  → oocheck run 실행 권장
```

## 7. 팁 & 주의사항

### 7.1 권장 사항

- **매일 첫 세션**: `oostart run`으로 시작
- **장기 중단 후**: 출력된 권장 워크플로우 순서 준수
- **다수 경고 발생 시**: 우선순위 높은 항목부터 처리

### 7.2 주의사항

- `oostart`는 파일을 수정하지 않음 (읽기 전용 체크)
- 에러 발견 시 자동 수정하지 않음 (수동 확인 필요)
- 서브에이전트 병렬 실행 시 네트워크/CPU 사용량 증가 가능

### 7.3 자동화

`oostart`는 다른 워크플로우와 체이닝 가능:

```bash
# 세션 시작 → 코드 체크 → 테스트
oostart run && oocheck run && ootest run
```

## 8. FAQ

**Q1. oostart와 ooenv의 차이점은?**

| 스킬 | 목적 | 출력 |
|------|------|------|
| `oostart` | 세션 시작 준비 체크 | 체크리스트, 권장 조치 |
| `ooenv` | 환경 상세 정보 | d0009_env.md 생성 |

**Q2. common_guide.md는 어떻게 로드되나요?**

`oostart` 실행 시 자동으로 `.claude/guides/common_guide.md`를 읽어 세션 컨텍스트에 로드합니다.

**Q3. 서브에이전트 병렬 실행을 비활성화할 수 있나요?**

현재 버전은 병렬 실행 고정입니다. 순차 실행이 필요하면 개별 명령어 사용:
```bash
# Explore만
# (Explore 에이전트 직접 호출)

# task-checker만
# (task-checker 에이전트 직접 호출)
```

**Q4. 경고 메시지는 무시해도 되나요?**

| 경고 유형 | 안전성 | 조치 |
|----------|-------|------|
| 문서 누락 | 중간 | 다음 세션 전 생성 권장 |
| 코드 체크 미실행 (7일 이상) | 낮음 | 시간 날 때 실행 |
| 높은 우선순위 TODO (3개 이상) | 높음 | 즉시 확인 필요 |
