# OAIS 문서 번호 체계 (레퍼런스)

> **단일 진실 공급원(SSOT)**: 이 파일이 OAIS 전체 문서 번호 규칙의 기준.
> 파생 문서: `.claude/tutorial/33_문서번호및파일명규칙.md`

---

## 1. 기본 구조

```
d  {SP 2자리}  {문서번호 4자리}  _  {문서명}.md
│       │              │
│       │              └─ 4자리 일련번호 (0001~9999)
│       └─ SP 식별자 (SP00→00, SP04→40, SP07→70)
└─ 문서 prefix
```

**SP 번호 매핑**:

| SP | 2자리 | 예시 |
|----|-------|------|
| SP00 (공통) | 00 | d0001, d0002 |
| SP01 | 10 | d10001, d10002 |
| SP04 | 40 | d40001, d40002 |
| SP05 | 50 | d50001, d50002 |
| SP07 | 70 | d70001, d70002 |
| SP08 | 80 | d80001, d80002 |

## 2. 문서 번호 할당표

### 2.1 예약 번호 (0001~0019) — SP 공통 핵심

> **규칙**: 지정 스킬 외 임의 생성 금지.

| 번호 | 파일명 패턴 | 용도 | 생성 스킬 |
|------|------------|------|----------|
| 0000 | `d{SP}0000_manual.md` | 전체 사용 매뉴얼 | `oodoc manual` |
| 0001 | `d{SP}0001_prd.md` | PRD — 요구사항 정의 | `ooprd` |
| 0002 | `d{SP}0002_plan.md` | 구현 계획 (Feature/Task) | `ooplan` |
| 0003 | `d{SP}0003_test.md` | 테스트 케이스 | `ootest` |
| 0004 | `d{SP}0004_todo.md` | TODO / 이슈 추적 | `ootodo` |
| 0005 | `d{SP}0005_lib.md` | 라이브러리 정보 | `oolib` |
| 0006 | `d{SP}0006_db.md` | DB 구조 | `oodb` |
| 0007 | `d{SP}0007_data.md` | data/ 폴더 구조·설명 | `oodata` |
| 0008 | `d{SP}0008_user.md` | 사용자 관리 | `oouser` |
| 0009 | `d{SP}0009_env.md` | 환경 현황 | `ooenv` |
| 0010 | `d{SP}0010_history.md` | 변경 이력 | `oohistory` |
| 0011~0019 | — | 미할당 (예약) | — |

### 2.2 자유 추가 번호 (0020~0099) — SP별 선택 할당

> **규칙**: 스킬이 지정한 번호 우선. 빈 번호는 SP별로 자유 할당 가능.

| 번호 | 파일명 패턴 | 용도 | 생성 스킬 |
|------|------------|------|----------|
| 0020~ | `d{SP}0020_연구노트.md` | 연구노트 (연번: 0020, 0021...) | `oonote` |
| 0021~0099 | (자유) | SP별 자유 할당 | 수동/자유 |

**연구노트 연번 규칙**: 같은 SP 내 여러 연구노트는 0020, 0021, 0022... 순차 부여.

### 2.3 선택 문서 (0100~0999) — 특정 스킬 전용

> **규칙**: 해당 스킬 사용 시에만 생성. 사용하지 않는 SP는 생략 가능.

| 번호 | 파일명 패턴 | 용도 | 생성 스킬 |
|------|------------|------|----------|
| 0110 | `d{SP}0110_서베이.md` | 서베이/문헌조사 결과 | `oosurvey` |
| 0120 | `d{SP}0120_기술연구.md` | 기술 연구 메모 | `ooresearch` |
| 0200 | `d{SP}0200_소개PPT.md` | 프로젝트 소개 PPT | `ooppt --intro` |
| **0500~0599** | **`d{SP}05XX_사업화_*.md`** | **사업화 카테고리 (사업계획서·시장분석·BM·재무·IR 등)** | **수동/oowork** |
| 0900~0999 | (자유) | 참고용 문서 | 수동/자유 |

**0500번대 사업화 권장 세부 할당** (자유롭게 확장 가능):

| 번호 | 파일명 예시 | 용도 |
|------|-----------|------|
| 0500 | `d{SP}0500_사업계획서.md` | 사업계획서 본문 |
| 0510 | `d{SP}0510_시장분석.md` | 시장·경쟁 분석 |
| 0520 | `d{SP}0520_사업모델.md` | BM·수익구조 |
| 0530 | `d{SP}0530_재무계획.md` | 재무·투자 |
| 0540 | `d{SP}0540_IR자료.md` | IR/피칭 자료 |
| 0550 | `d{SP}0550_사업화전략.md` | GTM·로드맵 |
| 0560~0590 | (자유) | 사업화 보조 문서 |

### 2.4 상세 문서 (1000~9999) — oofeature 전용

> **규칙**: 1000번은 SP 상세문서 번호체계 인덱스 (필수). 1001번부터 `oofeature`로 생성.

| 번호 | 파일명 패턴 | 용도 | 생성 스킬 |
|------|------------|------|----------|
| **1000** | `d{SP}1000_상세문서인덱스.md` | **상세문서 번호체계 인덱스 (필수)** | `oodoc` |
| 1001~ | `d{SP}1001_상세{단계}_{기능명}.md` | 기능 단위 상세 문서 | `oofeature` |

**단계 키워드**: `_상세기획_` → `_상세설계_` → `_상세개발_` → `_상세검증_`

## 3. 번호 범위 요약

| 범위 | 성격 | 규칙 |
|------|------|------|
| 0000~0019 | 예약 (핵심) | 지정 스킬만 생성 |
| 0020~0099 | 선택 (자유) | 스킬 우선, 빈 번호 자유 할당 |
| 0100~0999 | 선택 (스킬 전용) | 해당 스킬 사용 시만 생성 |
| **1000** | **상세문서 인덱스 (필수)** | **oodoc 관리, 번호체계 정의** |
| 1001~9999 | 상세 문서 | oofeature 전용 |

## 5. 다음 번호 결정 방법

```bash
# 현재 SP의 상세 문서 목록
oofeature list   # → 마지막 번호 + 1

# 연구노트 연번 확인
ls 00_doc/sp{N}/d{SP}0*_연구노트.md
```
