# SP{SP} 프로젝트 체크리스트

## 문서이력관리
- v01 {DATE} — 초기 생성

> 이 문서는 **해당 프로젝트 전용 체크리스트**다. (oo* 스킬 자체 체크리스트와 별개)
> `oocheck add "내용"` 으로 항목 추가 · `oocheck run` 으로 체크 수행
> 항목 코드는 성격에 따라 자동 부여된다 — C코드·D문서·S보안·T테스트·E환경·G일반

---

## 코드 체계

| 접두사 | 성격 | 판정 키워드 |
|--------|------|-----------|
| C | 코드 | 코드·구현·함수·클래스·pylint·리팩토링·API·모듈·알고리즘 |
| D | 문서 | 문서·doc·PRD·가이드·README·주석·명세·튜토리얼 |
| S | 보안 | 보안·취약점·인증·권한·시크릿·토큰 |
| T | 테스트 | 테스트·pytest·커버리지·TC·검증 |
| E | 환경 | 환경·설정·의존성·패키지·빌드·배포 |
| G | 일반 | 위 미매칭 |

> 항목 형식: `### [접두사][3자리] [성격] 제목` + `#### 등록일: YYYY-MM-DD | 우선순위` + `#### 체크 내용`
> 번호는 접두사별 독립 일련번호 (C001·C002… / D001·D002…)
