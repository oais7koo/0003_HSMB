# oodoc check 체크리스트

> oodoc 스킬 자체 건강 상태 검증 항목 (범용 템플릿)
> 템플릿: .claude/templates/oo_checklist_template.md
>
> 프로젝트 특화 체크(예: 특정 페이지/엔드포인트/프레임워크)는 각 프로젝트 저장소의 이 파일에서 별도 C11~ 항목으로 확장한다.

| ID | 항목 | 검증 내용 | 심각도 |
|----|------|----------|--------|
| C01 | 필수 파일 존재 | SKILL.md, scripts/oodoc_*.py 핵심 파일 존재 여부 | CRITICAL |
| C02 | 버전 일치 | metadata version ↔ 서브명령어 테이블 (vXX) 일치 | ERROR |
| C03 | 00_doc 디렉터리 | 문서 저장 루트(`00_doc/sp{NN}/`) 접근 가능 | CRITICAL |
| C04 | 이력 테이블 5행 규칙 | 이력 테이블 행 수 제한 정책 준수 (`oodoc clear`로 유지) | WARNING |
| C05 | 교차 정합성 검증 | d0001_prd / d0002_plan / d0003_test 간 정합성 체크 기능 | WARNING |
| C06 | d0001~d0010 생성 | 표준 문서 10종 생성/갱신 루틴 존재 | ERROR |
| C07 | SP 귀속 검사 | 각 문서의 SP 번호(d{SP}XXXX) 네이밍 규칙 준수 | WARNING |

## check 출력 형식

```
[oodoc check]

C01 필수 파일 존재             [OK]
C02 버전 일치                  [OK]
C03 00_doc 디렉터리            [OK]
C04 이력 테이블 5행 규칙       [OK]
C05 교차 정합성 검증           [OK]
C06 d0001~d0010 생성           [OK]
C07 SP 귀속 검사               [OK]

소계: OK:7 | WARN:0 | FAIL:0
```

## 심각도 기준

| 심각도 | 기호 | 의미 |
|--------|------|------|
| CRITICAL | ❌ | 스킬 동작 불가 (즉시 조치) |
| ERROR | ⚠️ | 정상 동작 어려움 (24시간 내) |
| WARNING | ⚡ | 품질 저하 (1주 내) |
| INFO | ℹ️ | 개선 권장 (백로그) |

## 프로젝트 특화 확장 가이드

각 프로젝트에서 고유 검증이 필요한 경우 C11~ 항목으로 추가:

| 프로젝트 유형 | 확장 예시 |
|--------------|-----------|
| FastAPI 백엔드 | "엔드포인트별 상세설계 문서(d4XXXX) 존재 여부" |
| Streamlit 앱 | "pages/ 디렉터리별 상세설계 커버리지" |
| Spring DB 프로젝트 | "이벤트 기반 처리, 우선순위, Spring DB 연동 흐름 기술" |

> 프로젝트 특화 확장은 oosync 동기화 대상에서 **제외**되어야 하며, 각 프로젝트가 자체 유지한다.
