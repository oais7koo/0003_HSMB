## 4. 워크플로우 상세

### 4.1 run 워크플로우 (종합 연구)

```
ooresearch run [topic] --ai
    |
    |-> 1. 주제 분석: 범위 정의, 핵심 키워드 추출
    |-> 2. {paper_root}/ 스캔 (Explore 에이전트)
    |       ├─ 관련 논문 필터링 (키워드 매칭)
    |       └─ 보유 논문 목록 확정
    |-> 3. 부족 판단 → oopaper 자동 호출 (필요 시)
    |       └─ oopaper run --lang en [keywords]
    |-> 4. 병렬 연구 수행 ─────────────────────
    |       ├─ [A] academic-researcher: 보유 논문 심층 분석
    |       ├─ [B] document-specialist: SOTA 웹 조사 (PWC, ArXiv)
    |       └─ [C] data-analyst: 벤치마크 수치 비교
    |-> 5. 결과 통합 (task-executor)
    |       ├─ SOTA 비교 분석
    |       ├─ 방법론 분류
    |       ├─ 갭 분석
    |       └─ 연구 방향 제안
    |-> 6. 연구 문서 저장
```

### 4.2 sota 워크플로우 (빠른 스냅샷)

```
1. {paper_root}/ 스캔 → 주제 관련 최신 논문 추출
2. document-specialist: SOTA 리더보드 실시간 조회
3. Top-5 모델 + 최근 6개월 하이라이트 출력
```

### 4.3 compare 워크플로우

```
1. 보유 논문에서 비교 대상 논문 선정
2. 부족 시 oopaper로 비교 논문 추가 수집
3. academic-researcher × N: 각 접근법 분석 (병렬)
4. data-analyst: 동일 벤치마크 기준 비교표 생성
5. 트레이드오프 분석 + 추천
```

### 4.4 병렬 실행 구성

```
[Explore] + [document-specialist × 2~3] + [academic-researcher × N]
    ↓ (동시)
[data-analyst] + [academic-researcher (추가 분석)]
    ↓
[task-executor] (문서 통합)
```

## 5. 출력 문서 형식

**파일명**: `00_doc/d{SP}0200_research_{topic}.md` (기본)

```markdown
# {topic} 연구 보고서

## 문서이력관리
| 버전 | 날짜 | 변경내용 |

## 1. 연구 개요
- 주제, 범위, 조사 기간, 도메인
- 활용 논문: 보유 N편 + 신규 수집 M편

## 2. SOTA 현황
### 2.1 주요 벤치마크 리더보드
| 순위 | 모델 | 성능 | 출처 | 연도 |
### 2.2 Top SOTA 모델 요약 (Top-5)
- 각 모델: 아키텍처, 핵심 기법, 성능, 장단점

## 3. 방법론 분류
### 3.X [접근법 카테고리]
- 대표 모델, 핵심 아이디어, 성능 범위, 보유 논문 연결

## 4. 핵심 기술 트렌드
- 최근 3년 주요 변화, 부상하는 기법

## 5. SOTA 대비 갭 분석
### 5.1 성능 갭 (정량)
### 5.2 방법론 갭 (정성)
### 5.3 미탐 영역

## 6. 연구 방향 제안
### 6.1 단기 (6개월 이내)
### 6.2 중기 (1~2년)
### 6.3 장기 (3년+)

## 7. 참고 자료
- 보유 논문 목록 ({paper_root}/ 경로 포함)
- 신규 수집 논문
- 웹 소스 링크
```
