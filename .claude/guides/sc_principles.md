# SuperClaude Principles Reference

> 이 파일은 SuperClaude 개발 원칙 참조 가이드입니다.
> 상세 내용은 글로벌 설정 ~/.claude/PRINCIPLES.md 참조

## 핵심 원칙

- Evidence > assumptions | Code > documentation | Efficiency > verbosity

## SOLID 원칙
- Single Responsibility: 하나의 변경 이유
- Open/Closed: 확장에 열림, 수정에 닫힘
- Liskov Substitution: 파생 클래스 대체 가능
- Interface Segregation: 불필요한 인터페이스 의존 금지
- Dependency Inversion: 추상화에 의존

## 설계 원칙
- DRY: 중복 제거
- KISS: 단순하게
- YAGNI: 필요한 것만
- Composition Over Inheritance
- Separation of Concerns
- Loose Coupling / High Cohesion
