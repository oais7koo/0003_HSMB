# TODO 기록 형식 표준안

## 문서 정보
- **문서번호**: todo_format_standard
- **작성일**: 2026-01-02
- **목적**: 모든 oo 스킬에서 d0004_todo.md에 TODO를 기록할 때 사용하는 통일된 형식 정의

---

## 1. 핵심 원칙

1. **단일 형식**: 모든 스킬은 동일한 테이블 형식 사용
2. **실제 사용 기반**: 현재 d0004_todo.md에서 사용 중인 형식을 표준으로 채택
3. **확장성**: 분류 태그로 이슈 유형 구분
4. **자동화 친화**: 파싱 가능한 일관된 구조
5. **태그 통일**: d0004 ↔ d0010 간 동일 태그 체계 사용

### 1.1 문서 역할 구분 (중요)

> **참조**: `.claude/guides/common_guide.md` 섹션 6.2

| 작업 유형 | 기록 문서 | 예시 |
|----------|----------|------|
| **에러/버그** | `d0004_todo.md` | Pylint 에러, 테스트 실패, 런타임 버그 |
| **개발 작업** | `d0002_plan.md` | 기능 구현, 기술 부채, 성능 최적화 |

**주의**: 개발이 필요한 작업(기능 구현, 리팩토링, 기술 부채)은 `d0004_todo.md`가 아닌 `d0002_plan.md`의 Epic/Feature로 등록합니다.

## 2. 표준 TODO 형식

### 2.1 테이블 구조 (필수)

| ID | 발생일 | 분류 | 내용 | 우선순위 | 상태 |
|----|--------|------|------|---------|------|
| T001 | 2026-01-02 | BUGFIX | oo\file.py:123 - 에러 설명 | 높음 | 대기 |

### 2.2 필드 정의

| 필드 | 형식 | 설명 | 예시 |
|------|------|------|------|
| **ID** | T + 3자리 숫자 | 순차적 고유 식별자 | T001, T042 |
| **발생일** | YYYY-MM-DD | 이슈 등록일 | 2026-01-02 |
| **분류** | 태그명 | 이슈 유형 분류 | BUGFIX, UPDATE |
| **내용** | 자유 텍스트 | 파일:라인 + 설명 | oo\seal.py:148 - cv2 import 오류 |
| **우선순위** | 높음/중간/낮음 | 처리 긴급도 | 높음 |
| **상태** | 상태값 | 현재 처리 상태 | 대기, 진행중, 완료 |

### 2.3 분류 태그 (통일 체계)

> **태그 체계 통일**: d0004 → d0010 이동 시 변환 없이 동일 태그 사용

| 태그 | 의미 | 사용 스킬 |
|------|------|----------|
| BUGFIX | 버그/에러 수정 (구문, 런타임, 테스트 실패) | oocheck, ootest |
| HOTFIX | 보안/긴급 수정 (취약점, 긴급 패치) | oocheck |
| UPDATE | 의존성 업데이트 (패키지, 버전) | ooenv |
| FEATURE | 기능 관련 (PRD 정합성, 미구현) | ooprd |
| IMPROVE | 최적화/개선 (성능, 아키텍처) | oodb, oolib |
| DOCS | 문서 수정 (불일치, 누락) | ooprd, oolib |
| REFACTOR | 코드 리팩토링 | oolib |
| CONFIG | 설정 변경 | ooenv |
| MISC | 미분류/기타 | - |

### 2.4 우선순위 기준

| 우선순위 | 기준 | 처리 기한 |
|---------|------|----------|
| **높음** | 빌드 실패, 보안 취약점, 런타임 에러 | 즉시 |
| **중간** | 기능 저하, 경고, 테스트 실패 | 1주 내 |
| **낮음** | 최적화, 코드 스타일, 문서 개선 | 여유 시 |

### 2.5 상태 값

| 상태 | 의미 |
|------|------|
| **대기** | 등록됨, 미처리 |
| **진행중** | 분석/수정 진행 중 |
| **완료** | 해결됨 → "해결된 이슈" 섹션으로 이동 |
| **보류** | 외부 의존성으로 보류 |
| **False Positive** | 오탐, 수정 불필요 |

### 2.6 이슈 워크플로우

[현재 이슈] → 해결 → [해결된 이슈 (1~2주 모니터링)] → oohistory run → [d0010 아카이브]

| 단계 | 위치 | 설명 |
|------|------|------|
| 1. 발생 | 현재 이슈 | 새 이슈 등록 |
| 2. 해결 | 해결된 이슈 | 해결 완료, 재발 모니터링 (1~2주) |
| 3. 아카이브 | d0010_history.md | 안정 확인 후 oohistory run 실행 |
