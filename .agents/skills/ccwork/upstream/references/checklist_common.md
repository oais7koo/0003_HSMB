# oowork 공통 체크리스트

> 모든 프로세스에 적용되는 범용 검증 규칙. 프로세스별 체크리스트가 이 규칙을 상속한다.

## 1. 문서 정합성

| # | 규칙 | 설명 |
|---|------|------|
| G01 | 파일명-내용 일치 | 파일명의 번호, 주제, 유형이 본문 내용과 일치 |
| G02 | 문서 스코프 준수 | 해당 문서의 담당 범위만 포함, 타 문서 범위 혼입 금지 |
| G03 | 상호 참조 정확성 | 다른 문서를 참조할 때 경로와 파일명이 실제 존재하는 파일과 일치 |

## 2. 넘버링 및 구조

| # | 규칙 | 설명 |
|---|------|------|
| G04 | 섹션 넘버링 일관성 | 전체 문서에서 동일한 넘버링 깊이 유지 (2단계 권장) |
| G05 | 번호 없는 섹션 금지 | 모든 `##` 레벨 섹션에 번호 부여 |
| G06 | 순번 연속성 | 섹션 번호가 빠짐 없이 연속 |

## 3. 출력 양식 준수

| # | 규칙 | 설명 |
|---|------|------|
| G07 | 양식 구조 일치 | 지정된 양식(템플릿)의 섹션 구조를 벗어나지 않음 |
| G08 | 양식 외 섹션 금지 | 양식에 정의되지 않은 섹션 추가 금지 |
| G09 | 이중 표현 금지 | 같은 내용을 마크다운 표와 코드블록 등 두 가지로 중복 작성 금지 |

## 4. 참조 무결성

| # | 규칙 | 설명 |
|---|------|------|
| G10 | 인용-참고문헌 매칭 | 본문에서 인용한 항목이 참고문헌에 전수 포함 |
| G11 | 참고문헌-원본 매칭 | 참고문헌의 폴더ID가 실제 원본 폴더에 존재 |
| G12 | 고아 참고문헌 없음 | 참고문헌에만 있고 본문에서 인용하지 않는 항목 없음 |
| G19 | 출처 없는 수치·사실 인용 금지 | 본문의 모든 수치·통계·사실 주장은 확인된 출처([XNNN])를 동반. 출처 미확보 내용은 작성하지 않음. 자체 산정은 [GNNN]+"자체 산정" 표기 필수 |

| G14 | 기호 일관성 | 문서 내 기호 사용 규칙 통일 (허용 기호와 금지 기호 명확) |

| G23 | **인용 ID ↔ 부록 등재 매칭** | 본문 인용 블록 또는 부록 본문에 등장한 모든 인용 ID(`[A001]`/`[F012]`/`[D001]` 등)는 부록 §A·B·C 표에 등재되어야 하며, 부록 등재 항목은 본문 또는 인용 블록에서 1회 이상 인용되어야 함. 고아 인용 ID(부록 미등재) 또는 고아 등재 항목(본문 미인용) 1건이라도 있으면 BLOCK. 외부 출처는 URL 동반, 자체 작성 자료(F접두사)는 "자체 산정·자체 분석" 명시 필수 |
| G24 | **샘플 기준 제안서 문체·형식 검토** | 제안서 산출물(70/71) 작성 완료 후 `templates/제안/` 또는 `samples/` 폴더의 샘플 문서와 문체·작성 형식·단락 구조·항목 깊이를 비교 검토. 샘플과 현저히 다른 서술 방식·표 구조·항목 계층이 발견되면 수정 후 재검증. 샘플 미존재 시 기존 완성본(70/71)을 기준 삼아 내부 일관성 확인 |
| G25 | **항목 내 ①②③④+→ 화살표 체인 금지 (S01)** | **S01 개조식 문서에서 `○`/`□`/`-` 항목 내에 `① 노이즈 제거 → ② ROI 추출 → ③ …` 형태의 원문자+화살표 체인 표현 사용 금지. 각 단계를 `-` 3단 마커 개별 항목으로 분리하거나 표로 변환. `scripts/oowork_check_style.py`로 자동 검출(G25). 위반 시 BLOCK** |
| G26 | **원문자(①~⑳) 전면 금지 (S01)** | **S01 개조식 문서에서 원문자(①②③④⑤ 등) 단독 사용 금지. □/○/- 3단 마커 체계 외 별도 기호 체계 혼용 불가. 순서 표현이 필요하면 표 또는 `-` 항목으로 대체. `scripts/oowork_check_style.py`로 자동 검출(G26). 위반 시 BLOCK** |
