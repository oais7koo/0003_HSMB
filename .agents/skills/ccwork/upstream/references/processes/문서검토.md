# 문서검토

## meta
- version: v02
- description: 검토기준 설정 → 문서 추출·분석·검토 → 개선사항 도출 파이프라인

## folder_structure
- 기본 세트: 00_검토기준, 01_원본, 02_추출, 03_구조분석, 04_검토, 05_수정제안
- 확장: 06_수정본, 07_최종 등 필요 시 추가
- 네이밍: {폴더번호}{순번}_{내용}.md (예: 21_추출본.md)
- N 할당: 기존 파일 최대 번호 + 1

## steps

### step.0 검토기준 설정
- tool: 사용자 입력 (직접 작성)
- input: 사용자가 정의한 검토 관점, 평가 항목, 중점 확인사항
- output: {workdir}/00_검토기준/01_검토기준.md
- confirm: true
- rules: 사용자가 파일을 직접 작성하거나, 대화로 입력한 내용을 정리하여 생성
- sub_steps:
  1. 00_검토기준/ 폴더 생성
  2. 사용자에게 검토기준 입력 요청 (관점, 항목, 중점사항)
  3. 입력 내용을 01_검토기준.md로 구조화 저장
  4. 기준 항목에 ID 부여 (U01, U02, ... — User criteria)
  5. 검토기준 확인 후 다음 단계 진행
- template: |
    # 검토기준
    ## 검토 관점
    - (예: 기술적 타당성, 예산 적정성, 일정 실현가능성 등)
    ## 중점 확인사항
    | ID | 항목 | 설명 | 심각도 |
    |----|------|------|--------|
    | U01 | (항목명) | (확인할 내용) | Critical/Major/Minor |
    ## 기타 참고사항
    - (추가 맥락, 배경, 제약조건 등)

### step.1 문서 추출
- tool: oowork extract (HWP/PDF/DOCX → MD 변환)
- input: 원본 문서 (HWP, PDF, DOCX 등)
- output: {workdir}/02_추출/21_{문서명}.md
- confirm: true
- sub_steps:
  1. 원본 파일 식별 (형식, 크기, 페이지 수)
  2. 비텍스트 문서 → 마크다운 변환 (hwp_to_md.py / Read / python-docx)
  3. 변환 품질 확인 (표, 이미지 참조, 특수문자 보존)
  4. 원본은 01_원본/에 보존

### step.2 구조 분석
- tool: 직접 작성
- input: ${step.1.output}
- output: {workdir}/03_구조분석/31_구조분석.md
- depends: step.1
- confirm: true
- sub_steps:
  1. 문서 목차·섹션 구조 파악
  2. 섹션별 분량 분석 (글자수, 비중)
  3. 핵심 주장·논거 추출
  4. 그림/표/참고문헌 목록 정리
  5. 문서 유형 판별 (제안서/보고서/논문/공문 등)
  6. 구조 요약표 생성

### step.3 내용 검토
- tool: 직접 작성
- input: ${step.0.output}, ${step.1.output}, ${step.2.output}
- output: {workdir}/04_검토/41_검토결과.md
- depends: step.0, step.1, step.2
- confirm: true
- rules: 객관적 근거 기반, 항목별 심각도(Critical/Major/Minor) 분류
- user_criteria: ${step.0.output} 의 검토기준(U01~)을 기본 5대 관점에 추가 적용
- sub_steps:
  1. **사용자 검토기준 로드** (00_검토기준/01_검토기준.md → U01~ 항목 적용)
  2. 논리 흐름 검토 (주장-근거 연결, 비약 여부)
  3. 사실 정확성 검증 (수치, 날짜, 명칭, 인용)
  4. 일관성 검토 (용어, 표기, 문체, 수치 간 정합)
  5. 완성도 검토 (누락 항목, 미완성 섹션)
  6. 표현·가독성 검토 (문장 길이, 모호 표현, 오탈자)
  7. **사용자 검토기준별 검토** (U01~ 각 항목에 대해 개별 판정)
  8. 검토 결과를 항목별 테이블로 정리 (기본 관점 + 사용자 기준 통합)

### step.4 수정 제안
- tool: 직접 작성
- input: ${step.2.output}, ${step.3.output}
- output: {workdir}/05_수정제안/51_수정제안서.md
- depends: step.2, step.3
- confirm: true
- sub_steps:
  1. 검토 결과 심각도별 정렬 (Critical → Major → Minor)
  2. 항목별 수정 방안 구체적 제시 (현재 → 수정안)
  3. 섹션별 보강 권고 (추가할 내용, 삭제할 내용)
  4. 우선순위 액션 리스트 (즉시/단기/선택)
  5. 수정 제안 요약표 생성

## extension
- 06_수정본/: 수정 반영 결과물
- 07_최종/: 최종 검토 완료본
- 폴더 추가 시 번호 연속 할당, 기존 번호 변경 금지

## naming
- 패턴: {폴더번호}{순번}_{내용}.md
- 00_검토기준: 01_검토기준.md
- 01_원본: (원본 파일 보존)
- 02_추출: 21_추출본.md, 22_추출본_부록.md
- 03_구조분석: 31_구조분석.md
- 04_검토: 41_검토결과.md, 42_세부검토.md
- 05_수정제안: 51_수정제안서.md, 52_액션리스트.md
- N 할당: 기존 파일 최대 번호 + 1

## validation
- step.0: 검토기준 파일 존재 + U01~ 항목 1개 이상 정의
- step.1: 추출 파일 존재 + 원본 대비 주요 내용 보존
- step.2: 구조분석 파일 존재 + 목차·섹션 목록 포함
- step.3: 검토결과 파일 존재 + 기본 5대 관점 + 사용자 기준(U01~) 전수 판정
- step.4: 수정제안서 존재 + 우선순위 액션 리스트 포함
