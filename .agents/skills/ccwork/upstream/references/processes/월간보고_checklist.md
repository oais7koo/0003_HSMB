# 월간보고 체크리스트

> 공통 규칙 상속: `../checklist_common.md` (G01~G18)
> 이 문서는 월간보고 프로세스에 특화된 추가 규칙만 정의

## step.0 양식·참고 추출

| # | 규칙 | 검증 방법 |
|---|------|---------|
| A01 | 01_양식 MD 변환 완료 | 01_양식/{N}_*.md 1개 이상 존재 |
| A02 | 02_참고 MD 변환 완료 | 02_참고/{N}_*.md 1개 이상 존재 |
| A03 | 원본 보존 | HWP/HWPX 원본 파일 삭제되지 않음 |
| A04 | 03_작성 폴더 존재 | 03_작성/ 폴더 생성됨 (없으면 자동 생성) |

## step.1 양식·참고 분석

| # | 규칙 | 검증 방법 |
|---|------|---------|
| B01 | 메타 필드 식별 | 업무기간/보고자/작성자/보고일 4개 필드 인식 |
| B02 | 제목 패턴 식별 | `{소속} ’{YY}년 {MM}월 보고` 패턴 인식 |
| B03 | 섹션 3개 식별 | 1.실적및계획 / 2.이슈사항 / 3.기타사항 |
| B04 | 범례 식별 | □(신규), ◨(진행중), ■(완료) 인식 |
| B05 | 좌우 2단 구조 인식 | 실적(좌) / 계획(우) 컬럼 분리 인식 |

## step.2 사용자 인터뷰

| # | 규칙 | 검증 방법 |
|---|------|---------|
| C01 | 메타 5필드 수집 | 소속·업무기간·보고자·작성자·보고일 모두 수집 |
| C02 | 섹션별 단계 진행 | 메타 → 실적 → 계획 → 이슈 → 기타 순서 준수 |
| C03 | 항목별 상태 부여 | 모든 실적·계획 항목에 □/◨/■ 중 하나 부여 |
| C04 | 자유 응답 허용 | 이슈·기타는 빈 응답 시 "특이사항 없음" 자동 입력 |
| C05 | 응답 요약 확인 | 작성 전 사용자가 응답 전체를 한 번 확인 |

## step.3 보고서 작성

| # | 규칙 | 검증 방법 |
|---|------|---------|
| D01 | 산출물 위치 | 03_작성/ 하위 저장 |
| D02 | 파일명 규칙 | `{YY-MM}_월간보고({부서명}).md` 패턴 준수 |
| D03 | 헤더 메타 완전 | 업무기간·보고자·작성자·보고일 모두 채워짐 |
| D04 | 범례 헤더 유지 | `* 범례 (□ : 신규, ◨ : 진행중, ■ : 완료)` 표기 |
| D05 | 머리표 일관 | 1번 섹션 `✔`, 이슈·기타는 `ㅇ` 사용 |
| D06 | 들여쓰기 트리 | 사업명 → `-` 세부 작업 2단 깊이 |
| D07 | 미래형/과거형 일치 | 실적=과거 완료형, 계획=미래 예정형 |
| D08 | 인용·출처 미포함 | 외부 인용/출처 표기 없음 |
| D09 | HWP 붙여넣기 가능 | 본문이 양식 셀에 그대로 들어갈 수 있는 형태 |
