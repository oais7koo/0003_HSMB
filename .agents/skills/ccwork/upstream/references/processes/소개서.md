# 소개서

## meta
- version: v01
- description: 회사/프로젝트/기술 소개서 작성 4단계 파이프라인 (결과/제품/기술 중심)

## folder_structure
- 기본 세트: 01_자료, 02_콘텐츠, 03_원고, 04_산출물
- 확장: 05_검토, 06_최종 등 필요 시 추가
- 네이밍: {폴더번호}{순번}_{내용}.md (예: 11_회사개요.md)
- N 할당: 기존 파일 최대 번호 + 1

## 작성 원칙
- **결과/제품/기술 중심** — "무엇을 했는가"가 아니라 "무엇을 만들었고, 무엇이 가능한가"
- 과정/방법론 설명은 최소화, 산출물/성과/수치 부각
- 대상 청중(고객/투자자/파트너)의 관심사에 맞춰 구성
- **개발 로드맵/일정 미포함** — 소개서는 "완성된 제품/기술"을 보여주는 문서이므로 개발 계획·일정·PoC 단계 등은 기재하지 않음
- **마무리/Thank you 슬라이드 미포함** — 모든 슬라이드에 실질 콘텐츠를 배치. 발표용 "마무리" 페이지는 홈페이지·소개서에 불필요

## steps

### step.0 자료 수집
- tool: 직접 작성 + Read (기존 문서 참조)
- input: 기존 프로젝트 문서 (PRD, 결과물, 성과 데이터 등)
- output: {workdir}/01_자료/1N_{자료명}.md
- confirm: true
- sub_steps:
  1. 소개 대상 파악 (회사/프로젝트/기술/제품)
  2. 기존 문서 수집 (PRD, README, 평가 결과, 데모 등)
  3. 핵심 수치 추출 (성능, 규모, 비용 절감, 사용자 수 등)
  4. 시각 자료 수집 (스크린샷, 다이어그램, 차트)
  5. 경쟁 우위/차별점 정리
  6. 대상 청중 정의 (누구에게 보여줄 것인가)

### step.1 콘텐츠 구성
- tool: 직접 작성
- input: ${step.0.output}
- output: {workdir}/02_콘텐츠/2N_{항목명}.md
- depends: step.0
- confirm: true
- sub_steps:
  1. 소개서 목차/슬라이드 구성안 작성
  2. 항목별 핵심 메시지 1줄 정의
  3. 항목별 근거 수치/데이터 매핑
  4. 시각 자료 배치 계획
  5. 스토리라인 검증 (청중 관점에서 흐름 확인)

### step.2 원고 작성
- tool: 직접 작성
- input: ${step.1.output}
- output: {workdir}/03_원고/d{SP}0200_프로젝트소개PPT.md
- depends: step.1
- confirm: true
- sub_steps:
  1. 슬라이드별 제목 + 본문 + 발표 포인트 작성
  2. 테이블/차트 데이터 정리
  3. 결과/성과 수치 정확성 검증
  4. 전체 톤&매너 통일 (간결, 수치 중심, 전문적)
  5. 원고 최종 검토

### step.3 산출물 제작
- tool: ooppt run --intro
- input: ${step.2.output}
- output: {workdir}/04_산출물/d{SP}0200_프로젝트소개PPT.pptx
- depends: step.2
- confirm: true
- sub_steps:
  1. ooppt로 PPT 생성 (원고 기반)
  2. 디자인/레이아웃 확인
  3. 시각 자료 삽입 (필요 시)
  4. 최종 검토 및 수정
  5. PDF 버전 생성 (필요 시 oopdf)
