## steps

### step.0 RFP 분석
- tool: 직접 작성
- input: RFP 문서 (공고문, 과업지시서, 제안요청서, 양식)
- output: {workdir}/01_RFP/11_RFP분석.md, {workdir}/01_RFP/12_양식분석.md
- confirm: true
- sub_steps:
  1. RFP 문서 전문 읽기 (HWP → HWPX → PDF → MD 변환 후 읽기, PDF → Read/pdfplumber)
  2. 원본 문서 목록 테이블 작성 (11_RFP분석.md 상단 필수) — 파일명, 유형(공고/과업/양식), 페이지수, 분석여부(O/X)
  3. 핵심 요구사항 추출 (사업 개요, 목적, 범위, 일정, 예산)
  4. 평가 기준 분석 (배점표, 가점/감점 항목) — 해당 시
  5. 필수 제출 서류·양식 항목 목록 정리
  6. 기술 요구사항 항목별 분해
  7. 제약 조건 정리 (기간, 인력, 장비, 기술 스택)
  8. 차별화 포인트 도출
  9. RFP 분석 결과 문서 생성 (11_RFP분석.md) — 골격: `templates/제안/11_RFP분석_템플릿.md`
  10. 제안서 양식 분석 문서 생성 (12_양식분석.md) — 골격: `templates/제안/12_양식분석_템플릿.md` / 양식별 필드 목록, 입력 형식(텍스트/표/체크), 글자수 제한, 필수/선택 구분

### step.2 선행연구
- tool: oopaper + oosurvey
- input: RFP 기술 키워드(--keywords), 제안 주제(--topic), ${step.0.output}, ${step.1.output}
- output:
    overview: {workdir}/03_선행연구/30_선행연구개요.md
    papers: 03_paper/11_paper_en/{폴더ID}/ (서머리, 영문, 한글 완료 상태)
    list: {workdir}/03_선행연구/31_논문목록.md
    analysis: {workdir}/03_선행연구/39_{폴더ID}_{논문약칭}.md (논문별 핵심 분석, EN+KO)
    surveys: {workdir}/03_선행연구/3N_{세부기술명}_서베이.md (세부기술별 1개씩, 32~38)
- depends: step.1
- confirm: true
- sub_steps:
  1. **[선행연구 기획] 30_선행연구개요.md 생성** ← confirm 필수, 수집 실행 전 사용자 확인
     - RFP/과업 기술 구조에서 세부기술별 수집 **키워드 도출 + RFP 근거 명시** (왜 이 키워드인가)
     - 세부기술별 검색 범위(포함/제외)·전략·목표 편수 정의
     - 세부기술별 서베이 **주제·범위·핵심 질문** 정의 (각 세부기술 1개 섹션)
     - 기존 SOTA 서베이(05_기술연구/) 갭 분석 결과 반영 — 갭 보강 수집 시
     - 공통 제외 기준 사전 정의 (도메인 외, 품질 미달, 오매핑 처리 방침)
  2. 사용자 확인 후 — `oopaper search "키워드"` — arXiv, Semantic Scholar 검색
  3. 사용자 확인 — 다운로드할 논문 선택
  4. `oopaper download` — PDF 자동 다운로드
  5. `oopaper run` — Phase 0~6 (이동, 서머리, 영문, 한글, 참고문헌, paper_list)
  6. `oopaper keyword-sync` — 키워드 동기화
  7. 수집 논문 목록 리포트 출력 → {workdir}/03_선행연구/31_논문목록.md 저장
  8. 30_선행연구개요.md 수집 실적 + 세부기술별 논문 배분 매핑 업데이트
  9. **[논문별 분석] 39_{폴더ID}_{논문약칭}.md 생성** — 배분 논문 전수, EN+KO 포함
     - 입력: 각 논문의 `*_00_*_서머리.md` (전문 재읽기 불필요)
     - 추출 항목: ① 초록(한국어 2~3문장) ② 핵심 알고리즘·방법론 ③ 주요 성과(정량 수치) ④ 서베이 활용 포인트
     - KO 보고서도 동일 형식 적용
     - Agent(haiku) 병렬 처리 — 논문당 독립 실행
  10. 사용자 확인 후 — 세부기술별 서베이 병렬 생성 (39_*.md 참조)
     - 각 세부기술: `oosurvey run --topic "세부기술명" --paper-dir 03_paper/11_paper_en`
     - 출력 파일명: 32_{세부기술1}.md, 33_{세부기술2}.md … (세부기술 수만큼)
     - KO 국내 보고서는 관련 세부기술 서베이에 통합 (별도 파일 불필요)
  11. 서베이 문서 구조 검증 (섹션, 비교표, 시사점) + 기술 갭 분석 확인
  12. RFP 요구사항 대비 기술 커버리지 매핑 (30_선행연구개요.md 업데이트)

### step.5 창의기획
- tool: 직접 작성
- input: ${step.3.output}, ${step.4.output}
- output: {workdir}/06_창의기획/61_창의기획.md
- depends: step.3, step.4
- confirm: true
- purpose: |
    두 단계로 구성: ① 창의적 접근 탐색 (발산→수렴) — 목표 달성을 위한 다양한 방법 발굴
                    ② 제안 전략 설계 (메시징) — 선택된 접근법을 어떻게 설득력 있게 전달할 것인가
    step.4 기술연구가 "현재 SOTA 기반 문서"라면, step.5는 "목표 달성을 위한 창의적 탐색"이다.
- sub_steps:
  0. **[창의적 접근 탐색 — 발산→수렴]** (DP 확정 전 필수, 핵심 단계)
     - **발산**: 목표 달성 방법을 다각도로 탐색 → 최소 5~10개 아이디어 도출
       * 기존 SOTA의 예상치 못한 조합·변형
       * 인접 도메인 기술 이전 (자율주행·의료영상·제조검사·우주항공 등)
       * 미래 기술 조기 적용 (Event Camera·SNN·Foundation Model Adapter 등)
       * SCAMPER (대체·결합·응용·수정·다른 용도·제거·역전) 기법 적용
       * 역발상: "전혀 다른 방식으로 접근한다면?" 가정에서 출발
     - **수렴**: 도출된 아이디어를 3축 평가 → 채택/보류 결정
       * 실현가능성 (기술 성숙도 TRL 1~9, 현재 프로젝트 기간 내 가능 여부)
       * 임팩트 (성능 개선폭, 비용 절감, 차별성)
       * 제안 적합성 (RFP 요구사항 부합, 심사위원 이해 가능성)
     - 채택 아이디어 → 하위 단계 §1 차별화 포인트(DP1~DP3)로 발전
  1. 창의 탐색 결과 기반 차별화 포인트 최종 확정 (최소 3개, 정량 수치 포함)
  2. **설득 스토리라인 설계** — 제안 전체를 관통하는 서사 구조 1장 요약 (문제→접근→해결→기대효과 흐름)
  3. **섹션별 임팩트 문장 초안** — 각 주요 섹션의 오프닝 문장 1줄 (평가자가 첫눈에 읽는 문장)
  4. **평가자 시선 시뮬레이션** — 고배점 섹션 강조 전략, 평가자가 가장 먼저 볼 포인트 파악
  5. 비주얼 요소 계획 — 그림/표/다이어그램 배치 계획, 메시지 강화 역할 확인 (장식 0건)
  6. 창의기획 문서 생성 (61_창의기획.md) — 골격: `templates/제안/61_창의기획_템플릿.md` / 제안작성 시 참조 기준으로 활용

  3. 필드별 입력값 작성 (양식 형식에 맞게)
  4. 장문 기재란 본문 작성 (HWP 붙여넣기용, 선택한 본문 스타일 적용)
  5. 양식 항목 전수 대응 검증 (누락 0건)
  6. 용어 일관성·수치 정합성 검수
  7. 오탈자/문법 검수
  8. hwp 붙여넣기용 txt 추출 — 해당 시
  9. (선택) Word/PDF 변환 — `ooword run` 또는 `oopdf run`

## validation
- step.0: 분석 파일(11_RFP분석.md) + 양식분석 파일(12_양식분석.md) 존재 + 요구사항·양식 항목 추출 완료
- step.1: 자료 파일 1개 이상 존재 + 핵심 정보 기재
- step.2: [선택] 30_선행연구개요.md 생성 + 사용자 확인 완료 + 31_논문목록.md 생성 + 39_*.md 논문별 분석 생성(배분 논문 전수, EN+KO) + 세부기술별 서베이 파일(3N_*.md) 생성 + 섹션 구조(개요, 동향, 비교표, 시사점) + RFP 기술 커버리지 매핑 — 생략 가능(논문 미필요 시)
- step.3: 04_기획/ 내 파일 존재 + 구조/목차 정의 + 마일스톤 설정
- step.4: 05_기술연구/ 내 파일 존재 + 차별화 포인트 3개 이상 + 정량 근거 + 파일명 5x_ 준수
- step.5: 06_창의기획/61_창의기획.md 존재 + 차별화 포인트 3개 이상(정량 수치 포함) + 스토리라인 정의 + 임팩트 문장 초안
- step.6: 07_제안작성/ 내 파일 존재 + 양식 항목 전수 대응 + 통합본(7x_통합본.md) 생성 + 파일명 7x_ 준수
- step.7: 08_완성도검수/81_완성도검수.md 존재 + 창의기획 반영 확인 + 개선 사항 step.6 전수 반영 완료
