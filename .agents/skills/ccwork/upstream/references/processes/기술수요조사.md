# 기술수요조사

## meta
- version: v01
- description: 논문 수집부터 기술수요조사서 작성까지 5단계 파이프라인

## steps

### step.0 논문 수집
- tool: oopaper
- input: 키워드(--keywords), 주제(--topic)
- output: 03_paper/11_paper_en/{폴더ID}/ (서머리, 영문, 한글 완료 상태)
- confirm: true
- sub_steps:
  1. `oopaper search "키워드"` — arXiv, Semantic Scholar 검색
  2. 사용자 확인 — 다운로드할 논문 선택
  3. `oopaper download` — PDF 자동 다운로드
  4. `oopaper run` — Phase 0~6 (이동, 서머리, 영문, 한글, 참고문헌, paper_list)
  5. `oopaper keyword-sync` — 키워드 동기화
  6. 수집 논문 목록 리포트 출력

### step.1 서베이
- tool: oosurvey
- input: ${step.0.output} + 기존 논문, RFP
- output: {workdir}/02_기획/2N_{구성기술}_서베이.md
- depends: step.0
- confirm: true
- sub_steps:
  1. 수집 논문 + 기존 관련 논문 식별
  2. `oosurvey run --topic "주제" --paper-dir 03_paper/11_paper_en`
  3. 서베이 문서 구조 검증 (섹션, 비교표, 시사점)
  4. 기술 갭 분석 포함 확인

### step.2 기술연구
- tool: 직접 작성
- input: ${step.1.output}, RFP
- output: {workdir}/03_기술연구/3N_{구성기술}_기술연구.md
- depends: step.1
- confirm: true
- rules: 표준넘버링(N.N), 폴더ID 전수 부여, 참고문헌 테이블 포함
- sub_steps:
  1. 31번 형식 참조하여 문서 구조 생성 (1~11장)
  2. 서베이 내용 기반 기술 동향 비교표 작성 (폴더ID 컬럼 포함)
  3. 파이프라인/아키텍처 설계
  4. 플랫폼/장비 분석 (연산 요건만, 물리 요건은 타 구성기술 범위)
  5. 분류 체계 정리
  6. 정량 목표 설정 (선행연구 수치 기반 근거)
  7. 기술적 과제 및 해결 방안
  8. 연구 로드맵
  9. 참고문헌 전수 검증 (oopaper 폴더 존재 확인)

### step.3 기술수요
- tool: 직접 작성
- input: ${step.2.output}, 40번 양식
- output: {workdir}/04_기술수요작성/4N_{구성기술}_기술수요.md
- depends: step.2
- confirm: true
- rules: 40번 양식 준수(1~2.6), 인용 제거, 기호 자연어화, 코드블록만
- sub_steps:
  1. 40번 양식 구조(제안자, 2.1~2.6) 로드
  2. 기술연구 내용을 양식에 맞춰 변환
  3. 논문 인용 제거 (폴더ID, 저자 연도 등)
  4. 기호 자연어화 (가운데점→쉼표, 화살표→서술, 슬래시→및)
  5. 구체 기술명 → 일반 서술로 변환
  6. 양식 외 섹션 삭제, 기술연구 참조 링크만 추가

### step.4 hwpx 반영
- tool: 직접 작성
- input: ${step.3.output}
- output: {workdir}/04_기술수요작성/4N_hwpx붙여넣기용.txt
- depends: step.3
- confirm: false
- sub_steps:
  1. 기술수요 md에서 코드블록(``` 내부) 내용 추출
  2. 섹션별 구분 헤더 추가 ([2.1], [2.3], [2.4], [2.5], [2.6])
  3. 붙여넣기용 txt 파일 생성
  4. 사용자에게 hwpx 수동 붙여넣기 안내

## naming
- 서베이: 2N_{구성기술}_서베이.md (N은 순번)
- 기술연구: 3N_{구성기술}_기술연구.md
- 기술수요: 4N_{구성기술}_기술수요.md
- N 할당: 기존 파일 최대 번호 + 1

## validation
- step.0: 폴더 존재 + 서머리 파일 존재
- step.1: 서베이 파일 생성 + 섹션 구조(개요, 동향, 시사점, 참고문헌)
- step.2: 표준넘버링 + 폴더ID 전수 + 참고문헌 테이블 + oopaper 폴더 매칭 100%
- step.3: 40번 양식 일치 + 인용 0건 + 기호(가운데점, 화살표, 슬래시) 0건
- step.4: txt 파일 생성 + 섹션 구분 헤더 포함
