## 이슈 ID 체계

| 접두사 | 용도 |
|--------|------|
| T | 일반 이슈 |
| A | 자동화 발견 |
| D | 의존성 |
| S | 보안 |
| W | 경고 (oocheck [WARNING] 이슈) |
| C | 커스텀 Todo (ootodo add 항목) |

| `oofix` | d0004 기반 이슈 수정 |
| `oohistory` | 해결된 이슈 → d0010 아카이브 |

## 템플릿 사용법

### 1. 문서 생성 시

1. 이 템플릿을 복사하여 `00_doc/sp{N}/d{SP}0004_todo.md` 생성
2. `{SP}` 부분을 서브프로젝트 번호로 변경
   - SP=00 (공통): `d0004_todo.md`
   - SP=01 (obsidian): `d10004_todo.md`
   - SP=02 (pycode): `d20004_todo.md`
3. `[프로젝트명]` 부분을 실제 프로젝트/서브프로젝트명으로 변경
4. 작성일 업데이트

### 2. 항목 등록 형식

```markdown
### T001 [카테고리] 내용 요약
등록일: 2026-01-03 | 우선순위: high | 상태: 분석중
```

### 3. 섹션 구조

| 섹션 | 용도 |
|------|------|
| `## 대기 ToDo` | 처리 대기 중인 모든 이슈/Todo |
| `## 완료 ToDo` | 완료된 항목 (ootodo clear로 d0010 아카이브) |
