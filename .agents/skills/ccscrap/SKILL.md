## 3. 통합 처리 워크플로우

### 3.1 URL 자동 감지

`## 다운` 단일 섹션에 모든 URL을 넣으면 패턴으로 자동 분류한다.

| URL 패턴 | 종류 | 처리 | 출력 | 이력 |
|----------|------|------|------|------|
| `youtube.com/shorts/*` | 유튜브 쇼츠 | **스킵** (즉시 삭제, FAIL 없음) | - | - |
| `youtu.be/*`, `youtube.com/watch*` | 유튜브 동영상 | STT (자막→Whisper 폴백) | `01_유튜브서머리/` | `02_유튜브동영상리스트.md` |
| `youtube.com/@*`, `youtube.com/channel/*` | 유튜브 채널 | 채널 등록 + 영상 목록 → `## 다운`에 추가 (채널 URL 즉시 삭제) | - | `13_유튜브채널리스트.md` |
| 그 외 모든 URL | 웹 기사/블로그 | 전문 스크래핑 | `04_읽을거리/` | `04_읽을거리리스트.md` |

### 3.3 실행 프로세스

> 코드 예시: references/guide.md 참조

**처리 순서 규칙**:
1. **Phase 1** (채널 이외): 일반 URL → 유튜브 동영상 → 동영상 다운로드 → 음악 추출
2. **Phase 2** (채널 전개): 채널 URL → 영상 목록 추출 → `## 다운`에 개별 URL 추가 → 채널 URL 즉시 삭제
3. **Phase 2 후 재처리**: 전개된 영상 URL을 다시 Phase 1 Step 5 흐름으로 순차 처리
4. **즉시 삭제**: URL 1건 처리 완료 시마다 01_다운로드.md에서 해당 줄 즉시 제거 (중단 시 재개 가능)
5. **쇼츠 자동 스킵**: `youtube.com/shorts/*` URL은 STT 대상에서 제외하고, `# [FAIL]` 없이 01_다운로드.md에서 즉시 삭제 (쇼츠는 짧아서 서머리 가치 없음)
6. **채널 처리 원칙**: 채널 URL 1개 → 영상 URL N개로 전개 후 개별 처리 (일괄 STT 금지)

## 샘플 참조 (산출물 품질 향상)

> 산출물 작성 직전, `samples/` 폴더가 존재하면 샘플을 few-shot 참고 자료로 활용한다.

| 항목 | 내용 |
|------|------|
| 샘플 위치 | `.agents/skills/{스킬명}/samples/` |
| 참조 시점 | 산출물 작성 직전 (on-demand, 자동 로드 X) |
| 샘플 있는 경우 | 샘플의 스타일·깊이·어조를 참고하여 산출물 작성 |
| 샘플 없는 경우 | 템플릿(`templates/`)만으로 진행 (현재 상태) |
| 샘플 추가 방법 | 품질 좋은 기존 산출물을 `samples/` 폴더에 저장 |
