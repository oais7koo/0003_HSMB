# 테스트 체크리스트 초기화 템플릿

> 중앙 안내: 상세한 에이전트 활용 원칙 및 공통 가이드라인은 `.claude/guides/common_guide.md`의 '2. 에이전트 활용 원칙' 섹션을 참조하십시오.

## 템플릿 설명

이 템플릿은 테스트 사이클 시작 전에 체크리스트를 초기화하는 절차를 정의합니다.
프로젝트별로 체크리스트 항목을 커스터마이즈하여 사용하세요.

## 초기화 절차 템플릿

### 테스트 사이클 시작 절차

**중요**: 매 테스트 사이클 시작 전에 d0003_test.md의 체크리스트를 초기화해야 합니다.

#### 1. 체크리스트 초기화

테스트 시작 전 d0003_test.md의 모든 체크박스를 대기 상태(`[ ]`)로 초기화:

```markdown
# 초기화 전
- [x] 문법 오류 없음
- [!] SQL 인젝션 방지
- [-] 스킵된 항목

# 초기화 후
- [ ] 문법 오류 없음
- [ ] SQL 인젝션 방지
- [ ] 스킵된 항목
```

#### 2. 초기화 방법

1. d0003_test.md 파일 열기
2. 모든 체크박스 상태를 `[ ]`로 변경
3. 변경 사항 커밋 (선택사항)

#### 3. 상태 의미

| 상태 | 의미 | 사용 시기 |
|------|------|----------|
| `[ ]` | 대기/미실행 | 테스트 시작 전 |
| `[x]` | 성공/통과 | 테스트 완료 시 |
| `[!]` | 중요 이슈 | 즉시 조치 필요 |
| `[-]` | 스킵 | 해당 테스트 불필요 시 |

## 커스터마이즈 예시

프로젝트별로 다음 항목들을 체크리스트에 추가:

- 코드 품질 체크
- 보안 취약점 검사
- 성능 테스트
- UI/UX 테스트
- 통합 테스트
- 배포 테스트
