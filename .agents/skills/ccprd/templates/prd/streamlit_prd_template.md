## 4. 시스템 아키텍처

### 4.1 구조

```
┌─────────────────────────────────────────────────────────────┐
│                      Client (Browser)                        │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                   Streamlit Application                      │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │   Pages     │  │    oo     │  │    Session State    │  │
│  │  (UI Layer) │  │  (Modules)  │  │    (Auth/Data)      │  │
│  └─────────────┘  └─────────────┘  └─────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                     Database Layer                           │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 폴더 구조

```
{서브프로젝트}/
├── login.py                          # 로그인 페이지 (진입점)
├── config.py                         # 서버 설정
├── pages/                            # Streamlit 멀티페이지
│   ├── 0_00_메인===.py              # [홈] 메인 대시보드
│   ├── 1_10_서비스1===.py           # [서비스1] 그룹 헤더
│   └── ...
└── __pycache__/
```

> **참고**: `===` 접미사는 서비스 그룹 헤더 페이지를 의미

## 5. 메뉴 구조

> **파일 위치**: `{서브프로젝트}/pages/`

### 5.1 페이지 개요

| ID | 대분류 | 메뉴명 | 설명 | 권한 | 진행 | 기획문서링크 |
|----|--------|--------|------|------|:----:|-------------|
| login | 홈 | login | 로그인 | 전체 | 예 | - |
| 0_00 | 홈 | 메인 | 메인 대시보드 | 전체 | 예 | - |

> **진행 컬럼**: `예` = 개발 진행, `아니오` = 이후 단계에서 제외
> **기획문서링크 컬럼**: 페이지별 상세 기획문서 번호 (예: d2XXXX)

### 5.2 페이지 상세

> 진행=예인 페이지의 상세 기획. Feature/Task 도출 기준.

#### 5.2.1 페이지 요약

| ID | 페이지명 | 설명 | UI 구성요소 | 데이터 소스 |
|----|----------|------|-------------|------------|
| login | 로그인 | 사용자 인증 | form, button | common_user |

#### 5.2.2 페이지별 기능 상세

| 기능 | 설명 | 우선순위 |
