# ccmemo - 임시 메모 스킬

> `00_doc/zzz.md` 파일에 메모를 쓰거나 읽는 단순 임시 메모 스킬.

## 0. 스킬 요약

| 항목 | 내용 |
|------|------|
| **핵심 역할** | `00_doc/zzz.md`에 메모 쓰기/읽기 |
| **하는 것** | `00_doc/zzz.md` 덮어쓰기(w), 읽기(r), 자동 판단(run) |
| **하지 않는 것** | 기존 내용 보존, 이력 관리, 태그/날짜 자동 추가 |
| **파일 위치** | `00_doc/zzz.md` |
| **실행 레벨** | [자동] — 즉시 처리 |
| **⚠️ 필수 규칙** | **쓰기(`w` / 인자없음 / `run`의 쓰기분기) 시 항상 `Read → Write` 순서** — Codex harness 가드(이미 존재하는 파일은 Read 선행 필수) 회피 |

## 1. 서브명령어

| 명령어 | 설명 |
|--------|------|
| `ccmemo` | **직전 대화 저장** — 직전 사용자 요청 + AI 응답을 `00_doc/zzz.md`에 덮어쓰기 |

## 2. 동작 규칙

### `ccmemo` (인자 없음)

```
0. [필수] Read 도구로 00_doc/zzz.md 선행 읽기 (파일이 있든 없든 시도 → harness 가드 충족)
   - FileNotFound 오류는 무시 (파일 없으면 신규 생성으로 진행)
1. 현재 대화 기록에서 "직전 사용자 입력"과 "직전 AI 응답"을 추출
   - 직전 사용자 입력: 이번 ccmemo 호출 바로 앞의 사용자 메시지
   - 직전 AI 응답: 그 사용자 메시지에 대한 AI의 마지막 텍스트 응답 (tool call 제외)
2. Write 도구로 아래 포맷으로 00_doc/zzz.md 덮어쓰기

   # 대화 메모 (YYYY-MM-DD HH:MM)

   ## 사용자 요청
   {직전 사용자 입력 원문}

   ## AI 응답
   {직전 AI 응답 원문}

3. 완료 후 "00_doc/zzz.md에 직전 대화를 저장했습니다." 출력
4. 직전 대화가 없으면 (세션 첫 호출 등): "저장할 대화가 없습니다." 출력
```

**주의**
- 직전 AI 응답은 **텍스트 응답**만 추출 (tool call 인자·결과 제외)
- 원문 그대로 저장 (요약·재작성 금지)
- 민감 정보(비밀번호·API 키 등)가 포함된 경우에도 가감 없이 저장되므로 사용자가 내용 확인 필요

### `ccmemo w [내용]`

```
0. [필수] Read 도구로 00_doc/zzz.md 선행 읽기 (harness 가드 충족)
   - FileNotFound 오류는 무시하고 진행 (파일 없으면 신규 생성 모드)
   - 내용은 분석하지 않음, 덮어쓰기 전제이므로 단순히 Read 호출만 수행
1. Write 도구로 00_doc/zzz.md 파일을 [내용]으로 덮어씀
2. 기존 내용은 완전히 삭제됨 (append 아님)
3. 파일이 없으면 새로 생성
4. 완료 후 "00_doc/zzz.md에 저장했습니다." 출력
```

### `ccmemo r`

```
1. 00_doc/zzz.md 파일 읽기
2. 내용을 그대로 출력
3. 파일이 없으면 "(비어 있음)" 출력
```

### `ccmemo run [지시사항]`

지시사항의 의도를 분석하여 읽기(r) 또는 쓰기(w)를 자동 판단하고 실행한다.

**판단 기준:**

| 의도 | 키워드 예시 | 동작 |
|------|------------|------|
| 읽기 | 읽어, 가져와, 보여줘, 확인해, 뭐야, 뭐가 있어, 조회 | `r` 실행 |
| 쓰기 | 저장해, 써줘, 기록해, 메모해, 바꿔, 업데이트, + 저장할 내용이 있는 경우 | `w` 실행 |

**판단 흐름:**
```
ccmemo run [지시사항]
    ↓
지시사항 분석
    ├─ 읽기 의도 감지 → ccmemo r 실행 (Read 1회)
    └─ 쓰기 의도 감지 (또는 저장할 내용 포함) → ccmemo w [내용] 실행
                                              (반드시 Read → Write 순서)
```

**예시:**
```
ccmemo run zzz에 뭐가 있어?         → r 실행 (읽기)
ccmemo run 내용 보여줘              → r 실행 (읽기)
ccmemo run 내일 배포 예정이라고 저장해 → w "내일 배포 예정" (쓰기)
ccmemo run API 키 확인 필요를 메모해  → w "API 키 확인 필요" (쓰기)
```

## 3. 주의사항

- `ccmemo w`는 **항상 덮어씀** — 기존 내용 복구 불가
- `00_doc/zzz.md`는 임시 메모용 파일로 git 추적 여부는 사용자가 결정
- 긴 내용도 그대로 저장 (포맷 변환 없음)
- `run` 판단이 모호할 경우 쓰기보다 읽기를 우선 (안전한 방향)
- **쓰기 작업 시 항상 Read 선행** — Codex Code의 Write 도구는 기존 파일을 읽지 않은 상태에서 덮어쓰기를 거부한다. `00_doc/zzz.md`는 세션 간 보존되므로 이 가드에 자주 걸린다. 반드시 `Read → Write` 순서로 호출할 것 (FileNotFound는 무시)
