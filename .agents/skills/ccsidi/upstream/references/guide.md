## 1. 개요

### 1.1 목적
옵시디언 볼트(01_obsidian/)의 마크다운 문서를 체계적으로 생성·관리·검색하는 스킬

### 1.2 주요 기능
- 볼트 상태 리포트 (폴더별 문서 수, 전체 현황)
- 문서 생성 (폴더 지정 + 제목)
- 인덱스 파일 자동 갱신 ([[위키링크]] 기반)
- 키워드 검색 (볼트 전체)
- 폴더별 문서 목록 조회

### 1.3 실행 시점
- 옵시디언 볼트에 새 문서를 작성할 때
- 볼트 구조를 파악하고 싶을 때
- 특정 키워드가 포함된 노트를 찾을 때
- 인덱스 파일을 최신 상태로 갱신할 때

## 2. 워크플로우

### 2.1 문서 생성 흐름

```
oosidi create <폴더> "<제목>"
    ↓
대상 폴더 확인 (존재 여부)
    ↓
동일 제목 문서 중복 검사
    ↓
마크다운 파일 생성 (## 개요 헤더 포함)
    ↓
인덱스 파일에 [[위키링크]] 추가
    ↓
완료 리포트
```

### 2.2 볼트 상태 점검 흐름

```
oosidi run
    ↓
01_obsidian/ 전체 스캔
    ↓
폴더별 문서 수 집계
    ↓
깨진 위키링크 탐지
    ↓
빈 문서(헤더만 있는) 목록
    ↓
상태 리포트 출력
```

## 3. 볼트 표준 폴더 구조

> 0003_vibe 폴더를 기준으로 파악한 표준 패턴

### 3.1 최상위 폴더 구조

```
{NNNN}_{폴더명}/
├── {NNNN}_{폴더명}.md          ← 루트 인덱스 (필수)
├── 01_개요/                    ← 개요 5종 세트
│   ├── 01_{주제} 개요.md
│   ├── 02_{주제} 환경.md
│   ├── 03_{주제} 사용.md
│   ├── 04_{주제} APIs.md
│   └── 05_{주제} 채널.md
├── 02_개념/                    ← 개념/참조 문서
├── {NN}_{하위주제}/            ← 번호 기반 서브폴더
└── 스터디/                     ← 한글 폴더명 허용
```

### 3.2 폴더 배치 규칙 (필수)

> **핵심 규칙**: 01~05 번호 파일(개요 5종 세트)은 반드시 `01_개요/` 폴더 안에 위치해야 한다. 루트 레벨에 직접 두지 않는다.

| 배치 | 예시 | 판정 |
|------|------|:----:|
| `reflex/01_개요/01_reflex 개요.md` | 01_개요 폴더 안 | **O (정상)** |
| `gemini/01_gemini 개요.md` | 루트에 직접 배치 | **X (비표준)** |

**올바른 구조:**
```
{주제}/
├── {주제}.md                   ← 루트 인덱스만 여기에
├── 01_개요/                    ← 개요 5종은 반드시 이 안에
│   ├── 01_{주제} 개요.md
│   ├── 02_{주제} 환경.md
│   ├── 03_{주제} 사용.md
│   ├── 04_{주제} APIs.md
│   └── 05_{주제} 채널.md
├── 02_개념/                    ← 개념 문서
└── ...
```

**잘못된 구조 (비표준):**
```
{주제}/
├── {주제}.md
├── 01_{주제} 개요.md           ← X: 루트에 직접
├── 02_{주제} 환경.md           ← X: 01_개요/ 안에 있어야 함
├── 03_{주제} 사용.md
└── ...
```

> `oosidi check <NNNN>` 실행 시 이 규칙 위반을 감지하여 리포팅한다.

### 3.3 개요 5종 세트 (번호 고정)

모든 주제 폴더에서 반복되는 고정 패턴:

| 번호 | 카테고리 | 내용 |
|------|---------|------|
| 01 | 개요 | 정의, 소개, 핵심 개념 |
| 02 | 환경 | 설치, 설정, 환경 구성 |
| 03 | 사용 | 사용법, 실습, 예제 |
| 04 | APIs | API 레퍼런스, 인터페이스 |
| 05 | 채널 | 참고 사이트, 학습 채널, 커뮤니티 |

### 3.4 하위 주제 폴더 (동일 패턴 반복)

```
{하위주제}/
├── {하위주제}.md               ← 서브 인덱스
├── 01_{하위주제} 개요.md
├── 02_{하위주제} 환경.md
├── 03_{하위주제} 사용.md
├── 04_{하위주제} APIs.md
└── 05_{하위주제} 채널.md
```

### 3.5 파일명 규칙

| 유형 | 패턴 | 예시 |
|------|------|------|
| 루트 인덱스 | `{번호}_{이름}.md` | `0003_vibe.md` |
| 서브 인덱스 | `{주제}.md` | `opencode.md` |
| 개요 시리즈 | `{NN}_{주제} {카테고리}.md` | `01_vibe 개요.md` |
| 스터디 | `YYMMDD-HHMM_{제목}.md` | `251121-1139_제목.md` |
| 일반 문서 | `{제목}.md` | 자유 형식 |

### 3.6 init 명령으로 자동 생성

```bash
# 최상위 폴더 생성 (인덱스 + 01_개요/ + 5종 세트)
oosidi init-folder 0025 "NewTopic"

# 기존 폴더 안에 하위 주제 추가 (서브폴더 + 5종 세트)
oosidi init-topic 0003_vibe "newlib"
```

### 3.7 라이브러리 문서 ↔ 테스트 코드 연결

라이브러리 폴더(`09_파이썬라이브러리/`)의 문서는 `02_pycode/ps90/`의 테스트 코드와 연결된다.

#### 경로 매핑

| 항목 | 위치 |
|------|------|
| 라이브러리 문서 | `01_obsidian/0002_codes/0000_python/09_파이썬라이브러리/{알파벳}/{라이브러리}/` |
| 테스트 코드 | `02_pycode/ps90/` |

#### 테스트 코드 파일명 규칙

```
ps9000_{라이브러리명}_{2자리숫자}_{테스트내용}.py
```

예시:
```
ps9000_dash_01_콜백기본.py
ps9000_dash_02_부트스트랩레이아웃.py
ps9000_streamlit_01_기본앱.py
```

#### 인덱스 파일 개발 섹션 연결

라이브러리 인덱스(`{라이브러리}.md`)의 **개발** 섹션에 테스트 코드를 [[위키링크]]로 목록화한다.

```markdown
## 개발
- [[ps9000_dash_01_콜백기본]]
- [[ps9000_dash_02_부트스트랩레이아웃]]
```

## 4. 문서 작성 규칙

### 4.1 문서 구조
- `## 개요` 또는 `## {카테고리}` 헤더로 시작
- 관련 문서는 `[[위키링크]]`로 연결
- 인덱스 파일에 하위 문서 [[링크]] 유지

### 4.2 페이지 정리 표준 포맷

`oosidi 정리` 요청 시 아래 포맷을 기준으로 정리한다.

**구조 규칙:**
1. `## 개요` — 한 줄 요약 설명으로 시작
2. `## 섹션` — 주요 주제별 H2 헤딩
3. `### N단계:` — 순차 절차는 번호 + 헤딩 (`### 1단계: 제목`)
4. 순서 있는 항목은 번호 리스트 (`1. 2. 3.`)
5. 순서 없는 항목은 불릿 리스트 (`- `)

**서식 규칙:**
- 메뉴 경로: 인라인 코드 (`` `Settings → SDK` ``)
- 핵심 용어: **볼드** (`**AVD**`)
- 명령어/코드: ` ```bash ` 코드 블록 (언어 지정 필수)
- 불필요한 공백 라인 제거, 들여쓰기 통일

**참조 예시:** `7934_android studio/02_android studio 환경.md`

```markdown
## 개요
한 줄 요약 설명.

## 설치 (Windows)
### 1단계: 제목
1. 순서 항목
2. **핵심 용어** 강조

### 2단계: 제목
`메뉴 → 경로 → 설정`

**하위 그룹:**
- 항목 A
- 항목 B

### 3단계: 제목
\`\`\`bash
명령어
\`\`\`
```

## 5. FAQ

**Q: 폴더명을 정확히 모를 때?**
A: 부분 매칭을 지원합니다. `oosidi list design` → `0004_DesignSystem` 자동 매칭

**Q: 기존 oodoc과 차이는?**
A: oodoc은 00_doc/ 프로젝트 문서 관리, oosidi는 01_obsidian/ 볼트 노트 관리
