## 2. 프로젝트 개요

### 2.1 목적

고속 이동 터널 스캐닝 시스템(MTSS) 환경에서 발생하는 모션블러를 정량 평가하기 위한 **No-Reference IQA(NR-IQA) 메트릭 HSMB를 개선**하고, 이를 SOTA 수준의 학술 논문으로 완성해 투고한다.

### 2.2 배경

| 항목 | 내용 |
|------|------|
| v1 논문 현황 | Reviewer 1 지적으로 Reject (2026-02) |
| 해결된 지적 | C1 (카메라 메커니즘), C2 (통계 조건 수) |
| 미해결 지적 | C3 (CNN crack detection downstream), C4 (파라미터 이론적 근거) |
| v1 알고리즘 | AEP + Weibull CDF 기반 edge-width estimation (block-size 64, EdgeWeight=1.5, JNB=3, β=2) |
| v1 최고 성과 | BEW 상관 PLCC=0.9354 (비교 메트릭 최고), NR-IQA 최고 모션블러 특화 |

### 2.3 전략 원칙

> **핵심 제약**: HSMB v1 알고리즘은 현 재투고 논문에서 **동결(freeze)**한다.
> 알고리즘 변경 시 논문 내 모든 수치(PLCC, SROCC, 테이블, Figure)를 전면 재실험해야 하므로,
> 재투고 완료 후 별도 후속 논문에서 v2 개선을 추진한다.

| 트랙 | 범위 | 우선순위 |
|------|------|---------|
| **Track 1 (현재)** | v1 알고리즘 동결 + Reviewer 대응 + 신규 실험 + 재투고 | **최우선** |
| **Track 2 (후속)** | HSMB v2 알고리즘 개선 + 별도 논문 | 재투고 완료 후 착수 |

### 2.4 범위

**In-Scope (Track 1):**
- HSMB v1 알고리즘 **그대로 유지**
- C3 대응: CNN crack detection downstream 실험 지원·분석
- C4 대응: v1 파라미터(EdgeWeight·JNB·β) 이론적 근거 확립 (ablation study)
- 신규 실험 E1-1단계: 터널표준영상(`data/ps1204_kict_eSFR/`) → NR-IQA 7종 스코어 산출
- 신규 실험 E1-2단계: 터널표준영상 → BEW-H, BEW-V 측정 → NR-IQA 상관성 분석 (PLCC/SROCC)
- 신규 실험 E2: HSMB-크랙검출 상관성 분석
- 관련 분야 선행연구 서베이
- 현장 50조건 실험 데이터 활용 (area-scan, 491장)
- SOTA 논문 작성 및 재투고

**Out-of-Scope (Track 1):**
- HSMB v2 알고리즘 재설계·구현 → Track 2로 이관
- 목표 저널/학회 선정 — 향후 결정
- 제출 일정 — 향후 결정
- 추가 평가 데이터셋 — 향후 결정
- Task A·B 직접 실험 실행 — 협업자 담당

## 2.6 데이터 특이케이스 (중요)

### ISO 400 — 의도적 Defocus 조건

| 항목 | 내용 |
|------|------|
| **설계 의도** | ISO 400 조건은 defocus(초점 이탈)가 발생하도록 **의도적으로 설계**한 실험 조건 |
| **BEW 특성** | BEW 절대값 높음 (모션블러 + defocus 복합 → 블러 심함) |
| **이방성 특성** | BEW_H − BEW_V **값이 작음** — defocus는 isotropic(등방성)이므로 H/V 축 차이가 크지 않음 |
| **논문 서술 수정** | ~~"autofocus 수렴 실패로 추정"~~ → **"의도적 defocus 조건 (complex-blur stress test)"** |
| **해석 의미** | 이방성 작음 = 순수 모션블러 아님을 확인, defocus + motion 복합 블러임을 이론적으로 정합 |
| **영향 범위** | `section06_draft.md` §VI-C complex-blur stress test 서술 전면 수정 필요 |

> **⚠️ 주의**: `00_doc/260422_HSMB논문 수정안/docs/section06_draft.md` 및 `d0800_수정안의견.md`의
> "autofocus 수렴 실패 **추정**" 표현을 "의도적 defocus 설계 조건"으로 교체해야 함.

## 4. 알고리즘 요구사항

### 4.1 HSMB v1 구조 (기준선)

```
입력 이미지
  → Sobel gradient (grad_x, grad_y, grad_mag)
  → 64×64 블록 분할
  → AEP 선택: grad_mag > mean_grad × EdgeWeight(1.5)
  → 방향별 edge profile 추출 (4방향)
  → Edge width 추정 (EVP 기반 sub-pixel 보간)
  → Blur probability: P_blur = 1 - exp(-(w/JNB)^β)
  → P_blur 히스토그램 → CDF
  → HSMB = CDF at p_jnb(0.63)
출력: float [0,1] — 높을수록 선명
```

### 4.2 v2 개선 검토 방향 (탐색 단계, 미확정)

| 레이어 | v1 현황 | v2 검토 옵션 |
|--------|---------|------------|
| 엣지 감지 | Sobel 3×3 | Canny, LoG, 구조 텐서 |
| 방향 처리 | 4방향 균일 | 주 블러 방향 적응형 (H/V 분리) |
| 블록 분할 | 고정 64×64 | 적응형 블록 (콘텐츠 복잡도 기반) |
| AEP 임계 | EdgeWeight × mean_grad | 통계적 임계 (Otsu, 퍼센타일) |
| Blur prob | Weibull CDF | 대안 분포 검토 (logistic, 경험적) |
| 집계 | 전역 히스토그램 CDF | 공간 가중 집계 |
| 파라미터 | EdgeWeight=1.5, JNB=3, β=2 | 이론 유도 + 최적화 |

### 4.3 성능 목표 (v2)

| 메트릭 | v1 기준 | v2 목표 |
|--------|---------|--------|
| BEW vs HSMB PLCC (50조건) | 0.9354 (실험실) | ≥ 0.95 |
| BEW vs HSMB SROCC | 0.8855 | ≥ 0.90 |
| 처리 속도 | 기준선 측정 필요 | v1 대비 ≤ 2× 증가 |
| Complex-blur 탐지 | stress test 대상 | 모노토닉 응답 |
