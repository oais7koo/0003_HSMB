## 3. 기능 개요

PRD v06 §2.7.2 E3-4 — **ps1010 + ps1301 이미지를 단일 데이터셋(통합 풀)으로 합쳐** d0100 표준 7종 NR-IQA(9컬럼)를 산출하고, 혼합 데이터에서의 IQA 분포를 파악한다. E3 실험군의 마지막 작업이다.

### 3.1 핵심 설계 — 재산출 불필요

NR-IQA는 **이미지별 독립 계산**이다 (한 이미지의 IQA 스코어는 다른 이미지와 무관). 따라서 통합 풀의 IQA 결과는 **E3-1 결과(ps1010) + E3-2 결과(ps1301)를 단일 테이블로 concat한 것과 완전히 동일**하다.

→ E3-4는 ps1010·ps1301 이미지를 물리적으로 다시 IQA 산출하지 **않고**, E3-1·E3-2 산출물을 통합하여 `source_ps` 출처 컬럼을 부여하는 방식으로 구성한다. (ps1301 737장 × ~55MB 재처리를 회피 — 결과 동일, 비용 절감)

> 단, E3-1·E3-2가 **동일 common_iqa7 버전·동일 device**로 산출됐음을 전제로 한다. 버전이 다르면 재산출 필요.

> **⚠️ crop 기준 혼재 (v04)**: E3-2(ps1301)는 v04부터 **가로·세로 가운데 50%(면적 25%) center crop** 후 IQA를 산출한다. 반면 E3-1(ps1010)은 **이미지 전체**로 산출한다. 통합 풀은 두 결과를 그대로 concat하므로, `source_ps=ps1010`은 전체 이미지 / `source_ps=ps1301`은 50% crop 기준의 IQA가 **혼재**한다. source별 분포 대조(R05)·d0046 리포트 해석 시 이 crop 기준 차이를 반드시 고려한다. ps3040 스크립트 자체는 수정하지 않는다(concat 방식 불변).

| 항목 | 타입 | 설명 |

| `data/ps3040/02_source별통계_{dsr}.xlsx` | XLSX | source별(ps1010/ps1301/통합) mean/std/min/max |
| `data/ps3040/04_histogram_{dsr}.png` | PNG | 9종 메트릭 히스토그램 (통합 + source별 오버레이) |
| §11 실험 결과 (본 문서 통합) | MD | 통합 풀 분포 분석 + 개별 데이터셋 대조 리포트 |

### 5.3 출력 스키마 (`01_통합풀데이터_{dsr}.xlsx`)

```
source_ps, condition, distance_m, speed_kmh, iso, frame, is_defocus,
hsmb, cpbd, niqe, niqe_matlab, piqe, brisque, brisque_matlab, dbcnn, arniqa
```

- `source_ps`: `ps1010` 또는 `ps1301` — 출처 데이터셋
- `condition`: 원본 조건 폴더명 (ps1010 `low_*` / ps1301 `crack_*`)
- 통합 풀 규모: ps1010 520행 + ps1301 737행 = **1,257행**
- 공통 규칙: d0100 §6.4 — UTF-8, 소수점 4자리(`.4f`), 결측 `NaN`
