## 3. 기능 개요

실제 터널 크랙 이미지 데이터셋(`ps1301_real_crack_cam2`, 50조건 frame 737장)에 대해 **d0100 표준 7종 NR-IQA**(9컬럼)를 `common_iqa7.py`로 일괄 산출한다. PRD v06 §2.7 E3 실험군의 두 번째 작업으로, 실제 크랙 데이터의 IQA 분포를 파악한다.

> **center crop 적용 (v04)**: ps1301 frame은 ~55MB(8K)의 대용량이므로, 이미지 전체가 아닌 **가로·세로 가운데 50%(중앙 영역, 면적 25%)** 만 사용하여 IQA를 산출한다. `common_iqa7.compute_all_from_path(center_crop=True)`로 적용한다.

E3-1(d3010, ps1010)과 동일한 IQA 산출 파이프라인을 사용하되, **A안**으로 진행한다 — IQA 9컬럼 산출까지만 수행하고, 후속 상관 분석은 별도 작업(E2 — HSMB×크랙검출 F1, F002-3)에서 다룬다.

> **§9 SFR 상관 미적용 사유**: ps1301은 실제 크랙 이미지로 eSFR/MTF 차트가 아니다. `Results/SFR_cypx.csv`·R1090·MTF50 측정값이 존재하지 않으므로, d3010 §9 SFR↔IQA 상관 분석은 E3-2에 적용하지 않는다.

### 5.1 입력

| 항목 | 타입 | 설명 | 필수 |
|------|------|------|------|

| `src/common_iqa7.py` | Python | 7종 IQA 정식 산출 라이브러리 (`compute_all_from_path()`) | Y |

### 5.2 출력

| 항목 | 타입 | 설명 |
|------|------|------|
| `src/ps3020_e3_2_ps1301_iqa.py` | Python | ps1301 50조건 일괄 실행 진입점 (common_iqa7 import) |
| `data/ps3020/01_전체데이터_{dsr}.xlsx` | XLSX | 이미지별 9컬럼 스코어 + 조건 메타 (737행) |
| `data/ps3020/02_그룹별통계_{dsr}.xlsx` | XLSX | 조건별 mean/std/min/max (50조건) |
| `data/ps3020/03_그룹별평균_{dsr}.xlsx` | XLSX | 조건별 평균 + best_group 행 |
| `data/ps3020/04_histogram_{dsr}.png` | PNG | 9종 메트릭 히스토그램 |
| §11 실험 결과 (본 문서 통합) | MD | 결과 요약 + 조건별 분포 표 + 히스토그램 |

### 5.3 출력 스키마 (`data/ps3020/01_전체데이터_{dsr}.xlsx`)

```
condition, distance_m, iso, speed_kmh, frame, is_defocus,
hsmb, cpbd, niqe, niqe_matlab, piqe, brisque, brisque_matlab, dbcnn, arniqa
```

- `condition`: 조건 폴더명 (예: `crack_d45_ISO400_V80`)
- `distance_m`: `d45` → `4.5` (× 0.1 변환)
- `is_defocus`: ISO=400 조건이면 1, 아니면 0
- IQA 9컬럼: common_iqa7 `METRIC_COLUMNS`
- 공통 규칙: d0100 §6.4 — UTF-8, 소수점 4자리(`.4f`), 결측 `NaN`

## 6. 제약조건 / 예외처리

| 상황 | 처리 방식 |
|------|----------|
| **이미지 대용량 (~55MB/장, 8K)** | 가로·세로 가운데 50% center crop으로 처리 영역을 원본의 25%로 축소 (R08). 그래도 GPU OOM 시 조건 단위 분할 + `torch.cuda.empty_cache()`, CPU 실행 시 처리 시간 과다 감안 |
| GPU OOM (50조건 일괄) | 조건 단위 분할 + 매 조건 후 `torch.cuda.empty_cache()` |
| 폴더명 파싱 실패 | 경고 로그 + 해당 조건 스킵 (수동 검토용 리스트) |
| 일부 NR-IQA 모델 로드 실패 | common_iqa7 `compute_all()`이 해당 컬럼만 `NaN` 처리, 나머지 진행 |
| ISO=400 조건 (의도적 defocus) | `is_defocus` 플래그 컬럼 표시 (PRD §2.6, F003-2 연계) |
| common_iqa7 의존성 미설치 (pyiqa/cpbd) | GPU 환경 `uv sync` + `patch_cpbd.py` 선결 (PRD §7.1) |

- **BRISQUE**: ps1301(19.90) > ps1302(17.96) > ps1010(10.38) — 실제 현장 크랙 이미지가 인쇄 크랙보다 블록 아티팩트/잡음 더 많음
