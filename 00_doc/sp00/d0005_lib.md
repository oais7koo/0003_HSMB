## 2. common_iqa7.py — 7종 NR-IQA 산출 라이브러리

### 2.1 개요

| 항목 | 내용 |
|------|------|
| 경로 | `src/common_iqa7.py` |
| 버전 | v01 (2026-05-15) |
| 역할 | d0100 §4.2 표준 7종 NR-IQA를 정식 알고리즘·라이브러리로 산출 |
| 출력 | **9컬럼** (7종 — NIQE·BRISQUE는 기본/MATLAB 2변형) |
| 사용처 | E3 실험군 (ps3010·ps3020·ps3030·ps3040 진입 스크립트) |

### 2.2 7종 메트릭 구현 매핑

| # | 메트릭 | 컬럼 | 구현 | 방향 |
|---|--------|------|------|------|
| 1 | HSMB | `hsmb` | 자체 구현 `HSMBCalculator` (ps5010 v1 — Sobel + mean 임계 + Weibull CDF) | 높을수록 선명 |
| 2 | CPBD | `cpbd` | PyPI `cpbd` (Narvekar & Karam 2011) | 높을수록 선명 |
| 3 | NIQE | `niqe` / `niqe_matlab` | pyiqa `niqe`, `niqe_matlab` | 낮을수록 선명 |
| 4 | PIQE | `piqe` | pyiqa `piqe` | 낮을수록 선명 |
| 5 | BRISQUE | `brisque` / `brisque_matlab` | pyiqa `brisque`, `brisque_matlab` | 낮을수록 선명 |
| 6 | DBCNN | `dbcnn` | pyiqa `dbcnn` (KonIQ10k pretrained) | 높을수록 선명 |
| 7 | ARNIQA | `arniqa` | pyiqa `arniqa` (pretrained) | 높을수록 선명 |

> HSMB 파라미터 (PRD §4.1 동결 사양): block_size=64, edge_weight=1.5, beta=2.0, JNB=3, p_jnb=0.63(CDF idx 63), threshold_ratio=0.1.

### 2.3 주요 함수 / 클래스

| 항목 | 역할 |
|------|------|
| `HSMBCalculator` | HSMB v1 계산 클래스 (ps5010 v1 정식) |
| `compute_hsmb_v1(img)` | HSMB 단일 산출 |
| `compute_cpbd(img)` | CPBD 단일 산출 |
| `compute_niqe/piqe/brisque/dbcnn/arniqa(img)` | pyiqa 메트릭 단일 산출 |
| `compute_all(img) -> dict` | 9컬럼 일괄 산출 (메트릭 실패 시 해당 컬럼만 NaN) |
| `compute_all_from_path(path) -> dict` | 경로 입력 버전 (한글 경로 대응) |
| `METRIC_COLUMNS` | 9컬럼 컬럼명 리스트 |
| `METRIC_DIRECTION` | 메트릭별 방향(↑/↓) 매핑 |

### 2.4 의존성

| 패키지 | 용도 | 필수 |
|--------|------|------|
| `opencv-python-headless`, `numpy` | HSMB·이미지 처리 | 필수 |
| `torch`, `pyiqa` | NIQE·PIQE·BRISQUE·DBCNN·ARNIQA | optional (미설치 시 해당 5종 NaN) |
| `cpbd` (PyPI) | CPBD | optional + **`scripts/patch_cpbd.py` 필요** |

> ⚠️ `cpbd` 패키지는 `scipy.ndimage.imread`(scipy 1.8+ 제거) 의존 → `uv sync`/`uv add` 후 **반드시** `uv run python scripts/patch_cpbd.py` 재실행. 절차: PRD §7.1.

## 3. ps2000 간이판 vs common_iqa7 정식판 — 코드 수준 비교

> `ps2000_통합IQA.py`(v04)는 CPBD·NIQE·PIQE·BRISQUE 4종이 **간이 구현**(원논문 알고리즘과 무관한 휴리스틱)이다. `common_iqa7.py`는 4종을 정식 라이브러리로 교체했다. HSMB도 ps2000은 v2 계통 변종, common_iqa7은 PRD §4.1 동결 사양(ps5010 v1)으로 다르다.

### 3.1 CPBD

| 구분 | 코드 | 산출물 |
|------|------|--------|
| ps2000 (`calculate_cpbd`, L233-236) | `edges = cv2.Canny(gray,50,150)` → `sum(edges>0)/(h*w)` | Canny **edge 픽셀 비율** (0~1) |
| common_iqa7 (`compute_cpbd`) | `cpbd.compute(gray)` (PyPI) | **정식 CPBD** — marziliano edge width + JNB(contrast별 5/3px) + `1-exp(-(w/JNB)^β)` 누적확률 |

→ ps2000은 JNB·blur 확률 누적이 전혀 없음. edge density는 CPBD(Cumulative Probability of Blur Detection)와 **다른 물리량**.

### 3.2 NIQE

| 구분 | 코드 | 산출물 |
|------|------|--------|
| ps2000 (`calculate_niqe_simple`, L325-330) | `mu=GaussianBlur(gray)`, `sigma=sqrt(GaussianBlur((gray-mu)²))` → `dist=sqrt((mean(mu)-127.5)²+(mean(sigma)-50)²)` | 임의 기준점 (127.5, 50.0)까지의 **유클리드 거리** |
| common_iqa7 (`compute_niqe`) | `pyiqa.create_metric("niqe"\|"niqe_matlab")` | **정식 NIQE** — NSS(Natural Scene Statistics) + MVG(다변량 가우시안) 모델 거리 |

→ ps2000은 NSS 통계 모델 자체가 없음. 밝기·표준편차의 임의 기준점 거리일 뿐.

### 3.3 PIQE

| 구분 | 코드 | 산출물 |
|------|------|--------|
| ps2000 (`calculate_piqe_simple`, L333-345) | 16×16 블록 중 `var>100` 블록 → Sobel gradient 평균 → `100-mean(grad)` | 활성 블록 **gradient 부족량 평균** |
| common_iqa7 (`compute_piqe`) | `pyiqa.create_metric("piqe")` | **정식 PIQE** — 블록별 distortion(blocking/noise) 분류 + spatial activity 가중 |

→ ps2000은 blocking·noise distortion 분류가 없음. gradient 평균만.

### 3.4 BRISQUE

| 구분 | 코드 | 산출물 |
|------|------|--------|
| ps2000 (`calculate_brisque`, L316-322) | `sharpness=100-lap_var/10`, `contrast=100-std`, `noise=absdiff(gray,blur)·10` → `sharpness×0.5+contrast×0.3+noise×0.2` | 3개 휴리스틱 **가중합** |
| common_iqa7 (`compute_brisque`) | `pyiqa.create_metric("brisque"\|"brisque_matlab")` | **정식 BRISQUE** — MSCN 계수 기반 NSS feature(36차) + SVR 회귀 |

→ ps2000은 MSCN·NSS feature·SVR이 전혀 없음. Laplacian·표준편차·blur diff의 임의 가중합.

### 3.5 HSMB (둘 다 자체 구현 — 계통 차이)

| 구분 | 코드 핵심 | 비고 |
|------|----------|------|
| ps2000 (`_HSMBCalculator`) | Scharr gradient + CLAHE 전처리 + edge width + Weibull CDF + **Lap/FFT 융합 보정** (`lap_score + (sharpness-0.5)·0.1 + (fft_score-0.5)·0.05`, `**0.25` 보정) | v7310 v2 계통 잔존 — 순수 v1 아님 |
| common_iqa7 (`HSMBCalculator`) | Sobel gradient + mean×edge_weight 임계 + edge width + **순수 Weibull CDF** (`cumsum(pdf)[63]`) | ps5010 v1 = PRD §4.1 동결 사양 |

→ ps2000 HSMB는 Lap+FFT 융합·CLAHE·CONTRAST_EXP 보정이 섞인 변종. common_iqa7은 보정 없는 순수 v1 (d0900 검증).

### 3.6 DBCNN / ARNIQA (동일)

ps2000·common_iqa7 둘 다 `pyiqa` pretrained 모델 사용 — 구현 동일.

### 3.7 결론

| 메트릭 | ps2000 | common_iqa7 | E3 실험 적합성 |
|--------|--------|--------|:--------------:|
| CPBD·NIQE·PIQE·BRISQUE | 간이 휴리스틱 (원논문 무관) | 정식 라이브러리 | common_iqa7만 적합 |
| HSMB | v2 계통 변종 (Lap/FFT 융합) | ps5010 v1 순수 (PRD 사양) | common_iqa7만 적합 |
| DBCNN·ARNIQA | pyiqa pretrained | pyiqa pretrained | 동일 |

→ ps2000 간이판 4종은 논문 비교 메트릭으로 부적합. **E3 실험(ps3010~ps3040)은 common_iqa7 정식판을 사용**하므로 산출 결과가 정식 IQA로 신뢰 가능.
