## 3. 설계 의도

ps2000의 HSMB 파라미터가 PRD 사양(ps5010 v1)과 불일치하는 상황에서,
다른 IQA 지표(NR-23, DL-2, FR-6)는 ps2000 값을 그대로 재사용하면서
**hsmb 컬럼만 d0900 검증 확정 파라미터(ps5010 v1)로 재계산**하기 위해 작성됨.

## 4. HSMB v1 파라미터 (확정)

| 파라미터 | 값 | 출처 |
|---------|-----|------|
| block_size | 64 | ps5010 v1 |

> `# ps5010_HSMB_v1.py 256~266줄 4번째 호출 — d0900 검증 확정 파라미터`

## 5. 처리 흐름

```
1. ps2000 결과 로드  ← data/ps2000/01_전체데이터_*.xlsx (최신 파일)
2. 작업 목록 구성   ← df의 sub_dir/fnm으로 이미지 경로 매핑
3. 병렬 HSMB v1 계산 ← _HSMBv1Calculator (ps5010 v1 인라인)
4. hsmb 컬럼 교체   ← df01["hsmb"] 갱신, 나머지 컬럼 유지
5. 그룹별 통계/평균 재계산
6. 저장 → data/ps2010/
```

---

## 6. _HSMBv1Calculator 클래스

ps5010_HSMB_v1.py를 인라인으로 구현한 클래스.

| 메서드 | 역할 |
|--------|------|
| `_gradient_magnitude(blk)` | Sobel gradient magnitude 계산 |
| `_edge_pixels(mag)` | AEP 선택: `mean(mag) × edge_weight` 임계 |
| `_edge_dir(blk, r, c)` | 4방향 중 인접 픽셀과 최대 강도 차 방향 |
| `_edge_profile(blk, r, c, dr, dc)` | 방향 따라 10픽셀 프로파일 추출 |
| `_edge_width(profile)` | threshold_ratio 기반 edge width 추정 (서브픽셀 보간) |
| `calculate_from_path(img_path)` | 이미지 경로 → HSMB 스코어 (float) |

출력: `cdf[63]` (CDF_INDEX=63)

## 9. ps2000 vs ps2010 HSMB 비교

| 항목 | ps2000 (수정 전) | ps2010 (ps5010 v1) |
|------|----------------|-------------------|
| threshold_ratio | 0.3 | 0.1 |
| beta | 1.8 | 2.0 |
| edge_weight | 0.8 | 1.5 |
| JNB | 2.5 | 3.0 |
| p_jnb | cumsum[45] | cumsum[63] |
| Lap+FFT 융합 | ✅ 포함 | ❌ 순수 Weibull |
| PLCC vs BEW | -0.7630 계통 | **-0.9532** |

> ps2000은 2026-05-15에 PRD 사양으로 파라미터 수정 완료. ps2010은 수정 전 ps2000 결과물 대상 재계산용.
