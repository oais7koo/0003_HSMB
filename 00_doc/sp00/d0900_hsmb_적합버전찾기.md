# d0900 - HSMB 적합 버전 찾기

## 미션
`data/ps1600_hsmb  기존값비교/` 폴더 안의 결과 파일이 **HSMB 어느 버전으로 산출된 값인지**를 식별한다.
- HSMB 버전에 따라 결과값이 달라지므로, 각 결과 파일의 출처(버전) 매핑이 미션의 핵심.
- 매핑이 끝나면 본 프로젝트(블러 정량화 + 크랙 검출)에 가장 적합한 버전을 선정한다.

## 입력 데이터 (확정)
경로: **`data/ps1204_kict_eSFR/`** ✅ 검증 완료

- 전체 47개 폴더 (15000lx + 40000lx 조건)
- xlsx는 그 중 **15000lx 20개 폴더**만 평가 대상으로 사용 (4 노출시간 × 5 속도)
- 폴더 내부: `01.png ~ 10.png` 등 PNG 파일 (KICT eSFR 차트)

| xlsx 표기 | ps1204 폴더 | 매칭 |
|----------|-------------|------|
| `15000lx-500us-XXkm/h` | `15000lx_0640_500us_XX` | ✅ |
| `15000lx-250us-XXkm/h` | `15000lx_1250_250us_XX` | ✅ |
| `15000lx-100us-XXkm/h` | `15000lx_1600_100us_XX` | ✅ |
| `15000lx-50us-XXkm/h`  | `15000lx_1600_050us_XX` | ✅ |

> 4자리 숫자(`0640`, `1250`, `1600`)는 ISO 게인 또는 입력 picture-style 코드로 추정.

## 대상 결과 데이터
경로: `data/ps1600_hsmb  기존값비교/`

| --------------------------- | --- | --- | ----------------------------- | ----- |
| MTF(15000)-IQA 지표.xlsx      | 20  | 13  | (구) v1 계열 추정                  | 미확인   |
| MTF(15000)-IQA 지표_0715.xlsx | 20  | 13  | 0715 시점(v2 또는 final_fixed) 추정 | 미확인   |

### 데이터 구조
- 공통 컬럼 (13): `Test`, `BEW(px)`, `MTF50(cy/px)`, `PSNR`, `SSIM`, `BRISQUE`, `NIQE`, `PIQE`, `CPBD`, **`HSMB`**, `DBCNN`, `ARNIQA`
- 테스트 케이스 (20): `15000lx-{500us|250us|100us|50us}-{00|10|30|50|70}km/h` 조합

### 핵심 발견 ⭐
- **다른 모든 IQA 지표(BEW, MTF50, BRISQUE, NIQE, PIQE, CPBD, DBCNN, ARNIQA)는 두 파일이 완전히 일치**
- **오직 HSMB 컬럼만 값이 다름** → 동일 입력 이미지셋에 대해 **HSMB 알고리즘 버전만 다르게 적용된 결과**

### HSMB 값 분포 비교
| 항목 | 파일1 (구버전 추정) | 파일2 (`_0715`, 신버전 추정) |
|------|---------------------|------------------------------|
| 최솟값 | 0.6285 | 0.5913 |
| 최댓값 | 0.9101 | 0.6789 |
| 범위(span) | 0.2816 | 0.0876 |
| 평균(추정) | ~0.84 | ~0.65 |

**해석**: 파일1은 sharp 이미지에서 0.89~0.91까지 높게 평가하는 반면, 파일2는 동일 이미지에서 0.66~0.68 수준. blur 심한 이미지에서는 두 버전 모두 0.59~0.71로 비슷 → **sharp 영역의 응답 곡선이 알고리즘 변경의 핵심 차이**.

### 대표 케이스 비교 (3건 발췌)
| Test | BEW | HSMB(파일1) | HSMB(파일2) | 차이 |
|------|-----|-------------|-------------|------|
| 15000lx-500us-00km/h (sharp) | 3.23 | 0.8893 | 0.6280 | -0.2613 |
| 15000lx-500us-30km/h (mid) | 15.18 | 0.7071 | 0.6002 | -0.1069 |
| 15000lx-500us-70km/h (blur) | 35.37 | 0.6285 | 0.5913 | -0.0372 |

> blur 강도가 클수록 두 버전의 차이가 줄어듦 → 정규화 또는 sharp 응답 saturation 로직 변경 가능성.

## 식별 절차
1. **파일 내부 구조 분석**
   - 시트 목록, 컬럼 구조, HSMB 관련 지표 컬럼명 확인
   - 두 파일의 차이(컬럼/행/값) diff
2. **HSMB 버전 후보 수집**
   - 코드 저장소 내 HSMB 구현 버전 식별 (ex. v1, v2, …)
   - 각 버전의 출력 포맷·시그니처 정리
3. **재현 실행**
   - 각 후보 버전을 동일 입력(MTF 15000)으로 실행
   - 산출된 결과값을 xlsx의 값과 비교
4. **버전 매핑 확정**
   - 일치하는 버전 = 해당 xlsx의 산출 버전
   - 불일치 시 파라미터/전처리 차이 추적

- 동일 입력(MTF 15000) → HSMB 출력값의 **수치 일치도** (오차 < ε)

- 보조 지표(IQA 등)의 구현 일관성

## 결론 — 적합 버전 선정

### 양쪽 파일 산출 버전 확정

| `MTF(15000)-IQA 지표_0715.xlsx` | ps7310 v2 | t=0.3, β=1.8, w=0.8 + Lap+FFT 융합 | 7/15 시점 개선판, 좁은 범위 0.59~0.68 |

### 적합성 평가 (다음 단계)
어느 버전이 본 프로젝트(블러 정량화 + 크랙 검출)에 더 적합한지는 **별도 평가 필요**:
- v1 (파일1): sharp/blur 분리도 우수 (range 0.28), 단일 지표 기반
- v2 (파일2): 좁은 응답 범위(0.09)지만 다중 특성 융합(Lap+FFT)으로 robust 가능성

→ d2040(NR-IQA 측정), d2050(BEW 상관), d2060(크랙 검출 상관) 단계에서 두 버전 모두 평가하여 최종 선정.
