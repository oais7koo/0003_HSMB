## 2. 목적

`d1600` 분석 및 `d0900` 버전 검증 결과:
- 논문 적합 버전(`MTF(15000)-IQA 지표.xlsx`, PLCC -0.9532)은 **ps5010 HSMB v1** (ew=1.5/β=2.0/JNB=3/cdf[63])
- PRD §4.1 사양과 완전 일치 (Δ=0.0000)
- 0715 열등 버전은 ps7310 HSMB v2 (ew=0.8/β=1.8/Lap+FFT, PLCC -0.7630)

본 문서는 다음 두 가지를 체계적으로 수행하기 위한 기획이다:

1. `ps5010 HSMB v1`을 기준 코드로 확정 → 버전 계통 정리
2. 기준 코드 위에서 파라미터 grid search → **논문 제출용 최적 파라미터 확정 및 C4 근거 확립**

## 5. 파라미터 탐색 설계

### 5.1 탐색 목적

d1600 결과(PLCC -0.9532)를 기준점으로, 더 높은 상관성을 내는 파라미터 조합을 찾거나
현재값이 local optimum임을 입증한다 (논문 C4 대응).

### 5.2 탐색 대상 파라미터 및 grid

| 파라미터 | 기준값 (ps5010 v1) | 탐색 범위 | grid 점 |
|---------|-----------------|---------|--------|
| edge_weight | **1.5** | 0.5 ~ 2.0 | 0.5, 0.8, 1.0, 1.2, 1.5, 1.8, 2.0 |
| beta | **2.0** | 1.0 ~ 3.6 | 1.0, 1.5, 1.8, 2.0, 2.5, 3.0, 3.6 |
| JNB | **3.0** | 2.0 ~ 4.0 | 2.0, 2.5, 3.0, 3.5, 4.0 |
| cdf 인덱스 | **63** | 40 ~ 70 | 40, 45, 50, 55, 60, 63, 65, 70 |

> grid 총 조합: 7 × 7 × 5 × 8 = **1,960 조합**
> 기준값이 grid 중심 부근에 위치하여 local optimum 여부 확인 가능
> 실험 데이터: `data/ps1204_kict_eSFR/` 15000lx 20개 조건 (d1600 기준 데이터)

### 5.3 탐색 실험 데이터

| 데이터셋 | 조건 수 | 역할 |
|---------|--------|------|
| `ps1204_kict_eSFR/` 15000lx | 20개 | 주 탐색 데이터 (d1600 기준) |
| `ps1204_kict_eSFR/` 40000lx | 28개 | 검증 데이터 (교차 검증용, 추후) |
| `ps1010_chungsong_MTF/` | 50조건 | 최종 검증 (논문 벤치마크 데이터) |

### 5.4 평가 지표

| 지표 | 설명 | 목표 |
|------|------|------|
| PLCC (Pearson) | HSMB vs BEW 선형 상관 | 절대값 최대화 |
| SROCC (Spearman) | HSMB vs BEW 순위 상관 | 절대값 최대화 |
| 단조성 | 속도 증가 → HSMB 감소 여부 | 단조감소 확인 |

## 6. 구현 계획

### 6.1 산출물

| 항목 | 경로 |
|------|------|
| 탐색 실행 스크립트 | `src/ps5000_hsmb_param_search.py` |
| 탐색 결과 CSV | `data/exp/ps5000_param_search_results.csv` |
| 최적 파라미터 리포트 | `00_doc/sp00/d5001_hsmb_param_search_result.md` |

### 6.2 실행 단계

| Step | 내용 | 산출물 |
|------|------|--------|
| 1 | v8 코드 기준 파라미터 grid 정의 | `src/ps5000_hsmb_param_search.py` |
| 2 | 15000lx 20개 조건 × 1,470 조합 일괄 실행 (multiprocessing) | CSV 누적 저장 |
| 3 | PLCC/SROCC 매트릭스 → 상위 20개 조합 추출 | 리포트 |
| 4 | 상위 조합으로 40000lx 교차 검증 | 교차 검증 결과 |
| 5 | 최적 파라미터 확정 → ps2000 코드 반영 | 코드 수정 + 리포트 |

### 6.3 중간 저장 및 재시작

- 매 조합 완료 시 CSV append (idempotent, 재시작 시 완료 조합 skip)
- 예상 소요: 20조건 × 10장 × 1,470조합 / 4-worker ≈ 2~3시간
