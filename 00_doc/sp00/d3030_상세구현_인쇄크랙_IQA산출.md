## 3. 기능 개요

프린트 크랙 이미지 데이터셋(`ps1302_print_crack`, 50장)에 대해 **d0100 표준 7종 NR-IQA**(9컬럼)를 `common_iqa7.py`로 일괄 산출한다. PRD v06 §2.7 E3 실험군의 세 번째 작업으로, 통제된 환경(프린트 크랙)의 IQA 분포를 파악하여 실제 크랙(E3-2, ps1301)의 controlled 비교군으로 활용한다.

> **center crop 적용 (v04)**: ps1302 이미지는 대용량이므로, 이미지 전체가 아닌 **가로·세로 가운데 50%(중앙 영역, 면적 25%)** 만 사용하여 IQA를 산출한다. `common_iqa7.compute_all_from_path(center_crop=True)`로 적용한다. E3-2(ps1301)와 동일한 crop 기준으로 controlled 비교의 정합성을 유지한다.

E3-1·E3-2와 동일하게 **A안**(IQA 9컬럼 산출만)으로 진행한다 — ps1302는 SFR 차트가 아니므로 §SFR 상관은 적용하지 않는다.

> **E3-1·E3-2와의 구조 차이**: ps1302는 **조건 폴더가 없다**. png 50장이 데이터셋 루트에 평면 배치되며, **파일명 자체가 조건**(조건당 1장)이다. 따라서 "폴더 순회 → 폴더 내 frame 수집" 방식이 아니라 **파일 직접 순회 + 파일명 파싱** 방식을 쓴다.

## 5. 입출력 정의

### 5.1 입력

| 항목 | 타입 | 설명 | 필수 |
|------|------|------|------|
| `data/ps1302_print_crack/` | folder | 루트 직속 `cam1_*.png` 50장 (조건 폴더 없음) | Y |
| `src/common_iqa7.py` | Python | 7종 IQA 정식 산출 라이브러리 (`compute_all_from_path()`) | Y |

### 5.2 출력

| 항목 | 타입 | 설명 |
|------|------|------|
| `src/ps3030_e3_3_ps1302_iqa.py` | Python | ps1302 50장 일괄 실행 진입점 (common_iqa7 import) |
| `data/ps3030/01_전체데이터_{dsr}.xlsx` | XLSX | 이미지별 9컬럼 스코어 + 메타 (50행) |
| `data/ps3030/04_histogram_{dsr}.png` | PNG | 9종 메트릭 히스토그램 |
| §11 실험 결과 (본 문서 통합) | MD | 결과 요약 + 분포 표 + 히스토그램 |

> 조건당 1장이므로 E3-1·E3-2의 `02_그룹별통계`·`03_그룹별평균`은 산출하지 않는다 (조건별 평균 = 단일 이미지 값과 동일).

### 5.3 출력 스키마 (`data/ps3030/01_전체데이터_{dsr}.xlsx`)

```
filename, speed_kmh, iso, distance_m, is_defocus, is_excluded,
hsmb, cpbd, niqe, niqe_matlab, piqe, brisque, brisque_matlab, dbcnn, arniqa
```

- `filename`: 이미지 파일명 (예: `cam1_v80_iso400_d45.png`)
- `distance_m`: `d45` → `4.5` (× 0.1 변환)
- `is_defocus`: ISO=400 조건이면 1, 아니면 0
- `is_excluded`: 파일명에 `_x` 접미사가 있으면 1 (사용 불가 이미지)
- IQA 9컬럼: common_iqa7 `METRIC_COLUMNS`
- 공통 규칙: d0100 §6.4 — UTF-8, 소수점 4자리(`.4f`), 결측 `NaN`

| 파일명 파싱 실패 | 경고 로그 + 해당 파일 스킵 (수동 검토용 리스트) |

| common_iqa7 의존성 미설치 (pyiqa/cpbd) | GPU 환경 `uv sync` + `patch_cpbd.py` 선결 (PRD §7.1) |
