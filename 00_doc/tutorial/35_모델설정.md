# 시나리오: 모델 설정

> Claude 모델 선택 전략 — 기본 Sonnet, 복잡 작업 Opus, 경량 탐색 Haiku

---

## 개요

Claude Code는 단일 모델을 고정해 쓰는 것이 아니라, 작업 성격에 따라 모델을 달리 써서 비용과 성능을 최적화합니다.
OAIS는 세 가지 레벨로 모델을 운용합니다.

| 레벨 | 모델 | 용도 |
|------|------|------|
| **기본** | Sonnet 4.6 | 일반 대화, 표준 구현, 디버깅, 리뷰 |
| **복잡** | Opus 4.6 | 아키텍처 설계, 심층 분석, 계획 수립 |
| **경량** | Haiku 4.5 | 파일 탐색, 빠른 스캔, 요약 |

**이럴 때 확인:** 응답이 너무 느리거나 비용이 높을 때, 모델을 명시적으로 바꾸고 싶을 때
**설정 위치:** `.claude/settings.json` + `common_guide.md` 섹션 2.13

---

## 1. 기본 모델 설정 (settings.json)

```json
// .claude/settings.json
{
  "model": "claude-sonnet-4-6"
}
```

이 설정이 **세션 기본값**입니다. 지정하지 않으면 Claude Code 기본 모델이 사용됩니다.

**세션 중 임시 전환:**
```
/model claude-opus-4-6      # 현재 세션만 Opus로
/model claude-sonnet-4-6    # 현재 세션만 Sonnet으로
```

## 2. OMC 에이전트 자동 라우팅

oo 스킬이 서브에이전트를 spawn할 때 OMC가 자동으로 모델을 선택합니다.
직접 개입 없이 아래 규칙이 적용됩니다.

### 에이전트별 모델 매핑

| 모델 | 에이전트 | 대표 oo 스킬 |
|------|----------|-------------|
| **haiku** | `explore`, `writer` | 모든 스킬의 탐색 단계 |
| **sonnet** | `executor`, `debugger`, `verifier`, `quality-reviewer`, `test-engineer`, `security-reviewer` | oodev, oofix, oocheck, ootest |
| **opus** | `architect`, `analyst`, `planner`, `critic`, `code-reviewer` | ooplan, ooprd, ooresearch |

### 실행 흐름 예시

```
oodev run 실행 시:
  탐색 → explore(haiku)        # 빠르고 저렴하게 코드베이스 파악
  구현 → executor(sonnet)      # 표준 구현
  리뷰 → quality-reviewer(sonnet)  # 코드 품질 검토

ooplan run 실행 시:
  탐색   → explore(haiku)
  계획   → planner(opus)       # 복잡한 계획은 Opus 필요
  설계   → architect(opus)
  검토   → critic(opus)
```

## 3. 수동으로 모델 지정 (Agent 도구)

oo 스킬 프롬프트나 직접 Agent 호출 시 명시적으로 모델 지정 가능:

```python
# Opus: 아키텍처, 심층 분석
Task(subagent_type="oh-my-claudecode:architect", model="opus", ...)

# Sonnet: 일반 구현
Task(subagent_type="oh-my-claudecode:executor", model="sonnet", ...)

# Haiku: 빠른 탐색
Task(subagent_type="oh-my-claudecode:explore", model="haiku", ...)
```

## 4. SKILL.md에 모델 명시 방법

oo 스킬의 서브에이전트 테이블에 `| 모델 |` 컬럼을 추가합니다:

```markdown
## 서브에이전트

| 단계 | 에이전트 | 모델 | 역할 | 병렬 |
|------|----------|------|------|:----:|
| 탐색 | Explore | haiku | 파일 탐색 | O |
| 구현 | executor | sonnet | 기능 구현 | O |
| 검증 | verifier | sonnet | 완료 검증 | - |
| 설계 | architect | opus | 복잡 설계 | - |
```
