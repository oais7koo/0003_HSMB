# 시나리오: 스킬 관리

> oo 스킬 생성, 검증, 최적화, 문서화

---

## 개요

**스킬이란?** 반복하는 작업을 Claude에게 한 번 가르쳐 두면, 다음부터는 짧은 명령어 하나로 실행할 수 있게 해주는 자동화 레시피다.

**이럴 때 만든다:**
- 같은 작업을 여러 번 반복하고 있을 때
- 매번 길게 설명해야 하는 작업이 있을 때
- 결과가 일관되지 않아서 품질을 고정하고 싶을 때

**결과물:** `.claude/skills/oo{스킬명}/` 폴더 + 검증된 SKILL.md

---

## 전체 흐름

```
① 스킬 생성 (Claude에게 요청)
② 실제 작업에 적용해보기
③ 결과가 맘에 안 들면 → 스킬에 규칙 추가
④ ooskill run 으로 품질 검증
⑤ (선택) oosync로 다른 프로젝트에 배포
```

---

## 1. 스킬 구조 이해

> 몰라도 쓸 수 있다. 참고용으로만 봐도 됨.

```
.claude/skills/oo{스킬명}/
├── SKILL.md          # 스킬 정의 — Claude가 읽는 "레시피북"
├── scripts/          # 자동화 스크립트 (Python 등)
└── references/
    └── checklist.md  # 결과 검증용 체크리스트
```

---

## 2. 스킬 생성

### 가장 쉬운 방법: Claude에게 말하기

직접 폴더를 만들거나 파일을 작성할 필요 없다. 그냥 Claude에게 요청하면 된다.

```
[사용자] CSS를 Bootstrap 5로 변환하는 스킬을 만들어줘.
         이름은 oocssconv 로 하고, 변환 규칙은 내가 나중에 추가할게.

[Claude] 알겠습니다. .claude/skills/oocssconv/ 구조를 생성합니다...
```

Claude가 SKILL.md, scripts/, references/checklist.md를 모두 만들어준다.

### 스킬 구조만 참고할 때 (고급)

```bash
# 완성형 템플릿 참고
.claude/skills/ootutorial/templates/SKILL_md_template.md
```

---

## 3. 스킬 검증

스킬을 만들거나 수정한 뒤 품질을 확인한다.

```bash
ooskill run        # 전체 스킬 품질 검증
ooskill check      # 체크리스트 기반 검증
```

**검증이 확인하는 것:**
- SKILL.md가 있고 형식이 맞는지
- 명령어(run, check 등)가 제대로 정의됐는지
- 스크립트가 실행 가능한지

---

## 4. 문서 최적화

스킬 문서가 너무 길어지면 Claude 컨텍스트를 낭비한다. 3KB 이내로 유지한다.

```bash
oodoc optimize     # 스킬 문서 3KB 이내로 자동 정리
oodoc check        # 다른 문서와 충돌 여부 확인
```

---

## 5. 배포 (동기화)

만든 스킬을 다른 프로젝트에서도 쓰고 싶을 때.

```bash
oosync run --push-only   # 연결된 프로젝트로 동기화
```

## 6. 커스텀 스킬 개발 — 반복 개선 방법론

반복적인 작업(포맷 변환, 데이터 정제, 코드 컨버팅 등)은 스킬로 만들면
**토큰 절약 + 일관된 결과**를 얻을 수 있다.
처음부터 완벽할 필요 없다. 돌려보고 고치면 된다.

### 기본 사이클

```
① Claude에게 스킬 만들어달라고 요청
      ↓
② 실제 작업에 적용 (스킬 실행)
      ↓
③ 결과 검토 → 원하는 것과 다른 부분 확인
      ↓
④ "이 부분은 이렇게 해줘" → Claude가 SKILL.md에 규칙 추가
      ↓
② 다시 적용 → 반복
```

### 단계별 실행 예시

**① 초기 스킬 생성**

```
[사용자] 레거시 CSS를 Bootstrap 5로 컨버팅하는 스킬 만들어줘.
         oocssconv 라는 이름으로.
```

Claude가 스킬 구조 전체를 생성한다.

**② 실행 및 결과 확인**

```bash
oocssconv run style.css
# 결과물 확인 → 잘못 변환된 부분 메모
```

**③ 규칙 추가**

결과가 맘에 안 들면 **구체적으로** Claude에게 말한다. 막연하게 말하면 규칙이 모호해진다.

**나쁜 예 (너무 모호)**:
```
[사용자] 결과가 좀 이상해. 고쳐줘.
```

**좋은 예 (구체적)**:
```
[사용자] oocssconv SKILL.md에 아래 규칙을 추가해줘.

         금지조건:
         - .container, .row, .col-* 같은 기존 레이아웃 클래스명은 절대 변경하지 말 것
         - @media 쿼리 블록은 삭제하거나 병합하지 말 것
         - !important 선언은 제거하지 말 것

         가이드:
         - 색상은 Bootstrap 5 변수($primary, $secondary 등)로 교체
         - margin/padding은 Bootstrap 유틸리티 클래스(mt-2, px-3 등)로 변환
         - 커스텀 폰트 선언은 variables.scss 파일로 분리

[Claude] SKILL.md 변환 규칙 섹션에 추가하겠습니다...
```

Claude가 SKILL.md에 규칙을 추가해준다:
```markdown
## 변환 규칙

### 금지조건 (절대 하지 말 것)
- 기존 레이아웃 클래스(.container, .row, .col-*) 변경
- @media 쿼리 블록 삭제 또는 병합
- !important 선언 제거

### 가이드 (이렇게 변환할 것)
- 색상 → Bootstrap 5 변수($primary, $secondary 등)
- margin/padding → 유틸리티 클래스(mt-2, px-3 등)
- 커스텀 폰트 → variables.scss 분리
```

**규칙 유형 구분**:

| 유형 | 언제 쓰나 | 표현 방법 |
|------|----------|-----------|
| 금지조건 | "이건 하면 안 된다" | "절대 하지 말 것: ..." |
| 가이드 | "이렇게 하는 게 좋다" | "~로 변환할 것" |
| 예외 케이스 | "이 경우만 다르게" | "단, ~일 때는 ..." |

> **팁**: 규칙은 한 번에 다 추가하려 하지 말고, 실행 결과에서 문제가 생길 때마다 하나씩 추가하는 게 효과적이다.

### check 서브명령어로 자동 검증

스킬이 안정되면 `check` + 체크리스트를 추가해 변환 품질을 자동으로 확인하게 한다.

```
[사용자] oocssconv check 명령어를 추가해줘.
         체크리스트: 클래스명 보존, breakpoint 유지, 인라인 스타일 제거
```

이후 사용 흐름:
```bash
oocssconv run style.css    # 변환 실행
oocssconv check            # 체크리스트 기준으로 결과 검토
```

### 핵심 원칙

| 원칙 | 설명 |
|------|------|
| Claude에게 말로 시킨다 | mkdir, 파일 작성 등 직접 할 필요 없음 |
| 스킬에 규칙을 누적한다 | 실패 사례 → Claude에게 말하면 SKILL.md에 추가됨 |
| check로 검증을 자동화한다 | 변환 후 수동 검토 대신 체크리스트 자동 실행 |
| 작게 시작한다 | 처음부터 완벽하지 않아도 됨 — 돌려보고 개선 |

## 공통 서브명령어 (모든 oo 스킬)

| 서브명령어 | 설명 |
|-----------|------|
| `help` | 서브명령어 목록 |
| `version` | 버전 정보 |
| `status` | 상태 확인 |
| `check` | 체크리스트 검증 |
| `show checklist` | 체크리스트 표시 |
| `add checklist "항목"` | 체크리스트 추가 |
| `run` | 기본 실행 |

## FAQ

**Q: 스킬 만들 때 Python을 알아야 하나?**
A: 아니다. Claude에게 "스킬 만들어줘"라고 하면 스크립트까지 다 만들어준다.

**Q: 스킬 카테고리는?**
A: core-dev (개발핵심), doc-env (문서/환경), meta-util (메타/유틸), content (콘텐츠)

**Q: 외부 스킬 검색은?**
A: `skillsmp-search "키워드"` (SKILLSMP_API_KEY 필요)
