# 시나리오: 스킬 관리

> oo 스킬 생성, 검증, 최적화, 문서화

---

## 개요

oo 스킬의 생성, 검증, 최적화를 관리합니다. 현재 38개 oo 스킬이 있으며, 각 스킬은 SKILL.md 정의 + scripts/ 실행 스크립트 + references/checklist.md 구조를 따릅니다. ooskill로 품질을 검증하고, oodoc으로 문서를 최적화합니다.

**이럴 때 사용:** 새 스킬을 만들 때, 기존 스킬을 개선할 때, 외부 스킬을 검색할 때
**결과물:** 검증된 스킬 + 최적화된 SKILL.md


## 전체 흐름

```
스킬 생성 → ooskill 검증 → oodoc 문서화 → oosync 배포
```

---

## 1. 스킬 구조 이해

```
.claude/skills/oo{스킬명}/
├── SKILL.md                    # 스킬 정의 (필수)
├── scripts/                    # 실행 스크립트
│   └── oo{스킬명}_run.py
└── references/
    └── checklist.md            # 체크리스트
```

### SKILL.md 필수 요소
```yaml
---
name: oo{스킬명}
description: "설명"
metadata:
  version: "v01"
  category: "core-dev|doc-env|meta-util|content"
---
```

---

## 2. 스킬 생성

### 수동 생성
```bash
mkdir -p .claude/skills/oo{새스킬}/scripts
mkdir -p .claude/skills/oo{새스킬}/references
```
- SKILL.md 작성 (명령어, 워크플로우, 서브에이전트)
- scripts/ 에 실행 스크립트 배치
- references/checklist.md 작성
- 완성형 템플릿: `.claude/skills/ootutorial/templates/SKILL_md_template.md`

---

## 3. 스킬 검증

```bash
ooskill run        # 전체 스킬 검증
ooskill check      # 체크리스트 기반 검증
```

**검증 항목:**
- SKILL.md 존재 및 형식
- metadata version 일치
- 서브명령어 7종 표준 준수 (help, version, status, check, show checklist, add checklist, run)
- scripts/ 실행 가능 여부

---

## 4. 문서 최적화

```bash
oodoc optimize     # 스킬 문서 3KB 이내 최적화
oodoc check        # 교차 정합성 검사
```

---

## 5. 배포 (동기화)

```bash
oosync run --push-only   # 다른 프로젝트로 동기화
```

---

## 공통 서브명령어 (모든 oo 스킬)

| 서브명령어 | 설명 |
|-----------|------|
| `help` | 서브명령어 목록 |
| `version` | 버전 정보 |
| `status` | 상태 확인 |
| `check` | 체크리스트 검증 |
| `show checklist` | 체크리스트 표시 |
| `add checklist "항목"` | 체크리스트 추가 |
| `run` | 기본 실행 |

## FAQ

**Q: 스킬 카테고리는?**
A: core-dev (개발핵심), doc-env (문서/환경), meta-util (메타/유틸), content (콘텐츠)

**Q: 외부 스킬 검색은?**
A: `skillsmp-search "키워드"` (SKILLSMP_API_KEY 필요)
