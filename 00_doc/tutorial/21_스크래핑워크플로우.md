## 개요

유튜브 영상과 웹 기사를 자동으로 수집하여 마크다운 텍스트로 보관하는 워크플로우입니다. 유튜브는 자막을 추출(STT)하고, 웹 기사는 본문을 스크래핑합니다. 채널 단위 일괄 처리와 중단 후 재개가 가능합니다.

**이럴 때 사용:** 유튜브 영상을 텍스트로 읽고 싶을 때, 기술 블로그를 옵시디언에 보관할 때
**결과물:** 01_유튜브서머리/*.md + 04_읽을거리/*.md + 유튜브팩토리.md 인덱스


## 전체 흐름

```
URL 등록 (01_다운로드.md) → ooscrap run → 이력 기록 → 팩토리 갱신
```

## URL 자동 감지 시스템

### 1. URL 패턴 (자동 분류)

`04_scraping/00_down/01_다운로드.md`의 `## 다운` 섹션에 URL을 넣으면 패턴으로 자동 분류됩니다.

| URL 패턴 | 종류 | 처리 | 출력 | 이력 |
|----------|------|------|------|------|
| `youtube.com/shorts/*` | 유튜브 쇼츠 | **스킵** (즉시 삭제) | - | - |
| `youtu.be/*`, `youtube.com/watch*` | 유튜브 동영상 | STT (자막→Whisper 폴백) | `01_유튜브서머리/` | `12_유튜브동영상리스트.md` |
| `youtube.com/@*`, `youtube.com/channel/*` | 유튜브 채널 | 채널 등록 + 영상 목록 → `## 다운`에 추가 | - | `13_유튜브채널리스트.md` |
| 그 외 모든 URL | 웹 기사/블로그 | 전문 스크래핑 | `04_읽을거리/` | `14_읽을거리리스트.md` |

## 방법 1: 통합 배치 처리 (모든 URL 한 번에)

### 1. URL 등록
```markdown
## 다운
https://youtu.be/xxxxx
https://youtube.com/@채널명
https://blog.example.com/article
```

마크다운 링크 형식도 지원:
```markdown
## 다운
- [동영상 제목](https://youtu.be/xxxxx)
- [채널명](https://youtube.com/@channel)
```

### 2. 실행
```bash
oocontext 4           # SP04 전환
ooscrap run           # 전체 처리 (URL 자동 감지 + 분류)
ooscrap run --url <URL>  # 특정 URL만 처리
```

### 3. 처리 결과
- **성공** → `01_다운로드.md`에서 자동 제거
- **실패** → `# [FAIL] URL | 사유` 주석 처리
- **스킵** (중복, 쇼츠) → `01_다운로드.md`에서 즉시 제거 (이력 기록 없음)
- **이력 기록** → 파일별로 자동 갱신:
  - `12_유튜브동영상리스트.md` (개별 동영상)
  - `13_유튜브채널리스트.md` (채널 메타)
  - `14_읽을거리리스트.md` (웹 기사)

---

## 방법 2: 채널 처리 (2단계)

### Phase 1 (채널 스캔)
```markdown
## 다운
https://youtube.com/@happyai_0612
```

```bash
ooscrap run
```

**동작**:
1. 채널 URL 인식 → yt-dlp로 영상 목록 추출
2. 쇼츠(60초 미만) 자동 제외
3. `12_유튜브동영상리스트.md` 대비 중복 제거
4. 신규 영상 URL을 `## 다운`에 추가 (채널 URL은 즉시 삭제)
5. `13_유튜브채널리스트.md`에 채널 메타 기록 (채널명/마지막처리일/추가된 영상수)

### Phase 2 (개별 STT 처리)
```bash
ooscrap run
```
추가된 영상 URL들을 순차 처리 (STT/자막 추출)

## 방법 3: 채널 신규 영상 자동 추가

```bash
ooscrap sync                       # 등록된 모든 채널 스캔
ooscrap sync --channel <URL>       # 특정 채널만 스캔
ooscrap sync --dry-run             # 신규 영상 목록만 출력 (추가 안 함)
```

**동작**:
1. `13_유튜브채널리스트.md`에서 등록 채널 목록 로드
2. 각 채널의 신규 영상 검색 (yt-dlp)
3. `12_유튜브동영상리스트.md` 대비 중복 제거
4. 신규 영상 URL을 `01_다운로드.md`의 `## 다운`에 자동 추가
5. `13_유튜브채널리스트.md` 메타 갱신 (마지막처리일, 추가 영상수)

## 주요 명령어

```bash
ooscrap help                   # 서브명령어 목록
ooscrap status                 # 상태 확인
ooscrap version                # 버전 정보
ooscrap check                  # 체크리스트 기반 검증

ooscrap summary                # 유튜브 STT (기본 모드)
ooscrap read                   # 웹 기사 전문 스크래핑

ooscrap add <URL> [URL...]     # URL 리스트에 추가
ooscrap add <URL> --video      # 동영상 다운로드 섹션에 추가
ooscrap add <URL> --music      # 음악 추출 섹션에 추가

ooscrap search "키워드"        # 서머리/읽을거리에서 검색
ooscrap search "키워드" --limit 10  # 최대 10건
ooscrap list                   # 다운로드 리스트 표시
```

## FAQ

**Q: naver 블로그/aitimes가 안 되는데?**
A: WebFetch에서 접근 차단. 브라우저에서 직접 복사 필요.

**Q: 채널 처리 중 중단하면?**
A: 미처리 URL이 `01_다운로드.md`에 남아서 재실행하면 이어서 처리.

**Q: 중복 URL도 강제로 재처리하려면?**
A: `ooscrap run --force` 사용.

**Q: 쇼츠는 왜 스킵되나?**
A: 쇼츠는 짧아서 서머리 가치 없음 (자동 스킵, 이력 기록 없음).

**Q: Whisper가 느릴 때는?**
A: CUDA 확인 후 CPU 폴백 자동 실행. 또는 자막 있는 영상부터 처리 추천.
