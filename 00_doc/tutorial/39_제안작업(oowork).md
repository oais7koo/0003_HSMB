# oowork 제안 사용법

> `oowork` 스킬의 **제안서 작성 프로세스** 실행 가이드

---

## 1. 개요

`oowork run 제안` 은 RFP 수령부터 제안서 완성까지 **5단계 파이프라인**을 순차 실행한다.
각 단계 완료 후 사용자 확인을 거쳐 다음 단계로 진행한다.

| 단계 | 이름 | 산출물 |
|------|------|--------|
| step.0 | RFP 분석 | `01_RFP/11_RFP분석.md`, `01_RFP/12_양식분석.md` |
| step.1 | 자료 수집 | `02_자료/2N_{자료명}.md` |
| step.2 | 기획 | `03_기획/31_작성전략.md` |
| step.3 | 기술 연구 | `04_기술연구/41_{주제}.md` |
| step.4 | 제안 작성 | `05_제안작성/51_{섹션명}.md` |

---

## 2. 폴더 구조

```
{작업폴더}/
├── 01_RFP/          # RFP·공고문·양식 원본 + 분석
├── 02_자료/         # 기업정보·실적·참고자료
├── 03_기획/         # 목차구조·작성전략
├── 04_기술연구/      # 기술분석·경쟁비교
├── 05_제안작성/      # 양식 입력값·붙여넣기용 본문
├── 06_행정/         # 제출 행정서류 (RFP별)
├── 07_검토/         # 외부검토·피드백
└── 08_최종/         # 최종본·제출본
```

---

## 3. 실행 명령어

### 기본 실행

```bash
# 전체 파이프라인
oowork run 제안 --topic "사업명" --workdir "경로"

# 특정 단계만
oowork run 제안 --step 0
oowork run 제안 --step 2

# 단계 범위 실행
oowork run 제안 --step 1-3

# 중단 후 재개
oowork run 제안 --resume

# 실행 계획 미리보기
oowork run 제안 --dry-run
```

### 실용 예시

```bash
# 건기원 지원사업 제안서 작성
oowork run 제안 --topic "건기원 지원사업 제안" --workdir "0000_work/260408_건기원 지원사업 제안서"

# RFP 분석만 먼저
oowork run 제안 --topic "칼리스26 제안" --step 0 --workdir "0000_work/15_KA칼리스26"

# 기술연구부터 제안작성까지
oowork run 제안 --topic "CC사업 제안" --step 3-4 --workdir "0000_work/12_CC척척파트너스"
```

## 4. 단계별 작업 내용

### step.0 — RFP 분석

**입력**: RFP 문서 (HWP → HWPX → PDF → MD 변환 후 읽기, PDF는 직접 Read)
**산출물**: `01_RFP/11_RFP분석.md`, `01_RFP/12_양식분석.md`

```bash
# HWP 원본 → HWPX → PDF → 마크다운 자동 변환
oowork extract 01_RFP/공고문.hwp

# PDF 변환 실패 시 XML 파싱으로 자동 폴백
```

**핵심 작업**:
1. RFP 전문 읽기 → 사업개요·목적·범위·일정·예산 추출
2. **원본 문서 목록 테이블 작성** (`11_RFP분석.md` 상단 필수)
3. 평가 기준 분석 (배점표, 가점/감점)
4. 필수 제출 서류 목록 정리
5. 기술 요구사항 항목별 분해
6. 차별화 포인트 도출
7. RFP 분석 결과 문서 생성 (`11_RFP분석.md`)
8. **양식 분석 문서 생성** (`12_양식분석.md`)

**`11_RFP분석.md` 상단 — 원본 문서 목록 (필수)**:
```markdown
## 원본 문서 목록
| # | 파일명 | 유형 | 페이지 | 분석 여부 |
|---|--------|------|--------|:---------:|
| 1 | 공고문.hwp | 공고 | 3p | O |
| 2 | 과업지시서.pdf | 과업 | 12p | O |
| 3 | 양식.hwp | 양식 | 5p | O (→12_양식분석.md) |
```
> 01_RFP 내 모든 원본 문서를 나열하고 분석 여부를 표시. 미분석 0건이어야 통과.

**`12_양식분석.md` 포함 내용**:
- 양식별 필드 목록 (필드명, 위치)
- 입력 형식 (텍스트/표/체크박스)
- 글자수 제한 — 해당 시
- 필수/선택 구분
- step.4 제안 작성 시 이 문서를 기준으로 필드별 입력값 작성

**체크리스트**:
- [ ] A01 문서명·발주처·유형 기재
- [ ] A02 원본 문서 목록 테이블 존재 (미분석 0건)
- [ ] A03 핵심 요구사항 추출 완료
- [ ] A04 양식 항목 빠짐없이 나열
- [ ] A05 제출 서류 목록 정리
- [ ] A06 제약 조건(기간/인력/기술) 명시
- [ ] A07 양식분석 문서 존재 (필드목록·입력형식·필수/선택)

---

### step.1 — 자료 수집

**입력**: `01_RFP/11_RFP분석.md`, `01_RFP/12_양식분석.md`
**산출물**: `02_자료/21_기업개요.md`, `22_매출현황.md`, `23_기술실적.md` 등

**핵심 작업**:
1. 기업 기본정보 (개요·대표자·연혁)
2. 매출·거래처·재무 현황
3. 기술 실적·인증·지재권
4. 유사 사업 수행 실적
5. 기존 제안서·기술문서 참고자료

**파일명 규칙**: `21_기업개요.md`, `22_매출현황.md`, `23_기술실적.md`

**체크리스트**:
- [ ] B01 02_자료/ 내 파일 1개 이상
- [ ] B02 RFP 요구 정보 포함
- [ ] B03 외부 자료 출처/날짜 기재

---

### step.2 — 기획

**입력**: step.0 + step.1 산출물
**산출물**: `03_기획/31_작성전략.md`, `32_항목매핑.md`

**핵심 작업**:
1. 제안서 목차·구조 설계 (양식 섹션 매핑)
2. 평가항목-목차 매핑표 작성
3. 배점 비중별 분량 배분
4. 장별 핵심 메시지 1줄 정의
5. 그림/표/다이어그램 배치 계획
6. 작성 마일스톤 (초안→검토→수정→최종)

**체크리스트**:
- [ ] C01 제안서 목차 또는 섹션 매핑 존재
- [ ] C02 주요 섹션에 핵심 메시지 1줄 정의
- [ ] C03 작성 일정(초안→검토→최종) 정의

---

### step.3 — 기술 연구

**입력**: step.2 + step.1 산출물
**산출물**: `04_기술연구/41_기술분석.md`, `42_경쟁비교.md` 등

**핵심 작업**:
1. 기술 현황 분석 (동향·문제점·개선방향)
2. 핵심 기술 상세 (아키텍처·알고리즘·방법론)
3. 경쟁사·경쟁 기술 비교 (품질/기술/가격)
4. 차별화 포인트 정리 (최소 3개)
5. 기술 검증·사업화 계획

**보조 도구**: `ooresearch`, `oopaper` (필요 시)

**체크리스트**:
- [ ] D01 04_기술연구/ 내 파일 1개 이상
- [ ] D02 차별화 포인트 최소 3개 이상
- [ ] D03 정량 근거 (성능 수치·비교 데이터) 포함
- [ ] D04 경쟁사·경쟁기술 비교 포함

---

### step.4 — 제안 작성

**입력**: step.2 + step.3 + step.1 산출물 (`12_양식분석.md` 기준)
**산출물**: `05_제안작성/51_기술파트.md`, `52_관리파트.md`, `53_통합본.md`

**본문 스타일 선택** (작성 시작 전 필수):

스킬이 등록된 본문 스타일 목록을 표시하고 선택을 요청한다.

| ID | 스타일명 | 용도 |
|----|---------|------|
| S01 | 개조식 3단 (기본) | 기술사업계획서, 정부과제, 공공사업 제안 |
| S02 | 서술식 | 연구보고서, 백서, 학술 제안 |
| S03 | 개조식 2단 (간결) | 요약서, 발표자료 원고, 짧은 제안 |
| S04 | 혼합식 (서술+개조) | 민간 제안서, 사업계획서, 투자 제안 |

- 번호 입력으로 선택 (미선택 시 기본값 S01)
- 새로운 스타일이 필요하면 `writing_styles.md`에 등록 후 사용
- 스타일 레지스트리: `.claude/skills/oowork/references/writing_styles.md`

**핵심 작업**:
1. RFP/양식 필드 목록 확인 (`01_RFP/12_양식분석.md` 기준)
2. 필드별 입력값 작성 (양식 형식에 맞게)
3. 장문 기재란 본문 작성 (HWP 붙여넣기용, **선택한 본문 스타일 적용**)
4. 양식 항목 전수 대응 검증 (누락 0건)
5. 용어 일관성·수치 정합성 검수
6. 오탈자/문법 검수

**출력 원칙** — 05_제안작성 산출물은 **양식에 직접 붙여넣기 가능한 형태**:
```
필드명: 입력값
장문기재란: (아래 코드블록)
```

**본문 스타일 예시 (S01 개조식 3단)**:
```
□ 사내 코드베이스 특화 AI 코딩 어시스턴트 개발
  ○ 사내 코드를 학습하여 컨벤션에 맞는 코드 자동 생성
    - GraphRAG 기반 의미론적 코드 검색으로 정확도 향상
```

**체크리스트**:
- [ ] E01 양식 항목 전수 대응 (미대응 0건)
- [ ] E02 동일 개념 용어 통일
- [ ] E03 문서 내 수치 교차검증 통과
- [ ] E04 맞춤법 검사 완료
- [ ] E05 통합본 존재 (`53_통합본.md`)
- [ ] E06 붙여넣기용 txt 생성 — 해당 시

## 5. 비텍스트 문서 변환

HWP/PDF/DOCX 원본이 있으면 **반드시 먼저 마크다운 변환** 후 작업.

```bash
# HWP → 마크다운 (HWPX → PDF → MD 파이프라인)
oowork extract 01_RFP/공고문.hwp

# 출력 경로 지정
oowork extract 01_RFP/제안요청서.hwp 01_RFP/12_제안요청서.md
```

| 형식 | 변환 방법 |
|------|----------|
| HWP | `pyhwpx` → HWPX → PDF 변환 → `pdfplumber` → 마크다운 |
| HWPX | `pyhwpx` → PDF 변환 → `pdfplumber` → 마크다운 |
| PDF | `Read` 내장 또는 `pdfplumber` |
| DOC/DOCX | `python-docx` |
