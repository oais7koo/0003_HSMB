# 시나리오: 연구노트 작성

> 실험 기록, 회의 메모, 아이디어를 날짜별로 체계 관리

---

## 개요

oonote 스킬로 연구 과정에서 발생하는 실험 결과, 분석 메모, 회의록, 아이디어 등을 날짜/시간 기반으로 체계적으로 기록합니다. 서브프로젝트별로 독립된 연구노트 파일(d{SP}0011_연구노트.md)을 관리합니다.

**이럴 때 사용:** 실험 결과를 기록할 때, 아이디어를 메모할 때, 회의 내용을 정리할 때
**결과물:** 00_doc/d{SP}0011_연구노트.md

## 전체 흐름

```
oonote init → oonote add "내용" → oonote today → oonote search "키워드"
```

---

## 1. 연구노트 초기 생성

```bash
oonote init           # 연구노트 파일 생성 (없을 때만)
```

파일 위치:

| SP | 파일 경로 |
|:--:|----------|
| 00 | 00_doc/sp00/d0011_연구노트.md |
| 02 | 00_doc/sp02/d20011_연구노트.md |
| 03 | 00_doc/sp03/d30011_연구노트.md |

## 2. 노트 추가

### 기본 추가

```bash
oonote add "Loss: 0.234, Accuracy: 91.2% 달성. 이전 대비 1.8% 향상"
```

자동으로 현재 날짜/시간 섹션에 기록됩니다:

```markdown
## 2026-03-25 (화)

### 14:30 | 메모
Loss: 0.234, Accuracy: 91.2% 달성. 이전 대비 1.8% 향상

---
```

### 옵션 활용

```bash
# 제목과 태그 지정
oonote add --title "모델 A 결과" --tag "실험" "Loss: 0.234"

# 참조 문서 연결
oonote add --ref "d0002_plan.md" --tag "분석" "Phase 2 계획 검토 완료"

# 시간 수동 지정
oonote add --time "09:00" --tag "회의" "주간 미팅 - 다음 주 배포 일정 확정"

# 다른 SP에 기록
oonote add --sp 03 --tag "참고" "RAG 논문 3편 추가 검토 필요"
```

## 3. 조회

```bash
oonote today              # 오늘 작성 노트
oonote list 2026-03-20    # 특정 날짜
oonote list --all         # 전체 날짜 목록
```

---

## 4. 검색

```bash
oonote search "Loss"              # 키워드 검색
oonote search "Loss" --tag "실험"  # 특정 태그 내 검색
oonote search "Loss" --date "2026-03"  # 특정 월만
oonote search "Loss" --limit 5    # 최근 5개만
```

출력 예시:
```
검색: "Loss"  |  3건 발견

[2026-03-25 14:30] [실험] 모델 A 결과
  Loss: 0.234, Accuracy: 91.2%

[2026-03-24 11:00] [실험] 베이스라인 측정
  Loss: 0.252, Accuracy: 89.4%
```

## 태그 체계

| 태그 | 용도 | 예시 |
|------|------|------|
| `[실험]` | 실험 수행 및 결과 | 모델 학습 결과, 하이퍼파라미터 |
| `[분석]` | 데이터/코드 분석 | 성능 분석, 코드 리뷰 |
| `[회의]` | 미팅/토론 내용 | 주간 미팅, 리뷰 |
| `[아이디어]` | 아이디어/가설 | 새로운 접근법, 가설 |
| `[문제]` | 발견된 문제/버그 | 성능 저하, 오류 |
| `[해결]` | 문제 해결 과정 | 디버깅, 원인 분석 |
| `[참고]` | 논문/문서 참조 메모 | 논문 리뷰, 레퍼런스 |
| `[계획]` | 향후 작업 계획 | 다음 실험, 마일스톤 |

## FAQ

**Q: d0004_todo.md와 다른 건가?**
A: d0004는 **이슈 추적** (버그/태스크), d0011은 **연구 기록** (실험/메모)입니다. 성격이 다릅니다.

**Q: 노트가 너무 길어지면?**
A: 월 단위로 자연스럽게 섹션이 나뉩니다. 필요하면 과거 기록을 별도 파일로 아카이브할 수 있습니다.

**Q: 옵시디언에서도 볼 수 있나?**
A: 네. 00_doc/ 하위 마크다운 파일이므로 01_obsidian/에서 링크하면 됩니다.
