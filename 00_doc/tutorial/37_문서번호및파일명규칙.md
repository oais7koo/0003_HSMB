# 33. OAIS 문서번호 및 파일명 규칙

> **기준 문서**: `.claude/skills/oodoc/references/doc_numbering.md` (SSOT)
> 이 튜토리얼은 위 레퍼런스를 기반으로 작성됨. 상세 규칙은 레퍼런스 파일 참조.

---

## 개요

OAIS 문서는 **SP(서브프로젝트) 번호 + 문서 일련번호**로 구성된 고유 ID를 가진다.

```
d  {SP 2자리}  {문서번호 4자리}  _  {문서명}.md

예) d40001_prd.md     → SP04, 번호 0001 (PRD)
    d41001_상세기획_데이터수집소스.md  → SP04, 번호 1001 (상세 문서)
```

---

## 1. 번호 범위 요약

| 범위 | 성격 | 규칙 |
|------|------|------|
| **0001~0019** | 예약 (핵심) | 지정 스킬만 생성 — 임의 생성 금지 |
| **0020~0099** | 선택 (자유) | 스킬 우선, 빈 번호 SP별 자유 할당 |
| **0100~0999** | 선택 (스킬 전용) | 해당 스킬 사용 시만 생성 |
| **1001~9999** | 상세 문서 | oofeature 전용 |

| 번호 | 파일명 | 용도 | 스킬 |
|------|--------|------|------|

## 5. 상세 문서 (1001~9999) — oofeature 전용

```
d{SP번호}{기능번호}_상세{단계}_{기능명}.md

예) d41001_상세기획_데이터수집소스.md   ← 기획 단계
    d41001_상세설계_데이터수집소스.md   ← 설계 단계 (rename)
    d41001_상세개발_데이터수집소스.md   ← 개발 단계 (rename)
    d41001_상세검증_데이터수집소스.md   ← 검증 단계 (rename)
```

**단계 아이콘**:

| 단계 | 아이콘 | 파일명 키워드 |
|------|--------|-------------|
| 기획 | ⚪ | `_상세기획_` |
| 설계 | 🔵 | `_상세설계_` |
| 개발 | 🟡 | `_상세개발_` |
| 검증 | 🟢 | `_상세검증_` |
| 완료 | ✅ | plan.md 8.2절 표시 |

**번호 부여**: `oofeature list`로 현재 번호 확인 후 +1 순차 부여.
- SP04 첫 상세 문서 → `d41001`
- SP05 첫 상세 문서 → `d51001`
