# 시나리오: 딥러닝 학습

> GPU 최적화, 학습 실행, 모니터링

---

## 개요

딥러닝 모델 학습을 위한 GPU 환경 최적화와 학습 실행 워크플로우입니다. CUDA 환경 확인, Mixed Precision 최적화, 학습 모니터링, 결과 분석까지 oodeep 스킬이 지원합니다. GPU 사용 불가 시 자동으로 CPU로 폴백합니다.

**이럴 때 사용:** 모델 학습 시 GPU 최적화가 필요할 때, 학습 결과를 체계적으로 분석할 때
**결과물:** 학습된 모델 + 학습 로그 + 결과 리포트


## 전체 흐름

```
oodeep 환경 확인 → 데이터 준비 → 학습 실행 → 모니터링 → 결과 분석
```

---

## 1. GPU 환경 확인

```bash
oodeep run         # GPU 환경 점검 + 최적화 제안
```

**확인 항목:**
- CUDA 버전 및 호환성
- GPU 메모리 (VRAM)
- cuDNN 설치 상태
- PyTorch/TensorFlow GPU 인식

---

## 2. 데이터 준비

```bash
oodev run          # 데이터 로더, 전처리 코드 구현
ootest run         # 데이터 파이프라인 테스트
```

**체크 항목:**
- 데이터 로딩 속도
- 전처리 정확성
- 메모리 사용량

---

## 3. 학습 실행

```bash
oodeep train       # 학습 실행
```

**최적화 옵션:**
- Mixed Precision (float16) → 메모리 절약 + 속도 향상
- Gradient Accumulation → 작은 GPU에서 큰 배치 사이즈
- DataLoader workers → CPU-GPU 파이프라인 최적화

---

## 4. 모니터링

**학습 중 확인:**
- Loss 수렴 추이
- 검증 정확도
- GPU 사용률 (`nvidia-smi`)
- 메모리 사용량

---

## 5. 결과 분석

```bash
ooreport run       # 학습 결과 리포트 생성
```

**분석 항목:**
- 학습/검증 Loss 그래프
- 혼동 행렬 (분류 문제)
- 클래스별 정밀도/재현율
- 추론 속도

---

## CUDA 폴백

GPU 사용 불가 시 자동 CPU 폴백:
```
CUDA 시도 → 실패 → CPU/int8로 자동 전환
```

Whisper STT도 동일한 폴백 적용:
```python
device = "cuda"      # 기본
compute_type = "float16"
# 실패 시:
device = "cpu"
compute_type = "int8"
```

---

## FAQ

**Q: GPU 메모리 부족하면?**
A: 배치 사이즈 줄이기, gradient accumulation, mixed precision 적용.

**Q: 학습이 수렴하지 않으면?**
A: learning rate 조정, 데이터 증강, 모델 구조 검토.
