# 시나리오: 환경 관리

> 환경 점검, 의존성 관리, 프로젝트 동기화, 백업

---

## 개요

OAIS 개발 환경의 건강 상태를 유지하는 워크플로우입니다. Python/uv/MCP 점검, 의존성 관리, 프로젝트 간 동기화, 백업을 포함합니다. 세션 시작 시 oostart가 기본 환경 체크를 하지만, 주기적으로 ooenv로 심층 점검이 필요합니다.

**이럴 때 사용:** 환경 문제 발생 시, 주기적 유지보수, 다른 프로젝트로 설정 배포 시
**결과물:** d0009_env.md + 동기화된 프로젝트 + zip 백업


## 1. 환경 전체 점검

```bash
ooenv run          # 전체 환경 점검 → d0009_env.md 갱신
```

**점검 항목:**
- Python 버전 (3.13+)
- uv 패키지 관리자
- MCP 서버 상태 (14개)
- 플러그인 설치 현황
- 환경변수 (PYTHONUTF8, SKILLSMP_API_KEY 등)

---

## 2. 의존성 관리

```bash
oouv check         # uv 의존성 무결성 확인
oouv update        # 패키지 업데이트
oouv add <pkg>     # 패키지 추가
oouv remove <pkg>  # 패키지 제거
```

**취약점 검사:**
```bash
oouv audit         # 알려진 취약점 검사
```

---

## 3. 프로젝트 동기화

### 동기화 대상 확인
```bash
oosync list        # 3_code/ 내 프로젝트 목록 + 차이 파일 수
```

### 일괄 동기화
```bash
oosync run --push-only           # push만 필요한 프로젝트
oosync run --push-only --delete  # 대상에만 있는 파일 삭제 허용
```

### 개별 동기화
```bash
oosync view 0001_SApp            # 차이점 비교 (읽기 전용)
oosync diff 0001_SApp CLAUDE.md  # 특정 파일 diff
oosync run 0001_SApp             # 대화형 동기화
```

### Pull 필요한 경우
대상 프로젝트가 최신인 파일이 있으면:
1. `oosync view`로 어떤 파일이 최신인지 확인
2. 해당 파일만 수동 `cp`
3. 다시 `oosync run --push-only`

## 4. 백업

```bash
oosync backup      # zip 백업 생성
```

**백업 대상:**
| 대상 | 설명 |
|------|------|
| `.claude/` | 스킬, 에이전트, 가이드, 설정 |
| `CLAUDE.md` | 프로젝트 루트 설정 |
| `.mcp.json` | MCP 서버 설정 |
| `pyproject.toml` | uv 의존성 |
| `00_doc/tutorial/` | 튜토리얼 문서 |
| `.omc/project-memory.json` | OMC 메모리 |

**저장 위치:** `data/04_claude_backup/YYMMDD-HHMMSS.zip`

---

## 5. 표준 검증 + 배포 파이프라인

```bash
oosync pipeline              # 표준 검증 → 동기화
oosync pipeline --dry-run    # 미리보기 (실제 배포 안 함)
oosync pipeline --force      # 검증 실패해도 강행
```

## 일상 루틴

| 시점 | 명령 |
|------|------|
| 세션 시작 | `oostart run` (환경 자동 체크) |
| 주 1회 | `ooenv run` + `oouv check` |
| 작업 완료 후 | `oosync run --push-only` |
| 세션 종료 전 | `oosync backup` |
