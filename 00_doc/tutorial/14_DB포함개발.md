# 개발 시나리오: DB 포함 개발

> DB 스키마 설계부터 구현, 마이그레이션까지의 흐름을 안내합니다.

---

## 개요

데이터베이스 스키마 변경이 포함된 개발 작업의 흐름입니다. oodb로 스키마를 설계하고, oodev로 연동 코드를 구현한 뒤, 마이그레이션까지 진행합니다.

**이럴 때 사용:** 새 테이블 추가, 컬럼 변경, 관계 설정 등 DB 구조가 바뀌는 개발 시
**결과물:** DB 스키마 + 모델/DAO 코드 + 마이그레이션 스크립트 + d0006_db.md


## 전체 흐름

```
oodb 설계 → oodev 구현 → ootest 검증 → oodb 마이그레이션
```

---

## 1. DB 설계

```bash
oodb run           # DB 스키마 설계/검증 → d0006_db.md
```

**설계 항목:**
- 테이블 구조 (컬럼, 타입, 제약조건)
- 관계 (1:N, N:M, FK)
- 인덱스 전략
- 마이그레이션 계획

**기존 DB 검증:**
```bash
oodb check         # 현재 스키마 정합성 검증
```

---

## 2. 구현

```bash
oodev run          # DB 연동 코드 구현 (모델, DAO, 서비스)
```

**구현 순서:**
1. 모델 정의 (ORM 또는 raw SQL)
2. CRUD 함수 작성
3. 서비스 레이어 연동
4. API 엔드포인트 연결 (필요 시)

---

## 3. 테스트

```bash
ootest run         # DB 연동 테스트
```

**테스트 항목:**
- CRUD 정상 동작
- 제약조건 위반 시 에러 처리
- 트랜잭션 롤백
- 동시 접근 (해당 시)

---

## 4. 마이그레이션

```bash
oodb migrate       # 스키마 변경 적용
```

**주의사항:**
- 프로덕션 DB 변경 전 백업 필수
- 롤백 스크립트 준비
- d0006_db.md에 변경 이력 기록
