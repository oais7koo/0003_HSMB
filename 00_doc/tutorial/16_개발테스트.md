# 16. 개발 테스트

> 참조: `.claude/guides/test_guide.md` | `.claude/skills/ootest/SKILL.md` | `.claude/skills/oocheck/SKILL.md`

OAIS 개발 환경의 전체 테스트 체계를 정리한다.  
테스트는 **5개 Part**로 구성되며 각각 목적과 도구가 다르다.

---

## 1. 테스트 구조 개요

| Part | 이름 | 목적 | 도구 | 자동화 |
|------|------|------|------|--------|
| A | 에러체크 | 정적 분석, 코드 품질 | pylint, mypy, py_compile | oocheck |
| B | 시나리오 | E2E/UI/통합 검증 | Playwright MCP | ootest |
| C | 단위 테스트 | TDD 기반 함수/클래스 | pytest | oodev, ootest |
| D | oo 모듈 테스트 | oo/*.py 전체 검증 | pytest | ootest |
| E | 런타임 검증 | import 시 런타임 에러 | pytest | ootest |

### 역할 분리

| 문서 | 역할 |
|------|------|
| `d*003_test.md` | **What** — 무엇을 테스트할 것인가 (TC 목록, 체크리스트) |
| `.claude/guides/test_guide.md` | **How** — 어떻게 테스트할 것인가 (방법론, 워크플로우) |

## 2. 빠른 실행 (ootest 명령어)

```bash
ootest run              # 전체 테스트 실행
ootest run --unit       # Part C 단위 테스트만
ootest run --e2e        # Part B 시나리오만
ootest run --module     # Part D oo 모듈만
ootest run --runtime    # Part E 런타임 검증만
ootest preview          # 테스트 계획 출력 (실행 없음)
```

pytest 직접 실행:
```bash
uv run pytest tests/ -v                          # 전체
uv run pytest tests/test_page_import.py -v       # Part E
uv run pytest tests/ -k "TC001" -v              # 특정 TC
```

## 3. Part A — 에러체크 (정적 분석)

코드 실행 없이 문법/타입/품질 문제를 사전에 감지한다.

```bash
# 문법 체크
python -m py_compile src/myfile.py

# 린터
pylint src/
flake8 src/

# 타입 체크
mypy src/

# oo 스킬로 통합 실행
oocheck run
```

결과는 `d*003_test.md` Part A 테이블에 기록:

```markdown
| 실행일 | 도구 | 대상 | 결과 | 비고 |
|--------|------|------|------|------|
| 2026-04-08 | pylint | src/ | 0 errors | W0621 다수 |
```

이슈 발견 시 → `d*004_todo.md` 등록 후 `oofix run`으로 수정.

## 4. Part B — 시나리오 테스트 (E2E)

사용자 관점의 전체 흐름 검증. PRD(`d*001_prd.md`) 기능 목록에서 시나리오를 도출한다.

### Playwright MCP 기본 흐름

```
browser_navigate → browser_snapshot → browser_click/type → browser_take_screenshot → 검증
```

| 도구 | 용도 |
|------|------|
| `browser_navigate` | URL 이동 |
| `browser_snapshot` | 페이지 스냅샷 |
| `browser_take_screenshot` | 스크린샷 캡처 |
| `browser_click` | 요소 클릭 |
| `browser_type` | 텍스트 입력 |
| `browser_fill_form` | 폼 전체 입력 |

스크린샷 저장 경로: `tmp/test_screenshots/[시나리오ID]_[단계]_[YYYYMMDD_HHMMSS].png`

### E2E 테스트 환경 설정

```env
AUTOLOGIN=true   # E2E 테스트 시 자동 로그인
```

| 환경 | AUTOLOGIN |
|------|-----------|
| 테스트 | true |
| 운영 | false |

## 5. Part C — 단위 테스트 (TDD)

### TDD 사이클

```
RED: 실패 테스트 작성 → d*003 Part C 등록
    ↓
GREEN: 최소 구현으로 통과
    ↓
REFACTOR: 코드 정리
    ↓
d*003 Part C 상태 업데이트 ([x])
```

### TC 네이밍 규칙

```
TC{Epic번호}-{Feature번호}.{Task번호}

예) TC002-1.1  →  tests/TC002-1.1_보건복지부탭.py
```

### 테스트 파일 구조

```
tests/
├── conftest.py                    # 공통 fixture
├── TC001-1.1_로그인성공.py
├── TC001-1.2_로그인실패.py
└── TC002-1.1_보건복지부탭.py
```

### 테스트 파일 템플릿

```python
"""

"""

    def test_tab_renders_correctly(self):
        """보건복지부 탭 정상 렌더링"""
        # Arrange
        # Act
        # Assert
        assert True
```

```markdown
| TC001-1.1 | F001-1.1 | 로그인 성공 | tests/TC001-1.1_로그인성공.py | [x] |
```

## 7. Part E — 런타임 검증 (필수)

정적 분석으로 잡히지 않는 런타임 에러를 사전에 감지한다.

| 에러 유형 | py_compile | pylint | Part E |
|----------|:----------:|:------:|:------:|
| StreamlitDuplicateElementKey | ❌ | ❌ | ✅ |
| 조건부 ImportError | ❌ | ❌ | ✅ |
| AttributeError (런타임) | ❌ | ❌ | ✅ |
| TypeError (런타임) | ❌ | ❌ | ✅ |

대상: `{서브프로젝트}/pages/*.py` 전체를 동적 import 하여 실행.

```bash
# Part E 실행
ootest run --runtime

# 직접 실행
uv run pytest tests/test_page_import.py -v
uv run pytest tests/test_page_import.py -k "7_74" -v   # 특정 페이지만
```

### 핵심 모킹 구조

```python
import importlib.util, sys

def test_page_import(page_file):
    spec = importlib.util.spec_from_file_location("page_module", page_file)
    module = importlib.util.module_from_spec(spec)
    sys.modules["page_module"] = module
    try:
        spec.loader.exec_module(module)
    except Exception as e:
        pytest.fail(f"Import failed: {page_file} - {e}")
```

템플릿: `.claude/templates/test_page_import_template.py`

## 9. 에러 분류 및 처리

| 분류 | 우선순위 | 대응 | 이력 태그 |
|------|---------|------|----------|
| [CRITICAL] | P0 — 즉시 | 작업 중단 후 수정 | [HOTFIX] |
| [ERROR] | P1 — 24시간 | d*004 등록 후 oofix | [BUGFIX] |
| [WARNING] | P2 — 1주일 | 백로그 등록 | [IMPROVE] |

### 이슈 처리 흐름

```
테스트 실패 감지
    ↓

    ↓
oofix run (자동 수정)
    ↓
재테스트 통과
    ↓

```

```
    ↓
[완료]

```
