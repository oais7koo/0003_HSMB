# 시나리오: 서브프로젝트 생성 및 관리

> SP 생성부터 문서 세팅, 컨텍스트 전환, 라이프사이클 관리까지

---

## 개요

OAIS는 서브프로젝트(SP) 체계로 프로젝트를 분리 관리합니다. 새 SP를 만들면 전용 문서 번호(d{SP}0001~d{SP}9999)가 할당되고, oocontext로 전환하면 모든 oo 스킬이 해당 SP 문서를 자동 참조합니다.

**이럴 때 사용:** 새 프로젝트를 시작할 때, SP 간 전환이 필요할 때, SP 문서 체계를 이해하고 싶을 때
**결과물:** SP 폴더 + 전용 PRD/Plan/Todo 문서 세트


## 전체 흐름

```
SP 폴더 생성 → oocontext N → ooprd run → ooplan run → 개발 → oodoc run
```

---

## 1. 서브프로젝트 생성

### SP 번호 체계
| SP | 폴더 | 문서 범위 |
|:--:|------|----------|
| 00 | (공통) | d0001~d9999 |
| 01 | 01_obsidian | d10001~d19999 |
| 02 | 02_pycode | d20001~d29999 |
| 03 | 03_paper | d30001~d39999 |
| 04 | 04_scraping | d40001~d49999 |
| 05 | 05_youtube_graphRAG | d50001~d59999 |
| 06 | 06_oohwp_skill | d60001~d69999 |

### 새 SP 폴더 생성
```bash
mkdir 07_새프로젝트명    # 07번 SP 생성
```

---

## 2. 컨텍스트 전환

```bash
oocontext 7            # SP07로 전환
```

**전환 후 효과:**
- 모든 oo 스킬이 SP07 문서 참조 (d70001~d79999)
- oocheck → d70004_todo.md에 이슈 등록
- oofix → d70004 + d0004 둘 다 확인

**컨텍스트 확인/초기화:**
```bash
oocontext              # 현재 컨텍스트 확인
oocontext list         # SP 목록 표시
oocontext clear        # 공통(SP00)으로 초기화
```

---

## 3. SP 문서 세팅

```bash
ooprd run              # d70001_prd.md 생성 (SP07 PRD)
ooplan run             # d70002_plan.md 생성 (구현 계획)
```

**자동 생성되는 문서:**
| 문서번호 | 파일 | 용도 |
|----------|------|------|
| d70001 | d70001_prd.md | 요구사항 정의 |
| d70002 | d70002_plan.md | 구현 계획 |
| d70004 | d70004_todo.md | 이슈 추적 (자동 생성) |
| d70010 | d70010_history.md | 변경 이력 |

**문서번호 공식:** `SP x 10000 + 기본번호` (예: SP07 → d70001)

---

## 4. 개발 진행

```bash
oodev run              # SP07 내 개발
oocheck run            # → d70004_todo.md에 이슈 등록
oofix run              # → d70004 + d0004 둘 다 확인
```

---

## 5. SP 간 전환

```bash
oocontext 2            # SP02로 전환 (다른 SP 작업)
# ... SP02 작업 ...
oocontext 7            # 다시 SP07로 복귀
```

**주의:** SP 전환 시 이전 SP의 미커밋 변경사항 먼저 커밋 권장

---

## 6. 문서 현행화

```bash
oodoc run              # SP07 문서 일괄 업데이트
oodoc check            # 교차 정합성 + SP 귀속 검사
oodoc clear            # 이력 5개 초과 행 제거
```

---

## FAQ

**Q: SP 번호를 건너뛸 수 있나?**
A: 가능. SP07 다음 SP10도 OK. 번호는 폴더 접두사와 매핑.

**Q: 공통 모듈 이슈는 어디에 등록?**
A: `oocontext clear` 후 d0004에 등록 권장.
