# oonow 스킬 튜토리얼

현재 세션의 작업 상태를 한눈에 파악하고 다음 단계를 빠르게 결정하는 스킬입니다.

## §1. 왜 필요한가?

개발 중 여러 작업이 동시에 진행될 때, 현재 어디까지 왔는지, 무엇이 남았는지 빠르게 파악해야 합니다.

**문제점**:
- 지난 세션 내용을 기억하지 못함
- 진행 상황을 매번 수동으로 확인해야 함
- 다음 단계를 결정하기 어려움
- TODO 목록과 실제 진행 상황이 불일치

**해결책**: `oonow`로 현재 상태를 즉시 확인하고 다음 액션을 결정합니다.

## §2. 빠른 시작 (3가지 자주 쓰는 명령)

```bash
# 1. 현재 상태 전체 조회
oonow

# 2. 작업 상태만 확인
oonow status

# 3. 다음 단계 추천받기
oonow next
```

## §3. 자주 쓰는 명령어

| 명령어 | 목적 | 사용 시점 |
|--------|------|----------|
| `oonow` | 전체 상태 + 진행률 + 남은 작업 | 세션 시작 후 |
| `oonow status` | 현재 진행 상황만 조회 | 빠른 체크 필요 |
| `oonow next` | 다음 할 일 추천 | 다음 단계 결정 시 |
| `oonow tasks` | TODO 목록 + 완료 상태 | 전체 작업 목록 확인 |
| `oonow summary` | 간단한 텍스트 요약 | 로깅/리포팅 시 |

## §4. 권장 워크플로우

**일반적인 사용 패턴**:

```
세션 시작
  ↓
oonow (현재 상태 파악)
  ↓
oonow next (다음 단계 추천)
  ↓
작업 실행
  ↓
완료 후 oonow (진행률 업데이트)
  ↓
세션 종료
```

## §5. 모든 명령어

- `oonow` — 전체 상태 표시
- `oonow status` — 상태만 조회
- `oonow next` — 다음 단계
- `oonow tasks` — TODO 목록
- `oonow summary` — 텍스트 요약
- `oonow --json` — JSON 출력
- `oonow --csv` — CSV 출력

## §6. 상세 사용법

### 6.1 전체 상태 조회 (`oonow`)

현재 세션의 모든 정보를 표시합니다:
- 활성 작업
- 완료된 작업
- 대기 중인 작업
- 총 진행률

### 6.2 상태 조회 (`oonow status`)

진행 상황만 간단히 조회합니다:
- 활성 작업 수
- 완료된 작업 수
- 전체 진행률

### 6.3 다음 단계 추천 (`oonow next`)

다음에 해야 할 일을 추천합니다:
- 우선순위 기반 정렬
- 의존성 고려
- 실행 가능 여부 판단

### 6.4 TODO 목록 조회 (`oonow tasks`)

현재 할당된 모든 작업을 나열합니다:
- 완료 상태 (✓/○/△)
- 우선순위
- 예상 시간

## §7. 실제 예시

**예시 1: 기능 개발 중 상태 확인**

```
$ oonow

=== 현재 상태 ===
활성 작업: 3개
완료: 2개
진행률: 40%

활성:
- [IN] 로그인 페이지 구현
- [IN] 데이터베이스 마이그레이션
- [PENDING] 성능 테스트

다음: oonow next
```

**예시 2: 다음 단계 결정**

```
$ oonow next

=== 추천 다음 단계 ===

1순위: 데이터베이스 마이그레이션 완료 (진행: 50%)
  └ 로그인 페이지 구현을 시작하기 전에 필수

2순위: 로그인 페이지 구현 (준비됨)
  └ 데이터베이스 준비 후 즉시 시작 가능

3순위: 성능 테스트 (예상: 2시간)
  └ 대기 중 - 구현 완료 후 시작
```

## §8. 입출력 정의

### 입력

**명령어 옵션**:
- `--json` — JSON 형식 출력
- `--csv` — CSV 형식 출력
- `--verbose` — 상세 정보 포함
- `--simple` — 간단한 형식

### 출력

**상태 객체**:
```json
{
  "active_tasks": 3,
  "completed_tasks": 2,
  "total_tasks": 5,
  "progress_percentage": 40,
  "next_recommended": "task_id",
  "blockers": []
}
```

## §9. FAQ

**Q: 이전 세션의 상태도 보여주나요?**  
A: 예, `.omc/state/` 폴더의 상태 파일을 참고합니다.

**Q: 작업 상태를 수동으로 업데이트할 수 있나요?**  
A: `oowork` 스킬로 작업을 수정할 수 있습니다.

**Q: 다음 단계 추천이 정확한가요?**  
A: 의존성과 우선순위를 기반으로 합니다. 수동 조정은 언제든 가능합니다.

**Q: JSON 출력은 어디에 쓰나요?**  
A: 자동화 스크립트, 대시보드, 외부 시스템과의 연동에 유용합니다.
