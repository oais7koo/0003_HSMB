# ooprevious Tutorial

> 이전 요청(직전) 내용과 결과 표시 스킬 | 버전: v03 | 카테고리: meta-util

## 1. 이 스킬은 왜 필요한가?

현재 요청 바로 직전의 사용자 요청 내용과 처리 결과를 표시합니다. 세션 히스토리에서 이전 작업의 내용, 결과, 변경 파일을 빠르게 확인할 수 있습니다.

## 2. 빠른 시작 (5분 가이드)

```bash
# 기본 실행
ooprevious run

# 상태 확인
ooprevious status

# 도움말
ooprevious help
```

## 3. 자주 쓰는 명령 요약

| # | 명령어 | 언제 쓰나 |
|---|--------|----------|
| 1 | `ooprevious` 또는 `ooprevious run` | 이전 요청 내용과 결과 표시 |
| 2 | `ooprevious status` | 동일 (status 별칭) |
| 3 | `ooprevious help` | 도움말 |

## 4. 상세 사용법

### 스킬 요약

| 항목 | 내용 |
|------|------|
| **핵심 역할** | 현재 요청 직전의 사용자 요청 내용과 결과 표시 |
| **하는 것** | 직전 요청-응답 단위의 내용과 결과 요약 출력 |
| **하지 않는 것** | 파일 수정, 작업 실행, 현재 상황 표시(→oonow) |
| **참조 범위** | 현재 세션 내 히스토리만 / 외부 파일·시스템 자동 포함 안 함 |
| **수정 대상** | 없음 (출력만) |
| **실행 레벨** | [수동] — 세션 히스토리 읽기만 |
| **에이전트 호환** | Claude Code 전용 — 세션 히스토리 접근 필요 / 다른 에이전트: 대화 히스토리 스크롤로 대체 |

### 동작 규칙

- **현재 요청 바로 직전** 사용자 메시지를 대상으로 함
- 대화 컨텍스트에서 직접 추출 (스크립트 불필요)
- `oonow`는 현재, `ooprevious`는 그 직전 요청
- 요청 원문을 인용하고, 처리 결과(완료/중단/커밋)를 간략 표시
