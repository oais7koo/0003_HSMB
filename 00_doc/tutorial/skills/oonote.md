## 1. 이 스킬은 왜 필요한가?

연구노트 작성 스킬

## 2. 빠른 시작 (5분 가이드)

```bash
# 기본 실행
oonote run

# 상태 확인
oonote status

# 도움말
oonote help
```

## 3. 전체 서브명령어

| 명령어 | 설명 |
|--------|------|
| `oonote help` | 서브명령어 목록 표시 |
| `oonote version` | 스킬 버전 정보 (v01) |
| `oonote status` | 서브명령어 리스트, 오늘/전체 노트 현황 |
| `oonote check` | references/checklist.md 기반 체크 및 리포팅 | 터미널 |
| `oonote run` | 연구노트 작성 실행 |
| `oonote update` | 현행화 — 최신 연구 내용 → 연구노트 갱신 | d{SP}0020_연구노트.md |
| `oonote update --dry-run` | 변경 예정 내용 미리 출력 (실제 수정 안 함) | 터미널 |
| `oonote show checklist` | 역할 수행 체크리스트 표시 | 터미널 |
| `oonote add checklist "항목"` | 체크리스트 항목 추가 | checklist.md |
| `oonote` | 오늘 연구노트 표시 (= today) |
| `oonote add "내용"` | **연구노트 추가** (현재 날짜/시간 자동 기록) |
| `oonote today` | 오늘 작성된 노트 표시 |
| `oonote list [날짜]` | 날짜별 조회 (예: `oonote list 2026-03-21`) |
| `oonote list --all` | 전체 날짜 목록 표시 |
| `oonote search "키워드"` | 키워드 전체 검색 |
| `oonote init` | 연구노트 파일 초기 생성 |

실행: `uv run python .claude/skills/oonote/scripts/oonote_run.py [args]`

## 4. 상세 사용법

### 스킬 요약

| 항목 | 내용 |
|------|------|
| **핵심 역할** | 날짜/시간 기반 연구노트를 d{SP}0020_연구노트.md에 작성·조회 |
| **하는 것** | 연구노트 항목 추가(타임스탬프 자동), 조회, 요약 |
| **하지 않는 것** | 옵시디언 노트(→oosidi), 이력 관리(→oohistory), 연구 수행(→ooresearch) |
| **참조 범위** | 현재 프로젝트 내부 파일만 / 외부 프로젝트 자동 포함 안 함 |
| **수정 대상** | `d{SP}0020_연구노트.md` |
| **실행 레벨** | [자동] — 입력 내용 즉시 파일에 기록 |
| **에이전트 호환** | 범용 — 파일 읽기·쓰기 작업 중심으로 모든 에이전트 처리 가능 |

### 노트 추가 워크플로우

### 3.1 `oonote add "내용"` 흐름

```
1. d{SP}0020_연구노트.md 존재 확인 (없으면 자동 생성)
2. 현재 날짜(YYYY-MM-DD) 섹션 확인
   - 오늘 날짜 섹션 없음 → 새 섹션 생성
   - 오늘 날짜 섹션 있음 → 해당 섹션 끝에 추가
3. 현재 시간(HH:MM) + 내용 기록
4. 문서이력관리 버전 자동 증가
```

### 3.2 노트 구조

```markdown

### 2026-03-21 (토)

### 10:30 | [실험] 모델 A 결과 확인
- Loss: 0.234, Accuracy: 91.2%
- 이전 대비 1.8% 향상
- 다음: 배치 크기 조정 실험 필요

**참조**: `00_doc/sp00/d0002_plan.md`

---

### 14:00 | [회의] 주간 미팅 메모
- ...

---
```
