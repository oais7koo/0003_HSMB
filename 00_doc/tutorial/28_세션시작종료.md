# 시나리오: 세션 시작/종료

> 매 세션의 첫 명령어(oostart)와 마지막 명령어(oostop) 사용법

---

## 개요

OAIS에서는 모든 작업 세션이 `oostart run`으로 시작하고 `oostop run`으로 끝납니다. oostart는 환경을 점검하고 문서를 로드하며, oostop은 작업 내역을 정리하고 문서를 동기화합니다.

**이럴 때 사용:** 매 Claude Code 세션의 시작과 끝
**결과물:** 환경 점검 리포트 + README.md 업데이트 + 00_doc/ 문서 동기화

## 전체 흐름

```
[세션 시작] oostart run → 작업 수행 → oostop run [세션 종료]
```

---

## Part 1: 세션 시작 (oostart run)

### 1. 실행

```bash
oostart run
```

### 2. 자동 수행 항목

| 순서 | 내용 | 확인 사항 |
|------|------|----------|
| 1 | common_guide.md 로드 | 공통 가이드 적용 |
| 2 | 서브프로젝트 감지 (01_*~10_*) | Active N/10 표시 |
| 3 | 문서 상태 점검 (PRD/TODO) | 미해결 이슈 수 |
| 4 | 환경 체크 | PYTHONUTF8=1 확인 |

### 3. 결과 확인

```
[oostart] 세션 시작
  서브프로젝트: Active 6 / 10
  [OK] PYTHONUTF8=1
  [OK] d0001_prd.md (v20)
  [WARN] d0004_todo.md: 미해결 이슈 5건
  준비 완료
```

**미해결 이슈가 있으면:**
- 즉시 처리할지, 이번 세션에서 별도 작업할지 판단
- 즉시 처리: `oofix run` → `oohistory run`

---

## Part 2: 세션 종료 (oostop run)

### 1. 종료 전 체크

작업을 마무리했는지 확인:
- 미커밋 변경사항이 있는지 (`git status`)
- 테스트가 통과하는지
- tmp/ 임시 파일이 있는지

### 2. 실행

```bash
oostop run
```

### 3. 자동 수행 항목

**1단계 - README.md 업데이트:**
```markdown
## 최근 작업 내역
- 날짜: 2026-03-25
- 작업: [이번 세션 작업 요약]
- 변경: [변경된 파일 목록]
- 다음: [후속 작업 안내]
```

**2단계 - 00_doc/*.md 동기화:**

| 문서 | 동기화 조건 |
|------|------------|
| d{SP}0001_prd.md | 기능 추가/변경이 있었을 때 |
| d{SP}0004_todo.md | 완료 항목 또는 새 이슈가 있을 때 |
| d0005_lib.md | 라이브러리 추가가 있었을 때 |
| d{SP}0010_history.md | 모든 변경 시 |

### 3. MCP 좀비 프로세스 정리 (Windows)

```bash
oostop cleanup    # node 좀비 프로세스 정리
```

---

## 개별 서브명령어 활용

```bash
oostop readme     # README.md만 갱신 (1단계만)
oostop sync       # 00_doc/*.md만 동기화 (2단계만)
oostop cleanup    # 좀비 프로세스 정리만
```

## FAQ

**Q: oostart를 안 하고 바로 작업해도 되나?**
A: 동작은 하지만, 환경 체크와 문서 로드를 건너뛰어 이슈를 놓칠 수 있습니다. CLAUDE.md에 "세션 시작 시 필수"로 지정되어 있습니다.

**Q: oostop 없이 세션을 닫으면?**
A: README.md와 문서가 업데이트되지 않아 다음 세션에서 컨텍스트가 끊깁니다.

**Q: 여러 SP를 오가며 작업했을 때?**
A: oostop은 마지막 활성 SP 기준으로 동기화합니다. 여러 SP 문서를 동시에 수정했다면 수동으로 `oodoc run`을 실행하는 것이 안전합니다.
