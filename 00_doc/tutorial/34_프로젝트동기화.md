# 시나리오: 프로젝트 동기화 및 백업

> OAIS ↔ 독립 프로젝트 간 vibe 환경 동기화 + Claude 환경 백업

---

## 개요

oosync 스킬로 현재 프로젝트(1_oais)와 3_code/ 내 독립 프로젝트 간에 Claude 환경 파일(.claude/, CLAUDE.md, .mcp.json 등)을 비교하고 동기화합니다. 또한 Claude 환경 전체를 zip으로 백업/복원할 수 있습니다.

**이럴 때 사용:** 새 프로젝트에 Claude 환경을 배포할 때, 프로젝트 간 설정을 맞출 때, 환경을 백업할 때
**결과물:** 동기화된 .claude/ 파일들 + (선택) data/04_claude_backup/*.zip

## 전체 흐름

```
oosync list → oosync view [project] → oosync run [project] → oosync backup
```

## 방법 1: 프로젝트 간 동기화

### 1. 동기화 가능 프로젝트 확인

```bash
oosync list      # 3_code/ 내 프로젝트 목록
oosync files     # 동기화 대상 파일/폴더 종류
```

### 2. 차이점 비교 (읽기 전용)

```bash
oosync view 1001_SeniorWorld     # 특정 프로젝트와 비교
oosync diff 1001_SeniorWorld CLAUDE.md  # 특정 파일 상세 비교
```

비교 상태 표시:

| 상태 | 기호 | 의미 |
|------|:----:|------|
| ONLY_SOURCE | `->` | 현재 프로젝트에만 있음 (push 가능) |
| ONLY_TARGET | `<-` | 대상에만 있음 (pull 가능) |
| NEWER_SOURCE | `>>` | 현재 프로젝트가 최신 |
| NEWER_TARGET | `<<` | 대상이 최신 |
| SAME | `==` | 동일 |

### 3. 동기화 실행

```bash
oosync run 1001_SeniorWorld        # 대화형 동기화 (파일별 방향 선택)
oosync run --push-only             # push만 필요한 모든 프로젝트 일괄
oosync run --push-only --add       # 대상 전용 파일 유지하며 일괄 동기화
```

### 4. 파일 병합 (양쪽 다 변경된 경우)

```bash
oosync merge 1001_SeniorWorld CLAUDE.md
```

병합 규칙:
- 문서이력관리: 양쪽 이력 통합 + 병합 버전 추가
- 서브명령어 표: 양쪽 명령어 합집합
- 의존성: 양쪽 패키지 합집합

## 방법 2: 파이프라인 (검증 → 배포)

```bash
oosync pipeline                    # 표준 검증(ooenv) → 배포(push-only)
oosync pipeline --dry-run          # 검증 후 미리보기만
oosync pipeline --force            # 검증 실패해도 배포 강행
oosync pipeline --skip-standard    # 검증 건너뛰고 바로 배포
```

## 방법 3: Claude 환경 백업/복원

### 백업

```bash
oosync backup
```

**백업 대상:**

| 파일/폴더 | 설명 |
|-----------|------|
| .claude/ | 스킬, 에이전트, 가이드, 템플릿, 설정 전체 |
| CLAUDE.md | 프로젝트 설정 |
| .mcp.json | MCP 서버 설정 |
| .omc/project-memory.json | OMC 프로젝트 메모리 |
| pyproject.toml | uv 의존성 |
| 00_doc/tutorial/ | 튜토리얼 문서 |
| cclaude.bat/sh | 실행 스크립트 |

**저장 위치:** `data/04_claude_backup/YYMMDD-HHMMSS.zip`

### 복원

```bash
oosync restore                     # 백업 목록 조회
oosync restore 260323-085605       # 특정 백업에서 복원
```

## FAQ

**Q: push-only와 일반 run 차이?**
A: `--push-only`는 현재 프로젝트 → 대상 방향만 일괄 처리합니다. `run`은 대화형으로 파일별 방향을 선택합니다.

**Q: 백업 용량은?**
A: .claude/ 폴더 크기에 따라 다르지만 보통 수 MB 수준입니다. data/ 폴더는 제외됩니다.

**Q: 새 프로젝트에 환경을 처음 세팅하려면?**
A: `oosync run --push-only` 또는 백업 zip을 풀어넣으면 됩니다. 이후 프로젝트별로 CLAUDE.md만 커스터마이징합니다.

**Q: settings.local.json은 왜 제외?**
A: 프로젝트별 로컬 설정(API 키, 경로 등)은 공유하면 안 되므로 의도적으로 제외합니다.
