# 개발 시나리오: 버그 수정 흐름

> 이슈 발견부터 수정, 검증, 아카이브까지의 전체 흐름을 안내합니다.

---

## 개요

코드에서 발견된 버그나 이슈를 체계적으로 수정하는 흐름입니다. oocheck가 자동 발견한 이슈든, 수동 등록한 이슈든 d0004_todo.md를 중심으로 추적하고, oofix로 자동 수정한 뒤, 해결 이력을 아카이브합니다.

**이럴 때 사용:** CRITICAL/ERROR 이슈가 발견되었을 때, 다수 이슈를 일괄 처리할 때
**결과물:** 수정된 코드 + d0004에서 d0010으로 이동된 해결 이력


## 전체 흐름

```
이슈 발견 → d0004_todo.md 등록 → oofix 수정 → oocheck 재검증 → oohistory 아카이브
```

---

## 1. 이슈 발견

### 자동 발견 (oocheck)
```bash
oocheck run        # 전체 코드 분석 → d0004_todo.md에 자동 등록
```

출력 예시:
```
[CRITICAL] S001: SQL injection 취약점 (graph_db.py:45)
[ERROR]    T005: ZeroDivisionError 가능성 (metrics.py:120)
[WARNING]  W003: sys.exit() 라이브러리 내 사용 (utils.py:88)
```

### 수동 등록
```bash
ootodo add "T015: 로그인 실패 시 에러 메시지 미표시 [ERROR]"
```

### 이슈 ID 체계
| 접두사 | 용도 | 예시 |
|--------|------|------|
| T | 일반 이슈 | T001, T002 |
| S | 보안 이슈 | S001, S002 |
| A | 자동화 발견 | A001, A002 |
| D | 의존성 | D001, D002 |

---

## 2. 이슈 수정

```bash
oofix run          # d0004_todo.md 기반 자동 수정
```

**oofix 처리 우선순위:**
1. [CRITICAL] → 즉시 수정
2. [ERROR] → 24시간 내
3. [WARNING] → 1주일 내
4. [INFO] → 백로그

**SP별 처리 규칙:**
- SP!=00인 경우: `d{SP}0004` + `d0004` 둘 다 확인
- 공통 모듈(oo) 이슈는 `d0004`에서 관리

---

## 3. 재검증

```bash
oocheck run        # 수정 후 재분석
```

**목표:** CRITICAL/ERROR 0건

이슈가 남아있으면:
```bash
oofix run          # 재수정 (반복)
```

---

## 4. 아카이브

```bash
oohistory run      # 해결된 이슈 → d0010_history.md 이동
```

**이동 전:**
```markdown
## 현재 이슈 (d0004_todo.md)
| ID | 내용 | 상태 |
| T005 | ZeroDivisionError | 해결됨 |
```

**이동 후:**
```markdown
## 해결된 이슈 (d0010_history.md)
| ID | 내용 | 해결일 | 해결방법 |
| T005 | ZeroDivisionError | 2026-03-22 | total==0 체크 추가 |
```

---

## 실전 예시: 다수 이슈 일괄 처리

```bash
# 1. 전체 분석
oocheck run
# → T008~T014, W001~W010 등록 (18건)

# 2. 일괄 수정
oofix run
# → 18건 자동 수정

# 3. 재검증
oocheck run
# → CRITICAL: 0, ERROR: 0 확인

# 4. 아카이브 + 커밋
oohistory run
oocommit run       # fix(SP00): oofix run 18건 자동 수정
```

## FAQ

**Q: oofix가 수정 못하는 이슈는?**
A: d0004_todo.md에 잔류합니다. 수동 수정 후 `oohistory run`으로 아카이브.

**Q: 이전에 해결한 이슈를 다시 보려면?**
A: `00_doc/sp00/d0010_history.md` (또는 SP별 `d{SP}0010_history.md`)
