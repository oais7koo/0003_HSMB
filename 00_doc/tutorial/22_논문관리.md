## 개요

학술 논문의 전체 라이프사이클을 관리합니다. PDF 수집부터 메타데이터 추출, 전문 마크다운 변환, 한글 번역, 서베이 분석, SOTA 연구까지 논문 관련 모든 작업을 지원합니다.

**이럴 때 사용:** 논문을 체계적으로 수집/정리할 때, 선행연구 서베이가 필요할 때, 논문 초안을 작성할 때
**결과물:** 논문 서머리 + 서베이 보고서 + 연구 동향 분석


## 전체 흐름

```
oopaper (수집/서머리) → oosurvey (서베이 분석) → ooresearch (SOTA 연구) → oosota (논문 작성)
```

## 2. 논문 수집/서머리

### 영문 논문 (8단계 파이프라인, EN)

```bash
oopaper run --lang en
```

**처리 순서** (v04+ 신규 순서):
1. **Phase 0**: `00_down/` → `11_paper_en/YYMMDD-HHMM/` 자동 이동 + 파일명 정규화
2. **Phase 1**: 미처리 논문 스캔 (상태: X/E/T/S/O)
3. **Phase 2**: 영문 전문 추출 (AI) → `*_03_*_전문(영어).md`
4. **Phase 3**: 한글 번역 (AI/translator 에이전트) → `*_04_*_전문(한글).md`
5. **Phase 4**: 서머리 생성 + 키워드 추출 (한글 전문 기반) → `*_00_*_서머리.md`
6. **Phase 5**: 참고문헌 추출 + 내부 매칭
7. **Phase 6**: 정밀 분석 → `05_분석.md`
8. **Phase 7**: paper_list.md 동기화 + 상태 갱신

**옵션**:
- `--lang ko`: 한국어 보고서 처리 (4단계)
- `--limit N`: 최대 N건만 처리
- `--phase [N]`: 특정 Phase만 실행
- `--dry-run`: 계획만 출력
- `--gemma`: Phase 2·4를 로컬 Gemma에 위임 (토큰 절감, 최종 Claude 검수)

### 국문 보고서 (4단계 파이프라인, KO)

```bash
oopaper run --lang ko
```

**처리 순서**:
1. **Phase 0**: `00_down/` 스캔 → 한국어 보고서 식별
2. **Phase 1**: `12_paper_ko/YYMMDD-HHMM/` 폴더 생성 → PDF 이동
3. **Phase 2**: 서머리 생성 (AI) → `*_00_*_서머리.md`
4. **Phase 3**: 한글 전문 추출 → `*_04_*_전문(한글).md`

## 3. 상태 체계 (5단계, EN만)

| 상태 | 설명 |
|------|------|
| **X** | 미처리 (PDF만 존재) |
| **E** | 영문 전문 추출 완료 |
| **T** | 한글 번역 완료 |
| **S** | 서머리 완료 (한글 전문 기반) |
| **O** | 완료 (서머리+영문+한글+참고문헌 매칭) |

KO는 X/S 2단계만 사용 (영문 추출/번역 없음).

## 4. 주요 명령어

### 상태 확인
```bash
oopaper status        # 현황 통계
oopaper check         # 체크리스트 기반 검증
```

### 추가 처리
```bash
oopaper anal --lang en               # 서베이 생성 또는 정밀 분석
oopaper anal --lang en --deep        # 정밀 분석 → 05_분석.md
oopaper anal --lang en --add [폴더]  # 새 논문 추가 → 기존 서베이 병합

oopaper search                       # 논문 검색 (arXiv, Semantic Scholar)
oopaper download                     # PDF 자동 다운로드 (리스트 파일 기반)

oopaper extend --limit 10            # 참고문헌 확장 수집 (외부 미보유 논문)
oopaper ref                          # 인용 통합 관리
oopaper keyword-sync                 # 키워드 동기화 (서머리 → paper_list)
oopaper ref-match                    # 참고문헌 내부 매칭
```

### 파일 관리
```bash
oopaper backup --dry-run             # PDF 백업 (dry-run으로 예정 내용 확인)
oopaper backup                       # 실제 백업 (03_paper → 외부 경로)
oopaper restore                      # PDF 복원 (백업 경로 → 03_paper)
```

### 자동화 명령 (v04+, oopaper_auto.py)
```bash
oopaper_auto organize                # Phase 0 자동화: 00_down → YYMMDD-HHMM 폴더 이동
oopaper_auto dedup --fuzzy           # 중복 검출 (정규화 + rapidfuzz)
oopaper_auto meta --limit 100        # PDF 메타데이터 자동 추출
oopaper_auto ref-match-fuzzy         # rapidfuzz 기반 참고문헌 매칭 (Claude 대체)
```

## 5. 한글 번역 2단계 프로세스 (v06+)

```
trans korean
├─ 1차: Gemma 가용성 체크
│   ├─ OK  → Gemma 자동 번역 (stage=1, v1_engine=gemma4:26b)
│   └─ 실패 → 사용자 확인 ("Gemma 미가용. Claude 단독 번역? y/n")
├─ 2차: Claude 검수 (옵션, 기본 활성)
│   ├─ Gemma 결과 + 영문 원문 입력
│   ├─ 오타/포맷/일관성 보정
│   └─ stage=2, v2_engine=claude-* 갱신
└─ 결과: 한글 번역 파일 + frontmatter 단계 표시
```

**플래그**:
- `--no-claude-review` — Gemma만 (1차에서 멈춤, 일괄 처리용)
- `--claude-only` — Gemma 건너뛰기 (Claude 직접 호출)
- `--stage 1|2` — 특정 단계만 처리

## 10. FAQ

**Q: PDF가 아닌 웹 논문은?**
A: `ooscrap read`로 웹페이지 전문 스크래핑 후 `04_읽을거리/`에 저장.

**Q: 논문 수가 많을 때 관리는?**
A: `oopaper status`로 현황 확인, SP03 문서 체계(d30001~d39999)로 관리.

**Q: Gemma 번역 후 Claude 검수를 건너뛸 수 있나?**
A: `--no-claude-review` 플래그 사용 (일괄 1차 번역만 처리).

**Q: 이미 진행 중인 논문을 계속하려면?**
A: `oopaper run this` 명령어 사용 (직전 논문 계속).
