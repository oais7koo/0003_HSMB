# 시나리오: 논문 관리

> 논문 수집부터 서머리, 서베이, SOTA 연구까지

---

## 개요

학술 논문의 전체 라이프사이클을 관리합니다. PDF 수집부터 메타데이터 추출, 전문 마크다운 변환, 한글 번역, 서베이 분석, SOTA 연구까지 논문 관련 모든 작업을 지원합니다.

**이럴 때 사용:** 논문을 체계적으로 수집/정리할 때, 선행연구 서베이가 필요할 때, 논문 초안을 작성할 때
**결과물:** 논문 서머리 + 서베이 보고서 + 연구 동향 분석


## 전체 흐름

```
oopaper (수집/서머리) → oosurvey (서베이 분석) → ooresearch (SOTA 연구) → oosota (논문 작성)
```

---

## 1. 논문 현황 확인

```bash
oocontext 3        # SP03 (03_paper) 전환
oopaper status     # 전체 논문 현황
```

출력 예시:
```
해외 논문: 362편
국내 보고서: 7편
합계: 369편 (100% 정상)
```

---

## 2. 논문 수집/서머리

### 영문 논문
```bash
oopaper run --lang en
```
**4단계 파이프라인:**
1. PDF 수집 → `D:/resilio/1_oais/03_paper/01_en/raw/`
2. 메타데이터 추출 (제목/저자/연도/DOI)
3. 영문 전문 마크다운 변환
4. 한글 번역본 생성

### 국문 보고서
```bash
oopaper run --lang ko
```

---

## 3. 서베이 분석

```bash
oosurvey run       # 논문 서베이 보고서 생성
```
- 수집된 논문 기반 선행연구 정리
- 연구 동향, 방법론 비교, 갭 분석

---

## 4. SOTA 연구

```bash
ooresearch run             # 최신 연구 동향 분석
ooresearch run --ai        # AI 도메인 특화
```
- 최신 SOTA 논문 검색 및 비교
- 벤치마크 성능 정리

---

## 5. 논문 작성 (선택)

```bash
oosota run         # 실험결과 → 저널 투고용 초안
```
- IEEE, Elsevier, Springer 형식 지원
- Abstract, Introduction, Method, Results, Conclusion 구조

## FAQ

**Q: PDF가 아닌 웹 논문은?**
A: `ooscrap read`로 웹페이지 전문 스크래핑 후 `04_읽을거리/`에 저장.

**Q: 논문 수가 많을 때 관리는?**
A: `oopaper status`로 현황 확인, SP03 문서 체계(d30001~d39999)로 관리.
