## 2. 현장 실험 설계 (Task A·B 공통)

### 카메라 및 장비
- 카메라: 8K area-scan, global shutter, 노출 50 μs
- 조명: 200 W LED strobe (노출 동기화)
- 렌즈: 24–85 mm 줌, autofocus (GSD = 0.3 mm/px 유지)

### 3-factor Factorial 설계
| 변수 | 수준 |
|------|------|
| 차량 속도 | 60, 80 km/h |
| 카메라-벽 거리 | 2.5, 3.5, 4.5, 5.5, 6.5 m |
| ISO 감도 | 100, 200, 400, 800, 1,600 |

- **총 조건 수**: 2 × 5 × 5 = **50조건**
- **유효 프레임 수**: 491장 (조건당 평균 9.8, 범위 7–13)
- **총 이미지**: 약 1,500장 (slant-edge 500장 + 일반 영역 1,000장)
- **ROI 측정 수**: 1,792개 (Imatest MTF50/BEW)

### 통계 검정력
- N=50 조건 단위, ρ≥0.5 탐지 → power ≥ 0.95 (α=0.05)
- Permutation test (10,000회) + Bootstrap 95% CI 예정

## 3. 특수 조건 — Complex-blur Stress Test

**조건**: ISO=400 — **의도적으로 설계한 defocus 조건** (complex-blur stress test)

| 항목 | 정상 조건 범위 | Complex-blur 조건 (ISO=400, d=4.5m) |
|------|---------------|--------------------------------------|
| BEW_H | 3.6–6.6 px | **7.78 px (60), 8.20 px (80)** |
| BEW_V | 3.5–6.3 px | **7.39 px (60), 7.65 px (80)** |
| MTF50_H | 0.08–0.15 cy/px | **0.065 cy/px (60), 0.061 cy/px (80)** |
| BEW_H − BEW_V | +0.37–+1.07 px | **+0.38 px (60), +0.55 px (80)** — 작음 |

**설계 의도 및 물리적 해석**:
- ISO 400 조건은 **의도적으로 defocus(초점 이탈)가 발생하도록 설계**한 실험 조건
- BEW 절대값이 높은 것은 motion blur + defocus 복합 블러의 결과
- BEW_H − BEW_V(이방성)가 **작은 것이 이론적으로 정합**: defocus는 isotropic(등방성) → H/V 축 차이 미미
- 순수 모션블러라면 H축만 열화되어 이방성이 커야 하지만, defocus는 양축을 동등하게 열화시킴

**⚠️ 논문 서술 수정 필요**:
- ~~"autofocus 수렴 실패로 추정"~~ → **"의도적 defocus 설계 조건"** 으로 교체
- `section06_draft.md` §VI-C complex-blur stress test 서술 전면 수정 대상

**활용**: 제거하지 않고 HSMB의 복합 블러 강건성 stress test로 활용.

## 5. Task B — CNN Crack Detection Downstream 분석

**목적**: Reviewer C3 정면 대응 (downstream task 실험 부재 지적 해소)

### 핵심 연구질문
- **RQ1**: 블러 수준 증가 → crack detection F1 감소 정량화
- **RQ2**: HSMB threshold 기반 pre-filtering → F1 개선 정량화

### 필요 자산
| 항목 | 담당 |
|------|------|
| ResNet34 crack detection 가중치 | **협업자 보유** |
| 균열 데이터셋 (이미지 + 라벨) | **협업자 보유** |
| 현장 area-scan 이미지 1,500장 | KICT 전달 |
| HSMB score | Task A 산출물 재사용 |

### 실험 단계 (6 Steps)

**Step 1**: Baseline (sharp) F1 측정 — clean 이미지 기준선

**Step 2**: 현장 블러 이미지 F1 측정 (RQ1 주실험)
- 라벨 확보 옵션:
  - **A** (권장): 현장 이미지 중 균열 영역 수동 라벨링 or 기존 라벨 활용
  - **B**: 합성 crack overlay
  - **C**: 합성 블러 이미지로 우선 진행

**Step 3**: 블러 수준 ↔ F1 관계 (bin 분석)

| 블러 구간 | BEW 범위 (px) |
|---|---|
| Low | < 4 |
| Mid | 4–5.5 |
| High | > 5.5 |

**Step 4**: HSMB threshold sweep (RQ2)
- T ∈ {0.1, 0.2, …, 0.9}
- 각 T에서 HSMB ≥ T인 이미지만 통과 → F1 재계산
- 대조군: unfiltered baseline, P_jnb=0.63 기존 threshold

**Step 5**: (HSMB_i, F1_i) scatter plot 데이터 저장

**Step 6**: 시각적 예시 3–5쌍 (blur 전후 predicted mask overlay PNG)

### 반환 파일 (필수)
1. `task_B_crack_detection_per_image.csv` — 이미지별 P/R/F1 + HSMB join key
2. `task_B_crack_detection_by_blur_bin.csv` — RQ1 블러 구간별 F1 요약
3. `task_B_threshold_sweep.csv` — RQ2 T–F1 curve (T ≥ 9 포인트)
4. `task_B_filtering_comparison.csv` — 3행: unfiltered / T=0.63 / T=optimal

## 8. HSMB 알고리즘 요약 (`scripts/hsmb_metric.py`)

```
입력 이미지
  → Sobel gradient (grad_x, grad_y, grad_mag)
  → 64×64 블록 분할
  → AEP 선택 (Active Edge Pixels): grad_mag > mean_grad × edge_weight(1.5)
  → AEP별 edge profile 추출 (방향: 4방향 또는 8방향)
  → Edge width 추정 (EVP 기반 sub-pixel 보간)
  → Blur probability: P_blur = 1 - exp(-(w/JNB)^β)
  → P_blur 히스토그램 → CDF
  → HSMB score = CDF값 at p_jnb(0.63)
출력: float [0, 1] (높을수록 선명)
```

**이상치 조건** (complex-blur):

| speed | iso | distance | bew_mean | mtf50_mean |
|-------|-----|----------|----------|------------|
| 60 | 400 | 4.5 | **7.64** | **0.066** |

거리 증가에 따른 BEW 증가 경향: 2.5 m → 6.5 m 구간에서 BEW 약 +1.5–2.0 px 증가.
속도 80 km/h가 60 km/h 대비 전반적으로 BEW 약 +0.3–0.5 px 높음.
